49. Group Anagrams
Medium

題目描述

給定一個字串陣列，將字母異位詞（anagrams）組合在一起。字母異位詞是由重新排列源單詞的字母得到的一個新單詞。

所有輸入均為小寫字母，不考慮答案輸出的順序。

範例 1:
輸入: strs = ["eat","tea","tan","ate","nat","bat"]
輸出: [["bat"],["nat","tan"],["ate","eat","tea"]]
範例 2:
輸入: strs = [""]
輸出: [[""]]
範例 3:
輸入: strs = ["a"]
輸出: [["a"]]

解題思路

方法：雜湊表 + 排序

兩個字串是字母異位詞，當且僅當它們的字母重新排列後相同。因此，我們可以將排序後的字串作為雜湊表的鍵。

核心思想：

相同的字母異位詞在排序後會得到相同的字串，可以用這個排序後的字串作為分組的依據。

步驟：

  1. 建立一個雜湊表（HashMap/Dictionary），鍵為排序後的字串，值為屬於該異位詞組的字串列表
  2. 遍歷輸入陣列中的每個字串
  3. 將當前字串排序，得到排序後的字串作為鍵
  4. 將原始字串加入到對應鍵的值列表中
  5. 返回雜湊表中所有的值列表

範例說明：

對於 ["eat", "tea", "tan", "ate"]：

  • "eat" 排序後 → "aet"
  • "tea" 排序後 → "aet"（與 "eat" 相同）
  • "tan" 排序後 → "ant"
  • "ate" 排序後 → "aet"（與 "eat" 相同）

因此 "eat", "tea", "ate" 會被分為一組。

程式碼實現

from collections import defaultdict

class Solution:
    def groupAnagrams(self, strs: List[str]) -> List[List[str]]:
        """
        使用排序後的字串作為雜湊表的鍵
        時間複雜度: O(n * k log k)，n 為字串數量，k 為字串平均長度
        空間複雜度: O(n * k)
        """
        # 使用 defaultdict 自動初始化空列表
        anagram_map = defaultdict(list)
        
        for s in strs:
            # 將字串排序作為鍵
            key = ''.join(sorted(s))
            # 將原始字串加入對應的列表
            anagram_map[key].append(s)
        
        # 返回所有分組
        return list(anagram_map.values())
#include <vector>
#include <unordered_map>
#include <algorithm>
#include <string>

class Solution {
public:
    /**
     * 使用排序後的字串作為雜湊表的鍵
     * 時間複雜度: O(n * k log k)
     * 空間複雜度: O(n * k)
     */
    std::vector<std::vector<std::string>> groupAnagrams(std::vector<std::string>& strs) {
        std::unordered_map<std::string, std::vector<std::string>> anagramMap;
        
        for (const std::string& s : strs) {
            // 複製字串並排序作為鍵
            std::string key = s;
            std::sort(key.begin(), key.end());
            
            // 將原始字串加入對應的列表
            anagramMap[key].push_back(s);
        }
        
        // 收集所有分組
        std::vector<std::vector<std::string>> result;
        for (auto& pair : anagramMap) {
            result.push_back(pair.second);
        }
        
        return result;
    }
};
import java.util.*;

class Solution {
    /**
     * 使用排序後的字串作為雜湊表的鍵
     * 時間複雜度: O(n * k log k)
     * 空間複雜度: O(n * k)
     */
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> map = new HashMap<>();
        
        for (String s : strs) {
            // 將字串轉為字元陣列並排序
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars);
            
            // 如果鍵不存在，建立新的列表
            if (!map.containsKey(key)) {
                map.put(key, new ArrayList<>());
            }
            
            // 將原始字串加入對應的列表
            map.get(key).add(s);
        }
        
        // 返回所有分組
        return new ArrayList<>(map.values());
    }
}

複雜度分析

時間複雜度: O(n * k log k)

其中 n 是字串的數量，k 是字串的平均長度。 對每個字串進行排序需要 O(k log k) 的時間，總共需要排序 n 個字串。

空間複雜度: O(n * k)

需要儲存所有 n 個字串，每個字串長度為 k。 雜湊表中的鍵和值都需要空間。

重點提示

  • 排序作為唯一標識：字母異位詞排序後會得到相同的字串，這是分組的關鍵
  • 替代方案：也可以使用字母計數陣列作為鍵，但在實作上較複雜
  • Python 技巧：使用 defaultdict(list) 可以自動初始化空列表，簡化程式碼
  • 注意事項：排序是穩定的操作，不會改變原始字串
