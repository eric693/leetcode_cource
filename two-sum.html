1. Two Sum
Easy

題目描述

給定一個整數陣列 nums 和一個整數目標值 target,請你在該陣列中找出和為目標值 target 的那兩個整數,並返回它們的陣列索引。


你可以假設每種輸入只會對應一個答案。但是,陣列中同一個元素在答案裡不能重複出現。


你可以按任意順序返回答案。

範例 1:
輸入: nums = [2,7,11,15], target = 9
輸出: [0,1]
解釋: 因為 nums[0] + nums[1] == 9,返回 [0, 1]。
範例 2:
輸入: nums = [3,2,4], target = 6
輸出: [1,2]

解題思路

方法一: 暴力解法 (Brute Force)

使用兩層迴圈遍歷所有可能的組合,檢查是否存在兩數之和等於目標值。

  • 時間複雜度: O(n²)
  • 空間複雜度: O(1)
方法二: 雜湊表 (Hash Table) - 推薦!

使用雜湊表(字典)來存儲已遍歷過的數字及其索引。對於每個數字,檢查 target - 當前數字 是否存在於雜湊表中。


演算法步驟:

  1. 建立一個空的雜湊表(字典)
  2. 遍歷陣列中的每個元素
  3. 計算補數: complement = target - nums[i]
  4. 檢查補數是否在雜湊表中:
    • 如果存在,返回兩個索引
    • 如果不存在,將當前數字和索引存入雜湊表

程式碼實現

Python - 雜湊表解法 最優解 
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        # 建立雜湊表,鍵為數字,值為索引
        hash_map = {}
        
        # 遍歷陣列
        for i, num in enumerate(nums):
            # 計算補數
            complement = target - num
            
            # 如果補數在雜湊表中,找到答案
            if complement in hash_map:
                return [hash_map[complement], i]
            
            # 將當前數字和索引存入雜湊表
            hash_map[num] = i
        
        return []
C++ - 雜湊表解法 最優解 
#include <vector>
#include <unordered_map>

class Solution {
public:
    std::vector<int> twoSum(std::vector<int>& nums, int target) {
        // 建立雜湊表,鍵為數字,值為索引
        std::unordered_map<int, int> hashMap;
        
        // 遍歷陣列
        for (int i = 0; i < nums.size(); i++) {
            // 計算補數
            int complement = target - nums[i];
            
            // 如果補數在雜湊表中,找到答案
            if (hashMap.find(complement) != hashMap.end()) {
                return {hashMap[complement], i};
            }
            
            // 將當前數字和索引存入雜湊表
            hashMap[nums[i]] = i;
        }
        
        return {};
    }
};
Java - 雜湊表解法 最優解 
import java.util.HashMap;
import java.util.Map;

class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 建立雜湊表,鍵為數字,值為索引
        Map<Integer, Integer> hashMap = new HashMap<>();
        
        // 遍歷陣列
        for (int i = 0; i < nums.length; i++) {
            // 計算補數
            int complement = target - nums[i];
            
            // 如果補數在雜湊表中,找到答案
            if (hashMap.containsKey(complement)) {
                return new int[] {hashMap.get(complement), i};
            }
            
            // 將當前數字和索引存入雜湊表
            hashMap.put(nums[i], i);
        }
        
        return new int[] {};
    }
}

複雜度分析

時間複雜度: O(n) - 只需遍歷陣列一次
空間複雜度: O(n) - 雜湊表最多存儲 n 個元素

為什麼雜湊表解法更優?

  • 暴力解法需要 O(n²) 時間,當陣列很大時效率低下
  • 雜湊表提供 O(1) 的查找時間,整體降低到 O(n)
  • 雖然使用了額外空間,但在大多數情況下是值得的權衡

重點提示

  1. 邊界條件: 確保陣列至少有兩個元素
  2. 重複元素: 注意處理陣列中有重複數字的情況
  3. 數據結構選擇: Python 用 dict,C++ 用 unordered_map,Java 用 HashMap
  4. 一次遍歷: 可以在查找的同時加入雜湊表,不需要預先建立
  5. 返回順序: 題目允許任意順序,但通常返回較小索引在前
