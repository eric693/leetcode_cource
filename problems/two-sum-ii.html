167. Two Sum II - Input Array Is Sorted
Easy

題目描述

給定一個已按升序排列的整數陣列 numbers，找出兩個數使得它們相加之和等於目標數 target。

函數應該返回這兩個數字的索引值 index1 和 index2，其中 index1 必須小於 index2。

注意：返回的答案（index1 和 index2）不是從零開始的，即第一個元素的索引為 1。

範例 1:
輸入: numbers = [2,7,11,15], target = 9
輸出: [1,2]
解釋: 2 與 7 之和等於目標數 9。因此 index1 = 1, index2 = 2
返回 [1, 2]
範例 2:
輸入: numbers = [2,3,4], target = 6
輸出: [1,3]
解釋: 2 與 4 之和等於目標數 6。因此 index1 = 1, index2 = 3
返回 [1, 3]
範例 3:
輸入: numbers = [-1,0], target = -1
輸出: [1,2]
解釋: -1 與 0 之和等於目標數 -1。因此 index1 = 1, index2 = 2
返回 [1, 2]
限制條件:
2 <= numbers.length <= 3 * 10^4
-1000 <= numbers[i] <= 1000
numbers 按升序排列
-1000 <= target <= 1000
題目保證有且只有一個有效答案
進階:

你可以只用 O(1) 的額外空間來實現嗎？

解題思路

關鍵差異：與 Two Sum (#1) 的區別

Two Sum (#1)：陣列未排序 → 使用哈希表 O(n) 時間 + O(n) 空間

Two Sum II (#167)：陣列已排序 → 使用雙指標 O(n) 時間 + O(1) 空間 ⭐

方法：雙指標（推薦）⭐

利用陣列已排序的特性，使用左右雙指標向中間靠攏。

核心思想：

  • 如果當前兩數之和 < target → 需要更大的數 → 左指標右移
  • 如果當前兩數之和 > target → 需要更小的數 → 右指標左移
  • 如果當前兩數之和 = target → 找到答案！

步驟：

  1. 初始化左指標 left = 0，右指標 right = n-1
  2. 當 left < right 時：
    • 計算 sum = numbers[left] + numbers[right]
    • 如果 sum = target → 返回 [left+1, right+1]（注意索引從 1 開始）
    • 如果 sum < target → left++
    • 如果 sum > target → right--

視覺化範例：numbers = [2,7,11,15], target = 9

索引: 0  1  2   3
數值: 2  7  11  15
      ↑           ↑
    left        right

步驟 1: sum = 2 + 15 = 17 > 9
       需要更小的數 → right--

索引: 0  1  2   3
數值: 2  7  11  15
      ↑      ↑
    left   right

步驟 2: sum = 2 + 11 = 13 > 9
       需要更小的數 → right--

索引: 0  1  2   3
數值: 2  7  11  15
      ↑  ↑
    left right

步驟 3: sum = 2 + 7 = 9 = 9 ✓
       找到答案！返回 [1, 2]
為什麼雙指標有效？

關鍵：陣列已排序

  • 左指標指向較小的數，右指標指向較大的數
  • 當和太小時，只能增大左邊的數（因為右邊已經是最大的了）
  • 當和太大時，只能減小右邊的數（因為左邊已經是最小的了）
  • 這樣可以保證不會錯過任何可能的組合

程式碼實現

class Solution:
    def twoSum(self, numbers: List[int], target: int) -> List[int]:
        """
        方法：雙指標（推薦）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        left = 0
        right = len(numbers) - 1
        
        while left < right:
            current_sum = numbers[left] + numbers[right]
            
            if current_sum == target:
                # 找到答案，注意索引從 1 開始
                return [left + 1, right + 1]
            elif current_sum < target:
                # 和太小，需要更大的數
                left += 1
            else:
                # 和太大，需要更小的數
                right -= 1
        
        # 題目保證有解，不會到這裡
        return []
    
    def twoSum_hashmap(self, numbers: List[int], target: int) -> List[int]:
        """
        方法二：哈希表（不推薦，空間複雜度較高）
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        seen = {}
        
        for i, num in enumerate(numbers):
            complement = target - num
            if complement in seen:
                return [seen[complement] + 1, i + 1]
            seen[num] = i
        
        return []

複雜度分析

時間複雜度: O(n)

雙指標最多遍歷整個陣列一次，其中 n 是陣列長度。

空間複雜度: O(1)

只使用兩個指標變數，不需要額外空間。

重點提示

  • 利用排序特性：這是本題的核心，使用雙指標 O(1) 空間
  • 索引從 1 開始：別忘了返回時要 +1
  • 保證有解：題目說有且只有一個答案，不需要處理無解情況
  • 與 Two Sum 對比：
    • Two Sum: 未排序 → 哈希表 O(n) 空間
    • Two Sum II: 已排序 → 雙指標 O(1) 空間 ⭐
  • 雙指標移動邏輯：
    • sum < target → left++（需要更大的數）
    • sum > target → right--（需要更小的數）
    • sum = target → 找到答案！
  • 為什麼不會錯過答案：因為排序保證了移動方向的唯一性
  • 面試技巧：
    • 強調利用"已排序"這個條件
    • 說明為什麼雙指標不會錯過答案
    • 對比 Two Sum 展現對問題的理解
