739. Daily Temperatures
Medium

題目描述

給定一個整數陣列 temperatures，表示每天的溫度，返回一個陣列 answer，其中 answer[i] 是指對於第 i 天，下一個更高溫度出現在幾天後。如果氣溫在這之後都不會升高，請在該位置用 0 來代替。

範例 1:
輸入: temperatures = [73,74,75,71,69,72,76,73]
輸出: [1,1,4,2,1,1,0,0]
解釋: 
第 0 天: 73°，下一個更高溫度是第 1 天的 74°，相差 1 天
第 1 天: 74°，下一個更高溫度是第 2 天的 75°，相差 1 天
第 2 天: 75°，下一個更高溫度是第 6 天的 76°，相差 4 天
...
範例 2:
輸入: temperatures = [30,40,50,60]
輸出: [1,1,1,0]
範例 3:
輸入: temperatures = [30,60,90]
輸出: [1,1,0]

解題思路

方法：單調遞減堆疊

使用堆疊維護一個單調遞減的溫度序列。當遇到更高的溫度時，可以一次性處理堆疊中所有較低的溫度。

核心思想：

堆疊中存儲索引，並保持對應溫度單調遞減。當遇到更高的溫度時，堆疊中所有較低溫度的「下一個更高溫度」就是當前溫度。

步驟：

  1. 初始化結果陣列 result，全部設為 0
  2. 建立一個空堆疊，用於存儲索引
  3. 遍歷溫度陣列：
    • 當堆疊不為空且當前溫度 > 堆疊頂部索引對應的溫度：
      • pop 堆疊頂部索引 prev_index
      • 計算天數差：result[prev_index] = 當前索引 - prev_index
      • 重複此過程直到堆疊為空或當前溫度 ≤ 堆疊頂部溫度
    • 將當前索引 push 到堆疊
  4. 返回結果陣列

視覺化範例：temperatures = [73, 74, 75, 71, 69]

i=0, T=73  堆疊: [0]               result: [0,0,0,0,0]
i=1, T=74  73<74, pop(0)          result: [1,0,0,0,0]
           堆疊: [1]
i=2, T=75  74<75, pop(1)          result: [1,1,0,0,0]
           堆疊: [2]
i=3, T=71  71<75, push(3)         result: [1,1,0,0,0]
           堆疊: [2,3]
i=4, T=69  69<71, push(4)         result: [1,1,0,0,0]
           堆疊: [2,3,4]

程式碼實現

class Solution:
    def dailyTemperatures(self, temperatures: List[int]) -> List[int]:
        """
        單調遞減堆疊
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        n = len(temperatures)
        result = [0] * n
        stack = []  # 存儲索引
        
        for i in range(n):
            # 當當前溫度高於堆疊頂部索引對應的溫度
            while stack and temperatures[i] > temperatures[stack[-1]]:
                prev_index = stack.pop()
                result[prev_index] = i - prev_index
            
            # 將當前索引加入堆疊
            stack.append(i)
        
        return result
#include <vector>
#include <stack>

class Solution {
public:
    /**
     * 單調遞減堆疊
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    std::vector<int> dailyTemperatures(std::vector<int>& temperatures) {
        int n = temperatures.size();
        std::vector<int> result(n, 0);
        std::stack<int> stack;  // 存儲索引
        
        for (int i = 0; i < n; i++) {
            // 當當前溫度高於堆疊頂部索引對應的溫度
            while (!stack.empty() && 
                   temperatures[i] > temperatures[stack.top()]) {
                int prevIndex = stack.top();
                stack.pop();
                result[prevIndex] = i - prevIndex;
            }
            
            // 將當前索引加入堆疊
            stack.push(i);
        }
        
        return result;
    }
};
import java.util.Stack;

class Solution {
    /**
     * 單調遞減堆疊
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    public int[] dailyTemperatures(int[] temperatures) {
        int n = temperatures.length;
        int[] result = new int[n];
        Stack<Integer> stack = new Stack<>();  // 存儲索引
        
        for (int i = 0; i < n; i++) {
            // 當當前溫度高於堆疊頂部索引對應的溫度
            while (!stack.isEmpty() && 
                   temperatures[i] > temperatures[stack.peek()]) {
                int prevIndex = stack.pop();
                result[prevIndex] = i - prevIndex;
            }
            
            // 將當前索引加入堆疊
            stack.push(i);
        }
        
        return result;
    }
}

複雜度分析

時間複雜度: O(n)

每個元素最多被 push 和 pop 各一次，所以總時間複雜度是 O(n)。

空間複雜度: O(n)

最壞情況下（溫度遞減），堆疊需要存儲所有索引。

重點提示

  • 單調堆疊：這是一個經典的單調遞減堆疊應用，用於找「下一個更大元素」
  • 為什麼存索引：需要計算天數差，所以存索引而非溫度值
  • 堆疊維護：堆疊中的索引對應的溫度始終保持遞減順序
  • 相似題目：這個模式可以用於解決「下一個更大元素」、「接雨水」等問題
  • 暴力解法：O(n²) - 對每個元素向後搜索，不推薦
