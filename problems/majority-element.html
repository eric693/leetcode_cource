169. Majority Element
Easy

題目描述

給定一個大小為 n 的陣列 nums，返回其中的多數元素。

多數元素是指在陣列中出現次數大於 ⌊n/2⌋ 的元素。

你可以假設陣列是非空的，並且給定的陣列總是存在多數元素。

範例 1:
輸入: nums = [3,2,3]
輸出: 3
範例 2:
輸入: nums = [2,2,1,1,1,2,2]
輸出: 2
限制條件:
n == nums.length
1 <= n <= 5 * 10^4
-10^9 <= nums[i] <= 10^9
進階挑戰:

你能設計時間複雜度為 O(n)、空間複雜度為 O(1) 的演算法來解決此問題嗎？

解題思路

核心觀察

多數元素的特性：

  • 出現次數 > n/2
  • 一定存在（題目保證）
  • 只有一個多數元素
  • 出現次數大於所有其他元素的總和

關鍵洞察：多數元素的數量比其他所有元素加起來還多！

方法一：Boyer-Moore 投票算法（最優）⭐⭐⭐

這是一個非常巧妙的算法，被稱為摩爾投票算法。

核心思想：

想像一場戰鬥，多數元素和其他元素互相抵消，最後剩下的一定是多數元素。

算法步驟：

  1. 維護一個候選人（candidate）和計數器（count）
  2. 遍歷陣列：
    • 如果 count = 0，選擇當前元素為候選人
    • 如果當前元素 = 候選人，count++
    • 如果當前元素 ≠ 候選人，count--
  3. 最後的候選人就是多數元素

視覺化範例：nums = [2,2,1,1,1,2,2]

i=0, num=2:
  count=0，選擇 2 為候選人
  candidate=2, count=1

i=1, num=2:
  2 == candidate(2)
  candidate=2, count=2

i=2, num=1:
  1 != candidate(2)
  candidate=2, count=1

i=3, num=1:
  1 != candidate(2)
  candidate=2, count=0

i=4, num=1:
  count=0，選擇 1 為候選人
  candidate=1, count=1

i=5, num=2:
  2 != candidate(1)
  candidate=1, count=0

i=6, num=2:
  count=0，選擇 2 為候選人
  candidate=2, count=1

最終候選人: 2 ✓

計數驗證: 2 出現 4 次，1 出現 3 次
4 > 7/2 = 3.5，所以 2 是多數元素

為什麼這個算法正確？

關鍵證明：

1. 多數元素出現次數 > n/2
2. 其他所有元素出現次數 < n/2
3. 當多數元素和其他元素"配對抵消"時：
   - 最壞情況：每個多數元素都和一個非多數元素抵消
   - 多數元素有 > n/2 個
   - 非多數元素有 < n/2 個
   - 抵消後，多數元素還會剩下
4. 所以最後剩下的一定是多數元素

比喻：
想像一場投票，多數黨 vs 其他黨派
- 多數黨有 100 票
- 其他黨派加起來只有 99 票
- 每次一對一抵消
- 最後多數黨還剩 1 票

優點：

  • 時間複雜度：O(n)
  • 空間複雜度：O(1)
  • 只需遍歷一次
  • 最優雅的解法
方法二：HashMap 計數

統計每個元素出現次數，找出大於 n/2 的。

  • 時間複雜度：O(n)
  • 空間複雜度：O(n)
  • 最直觀的方法
方法三：排序

排序後，中間位置（n/2）的元素一定是多數元素。

為什麼？

  • 多數元素出現 > n/2 次
  • 排序後，這些元素會連在一起
  • 跨越中間位置
  • 所以 nums[n/2] 一定是多數元素
  • 時間複雜度：O(n log n)
  • 空間複雜度：O(1) 或 O(n)（看排序算法）
方法四：隨機化

隨機選擇元素，檢查是否是多數元素。

  • 期望時間複雜度：O(n)
  • 每次有 > 50% 的機率選中多數元素
  • 不是確定性算法

程式碼實現

class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        """
        方法一：Boyer-Moore 投票算法（最優）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        candidate = None
        count = 0
        
        # 找出候選人
        for num in nums:
            if count == 0:
                candidate = num
            count += 1 if num == candidate else -1
        
        return candidate
    
    def majorityElement_hashmap(self, nums: List[int]) -> int:
        """
        方法二：HashMap 計數
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        from collections import Counter
        counts = Counter(nums)
        return max(counts.keys(), key=counts.get)
        
        # 或者
        # for num, count in counts.items():
        #     if count > len(nums) // 2:
        #         return num
    
    def majorityElement_sort(self, nums: List[int]) -> int:
        """
        方法三：排序
        時間複雜度: O(n log n)
        空間複雜度: O(1)
        """
        nums.sort()
        return nums[len(nums) // 2]
    
    def majorityElement_random(self, nums: List[int]) -> int:
        """
        方法四：隨機化
        期望時間複雜度: O(n)
        """
        import random
        
        while True:
            candidate = random.choice(nums)
            if nums.count(candidate) > len(nums) // 2:
                return candidate
#include <vector>
#include <unordered_map>
#include <algorithm>
using namespace std;

class Solution {
public:
    /**
     * 方法一：Boyer-Moore 投票算法
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int majorityElement(vector& nums) {
        int candidate = 0;
        int count = 0;
        
        for (int num : nums) {
            if (count == 0) {
                candidate = num;
            }
            count += (num == candidate) ? 1 : -1;
        }
        
        return candidate;
    }
    
    /**
     * 方法二：HashMap 計數
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    int majorityElement_hashmap(vector& nums) {
        unordered_map counts;
        int n = nums.size();
        
        for (int num : nums) {
            counts[num]++;
            if (counts[num] > n / 2) {
                return num;
            }
        }
        
        return -1;  // 不會執行到這裡
    }
    
    /**
     * 方法三：排序
     * 時間複雜度: O(n log n)
     * 空間複雜度: O(1)
     */
    int majorityElement_sort(vector& nums) {
        sort(nums.begin(), nums.end());
        return nums[nums.size() / 2];
    }
};
import java.util.HashMap;
import java.util.Map;
import java.util.Arrays;

class Solution {
    /**
     * 方法一：Boyer-Moore 投票算法
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int majorityElement(int[] nums) {
        int candidate = 0;
        int count = 0;
        
        for (int num : nums) {
            if (count == 0) {
                candidate = num;
            }
            count += (num == candidate) ? 1 : -1;
        }
        
        return candidate;
    }
    
    /**
     * 方法二：HashMap 計數
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    public int majorityElement_hashmap(int[] nums) {
        Map counts = new HashMap<>();
        int n = nums.length;
        
        for (int num : nums) {
            counts.put(num, counts.getOrDefault(num, 0) + 1);
            if (counts.get(num) > n / 2) {
                return num;
            }
        }
        
        return -1;  // 不會執行到這裡
    }
    
    /**
     * 方法三：排序
     * 時間複雜度: O(n log n)
     * 空間複雜度: O(1)
     */
    public int majorityElement_sort(int[] nums) {
        Arrays.sort(nums);
        return nums[nums.length / 2];
    }
}

複雜度分析

時間複雜度: O(n)

Boyer-Moore 算法只需遍歷陣列一次。

空間複雜度: O(1)

只使用兩個變數（candidate 和 count）。

重點提示

  • Boyer-Moore 算法精髓：
    • 抵消思想：不同元素互相抵消
    • 多數元素數量 > 其他元素總和
    • 最後剩下的一定是多數元素
  • 為什麼不需要驗證？
    • 題目保證存在多數元素
    • 如果不保證，需要第二次遍歷驗證
    • 驗證：計算候選人出現次數是否 > n/2
  • 算法正確性證明：
    • 設多數元素為 M，出現 m 次（m > n/2）
    • 其他元素總共出現 n - m 次（< n/2）
    • 最壞情況：每個 M 都和非 M 配對抵消
    • 抵消 n - m 對後，還剩 m - (n - m) = 2m - n 個 M
    • 因為 m > n/2，所以 2m - n > 0
    • 最後一定剩下 M
  • 排序法為什麼正確？
    • 多數元素出現 > n/2 次
    • 排序後會形成連續區間
    • 區間長度 > n/2
    • 必然包含中間位置 nums[n/2]
  • 各方法比較：
    • Boyer-Moore: O(n) 時間，O(1) 空間 ⭐最優
    • HashMap: O(n) 時間，O(n) 空間
    • 排序: O(n log n) 時間，O(1) 空間
    • 隨機化: O(n) 期望時間，非確定性
  • 面試技巧：
    • 先說 HashMap 暴力解
    • 提到排序法（展示思考）
    • 最後說 Boyer-Moore（展示深度）
    • 解釋為什麼算法正確
  • 常見錯誤：
    • count 初始化錯誤
    • 忘記 count == 0 時更新候選人
    • 誤以為需要驗證候選人
  • 邊界情況：
    • nums = [1] → 1
    • nums = [1,1] → 1
    • 所有元素相同
  • 延伸思考：
    • 如果沒有多數元素怎麼辦？需要第二次驗證
    • 如果要找出現 > n/3 次的元素？（變形題）
    • 如果要找所有出現 > n/k 次的元素？
  • 實戰應用：
    • 選舉制度（找出獲得多數票的候選人）
    • 錯誤檢測（找出主要錯誤類型）
    • 數據分析（找出主要模式）
  • 變形題：
    • Majority Element II (#229) - 找 > n/3 的元素
    • 最多需要兩個候選人
  • 記憶技巧：「多數元素像是戰場上的勝利者，即使和敵人一對一抵消，最後還會剩下」
  • 實際應用：投票系統、數據流分析、網路流量監控
