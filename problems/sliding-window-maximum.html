239. Sliding Window Maximum
Hard

題目描述

給定一個整數陣列 nums，有一個大小為 k 的滑動窗口從陣列的最左側移動到最右側。你只可以看到在滑動窗口內的 k 個數字。滑動窗口每次只向右移動一位。

返回滑動窗口中的最大值。

範例 1:
輸入: nums = [1,3,-1,-3,5,3,6,7], k = 3
輸出: [3,3,5,5,6,7]
解釋: 
滑動窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7
範例 2:
輸入: nums = [1], k = 1
輸出: [1]
限制條件:
1 <= nums.length <= 10^5
-10^4 <= nums[i] <= 10^4
1 <= k <= nums.length

解題思路

方法：單調遞減佇列（Monotonic Deque）⭐

使用雙端佇列（deque）維護一個單調遞減的索引序列。

核心思想：

佇列中保存索引，且對應的值是遞減的。佇列頭部永遠是當前窗口的最大值。

為什麼有效？

  • 如果新元素比佇列尾部的元素大，則尾部元素永遠不會成為最大值 → 移除
  • 佇列頭部的索引如果已經滑出窗口 → 移除
  • 佇列頭部永遠是當前窗口的最大值索引

步驟：

  1. 遍歷陣列，對每個元素：
    • 移除佇列中所有小於當前元素的索引（從尾部移除）
    • 移除超出窗口範圍的索引（從頭部移除）
    • 將當前索引加入佇列尾部
    • 如果窗口已形成（i >= k-1），記錄佇列頭部對應的值

視覺化範例：nums = [1,3,-1,-3,5], k = 3

i=0, num=1:
  deque: [0]  (值: [1])
  窗口未形成

i=1, num=3:
  3 > 1 → 移除索引 0
  deque: [1]  (值: [3])
  窗口未形成

i=2, num=-1:
  -1 < 3 → 保留
  deque: [1, 2]  (值: [3, -1])
  窗口 [1,3,-1] → max = nums[1] = 3 ✓

i=3, num=-3:
  -3 < -1 → 保留
  deque: [1, 2, 3]  (值: [3, -1, -3])
  窗口 [3,-1,-3] → max = nums[1] = 3 ✓

i=4, num=5:
  5 > -3 → 移除索引 3
  5 > -1 → 移除索引 2
  5 > 3 → 移除索引 1
  索引 1 已滑出窗口（4-1=3 > 1）
  deque: [4]  (值: [5])
  窗口 [-1,-3,5] → max = nums[4] = 5 ✓

結果: [3, 3, 5]
為什麼不用其他方法？
  • 暴力法 O(nk)：對每個窗口遍歷找最大值 → 太慢
  • 堆積 O(n log n)：無法高效移除窗口外的元素
  • 單調佇列 O(n) ⭐：每個元素最多進出佇列一次

程式碼實現

from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        """
        方法：單調遞減佇列
        時間複雜度: O(n)
        空間複雜度: O(k)
        """
        if not nums or k == 0:
            return []
        
        dq = deque()  # 存儲索引
        result = []
        
        for i in range(len(nums)):
            # 移除超出窗口範圍的索引
            while dq and dq[0] < i - k + 1:
                dq.popleft()
            
            # 移除所有小於當前元素的索引
            while dq and nums[dq[-1]] < nums[i]:
                dq.pop()
            
            # 加入當前索引
            dq.append(i)
            
            # 窗口形成後，記錄最大值
            if i >= k - 1:
                result.append(nums[dq[0]])
        
        return result
#include <vector>
#include <deque>

class Solution {
public:
    /**
     * 方法：單調遞減佇列
     * 時間複雜度: O(n)
     * 空間複雜度: O(k)
     */
    vector maxSlidingWindow(vector& nums, int k) {
        if (nums.empty() || k == 0) return {};
        
        deque dq;  // 存儲索引
        vector result;
        
        for (int i = 0; i < nums.size(); i++) {
            // 移除超出窗口範圍的索引
            while (!dq.empty() && dq.front() < i - k + 1) {
                dq.pop_front();
            }
            
            // 移除所有小於當前元素的索引
            while (!dq.empty() && nums[dq.back()] < nums[i]) {
                dq.pop_back();
            }
            
            // 加入當前索引
            dq.push_back(i);
            
            // 窗口形成後，記錄最大值
            if (i >= k - 1) {
                result.push_back(nums[dq.front()]);
            }
        }
        
        return result;
    }
};
import java.util.*;

class Solution {
    /**
     * 方法：單調遞減佇列
     * 時間複雜度: O(n)
     * 空間複雜度: O(k)
     */
    public int[] maxSlidingWindow(int[] nums, int k) {
        if (nums == null || nums.length == 0 || k == 0) {
            return new int[0];
        }
        
        Deque dq = new LinkedList<>();  // 存儲索引
        int[] result = new int[nums.length - k + 1];
        int idx = 0;
        
        for (int i = 0; i < nums.length; i++) {
            // 移除超出窗口範圍的索引
            while (!dq.isEmpty() && dq.peekFirst() < i - k + 1) {
                dq.pollFirst();
            }
            
            // 移除所有小於當前元素的索引
            while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) {
                dq.pollLast();
            }
            
            // 加入當前索引
            dq.offerLast(i);
            
            // 窗口形成後，記錄最大值
            if (i >= k - 1) {
                result[idx++] = nums[dq.peekFirst()];
            }
        }
        
        return result;
    }
}

複雜度分析

時間複雜度: O(n)

每個元素最多進出佇列一次，總共 2n 次操作。

空間複雜度: O(k)

佇列中最多存儲 k 個索引（窗口大小）。

重點提示

  • 單調佇列：維護遞減的索引序列
  • 佇列存索引：而非存值，方便判斷是否在窗口內
  • 兩次移除：
    • 從頭部移除：超出窗口的索引
    • 從尾部移除：比當前元素小的索引
  • 佇列性質：
    • 頭部：窗口最大值的索引
    • 遞減：確保能快速找到次大值
  • 常見錯誤：
    • 佇列存值而非索引
    • 忘記移除超出窗口的元素
    • 窗口起始判斷錯誤（應該是 i >= k-1）
  • 面試技巧：
    • 先說暴力法 O(nk)
    • 解釋為什麼堆積不夠優
    • 引入單調佇列概念
    • 用例子演示佇列變化
