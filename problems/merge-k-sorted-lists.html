23. Merge k Sorted Lists
Hard

題目描述

給你一個鏈表陣列，每個鏈表都已經按升序排列。

請你將所有鏈表合併到一個升序鏈表中，返回合併後的鏈表。

範例 1:
輸入: lists = [[1,4,5],[1,3,4],[2,6]]
輸出: [1,1,2,3,4,4,5,6]
解釋: 鏈表陣列如下：
[
  1→4→5,
  1→3→4,
  2→6
]
將它們合併到一個有序鏈表中得到：
1→1→2→3→4→4→5→6
範例 2:
輸入: lists = []
輸出: []
範例 3:
輸入: lists = [[]]
輸出: []
限制條件:
k == lists.length
0 <= k <= 10^4
0 <= lists[i].length <= 500
-10^4 <= lists[i][j] <= 10^4
lists[i] 按升序排列
lists[i].length 的總和不超過 10^4

解題思路

方法一：最小堆積 / 優先佇列（推薦）⭐

使用最小堆積（Min Heap）維護 k 個鏈表的當前最小節點，每次取出最小值加入結果鏈表。

核心思想：

在 k 個已排序鏈表中，每次都要找出當前最小的節點。使用堆積可以在 O(log k) 時間內找到最小值。

步驟：

  1. 將每個鏈表的頭節點加入最小堆積
  2. 每次從堆積中取出最小節點
    • 將該節點加入結果鏈表
    • 如果該節點有下一個節點，將下一個節點加入堆積
  3. 重複直到堆積為空
  4. 返回結果鏈表

視覺化範例：lists = [[1,4,5],[1,3,4],[2,6]]

初始堆積: [1(L0), 1(L1), 2(L2)]

步驟 1: 取出 1(L0)，加入 4(L0)
       堆積: [1(L1), 2(L2), 4(L0)]
       結果: 1

步驟 2: 取出 1(L1)，加入 3(L1)
       堆積: [2(L2), 3(L1), 4(L0)]
       結果: 1→1

步驟 3: 取出 2(L2)，加入 6(L2)
       堆積: [3(L1), 4(L0), 6(L2)]
       結果: 1→1→2

步驟 4: 取出 3(L1)，加入 4(L1)
       堆積: [4(L0), 4(L1), 6(L2)]
       結果: 1→1→2→3

步驟 5: 取出 4(L0)，加入 5(L0)
       堆積: [4(L1), 5(L0), 6(L2)]
       結果: 1→1→2→3→4

步驟 6: 取出 4(L1)，L1 結束
       堆積: [5(L0), 6(L2)]
       結果: 1→1→2→3→4→4

步驟 7: 取出 5(L0)，L0 結束
       堆積: [6(L2)]
       結果: 1→1→2→3→4→4→5

步驟 8: 取出 6(L2)，L2 結束
       堆積: []
       結果: 1→1→2→3→4→4→5→6
方法二：分治法（Divide and Conquer）

類似歸併排序，兩兩合併鏈表，直到只剩一個鏈表。

優點：不需要額外的堆積空間，時間複雜度相同。

時間複雜度：O(N log k)，每層合併 O(N)，共 log k 層。

程式碼實現

import heapq

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        """
        方法一：最小堆積
        時間複雜度: O(N log k)
        空間複雜度: O(k)
        """
        # 最小堆積
        heap = []
        
        # 將每個鏈表的第一個節點加入堆積
        # 使用索引 i 作為唯一標識，避免節點值相同時的比較問題
        for i, node in enumerate(lists):
            if node:
                heapq.heappush(heap, (node.val, i, node))
        
        # 虛擬頭節點
        dummy = ListNode(0)
        current = dummy
        
        # 從堆積中逐個取出最小節點
        while heap:
            val, i, node = heapq.heappop(heap)
            current.next = node
            current = current.next
            
            # 如果該節點有下一個節點，加入堆積
            if node.next:
                heapq.heappush(heap, (node.next.val, i, node.next))
        
        return dummy.next
    
    def mergeKLists_divideConquer(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        """
        方法二：分治法
        時間複雜度: O(N log k)
        空間複雜度: O(log k) - 遞迴堆疊
        """
        if not lists:
            return None
        
        def mergeTwoLists(l1, l2):
            """合併兩個有序鏈表"""
            dummy = ListNode(0)
            current = dummy
            
            while l1 and l2:
                if l1.val < l2.val:
                    current.next = l1
                    l1 = l1.next
                else:
                    current.next = l2
                    l2 = l2.next
                current = current.next
            
            current.next = l1 if l1 else l2
            return dummy.next
        
        def merge(lists, left, right):
            """分治合併"""
            if left == right:
                return lists[left]
            
            mid = (left + right) // 2
            l1 = merge(lists, left, mid)
            l2 = merge(lists, mid + 1, right)
            
            return mergeTwoLists(l1, l2)
        
        return merge(lists, 0, len(lists) - 1)
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */

class Solution {
public:
    /**
     * 方法一：最小堆積
     * 時間複雜度: O(N log k)
     * 空間複雜度: O(k)
     */
    ListNode* mergeKLists(vector& lists) {
        // 自定義比較器：最小堆積
        auto cmp = [](ListNode* a, ListNode* b) {
            return a->val > b->val;
        };
        
        priority_queue, decltype(cmp)> pq(cmp);
        
        // 將每個鏈表的頭節點加入堆積
        for (ListNode* node : lists) {
            if (node != nullptr) {
                pq.push(node);
            }
        }
        
        // 虛擬頭節點
        ListNode* dummy = new ListNode(0);
        ListNode* current = dummy;
        
        // 從堆積中逐個取出最小節點
        while (!pq.empty()) {
            ListNode* node = pq.top();
            pq.pop();
            
            current->next = node;
            current = current->next;
            
            // 如果該節點有下一個節點，加入堆積
            if (node->next != nullptr) {
                pq.push(node->next);
            }
        }
        
        return dummy->next;
    }
};
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */

class Solution {
    /**
     * 方法一：最小堆積
     * 時間複雜度: O(N log k)
     * 空間複雜度: O(k)
     */
    public ListNode mergeKLists(ListNode[] lists) {
        // 最小堆積：根據節點值排序
        PriorityQueue pq = new PriorityQueue<>((a, b) -> a.val - b.val);
        
        // 將每個鏈表的頭節點加入堆積
        for (ListNode node : lists) {
            if (node != null) {
                pq.offer(node);
            }
        }
        
        // 虛擬頭節點
        ListNode dummy = new ListNode(0);
        ListNode current = dummy;
        
        // 從堆積中逐個取出最小節點
        while (!pq.isEmpty()) {
            ListNode node = pq.poll();
            current.next = node;
            current = current.next;
            
            // 如果該節點有下一個節點，加入堆積
            if (node.next != null) {
                pq.offer(node.next);
            }
        }
        
        return dummy.next;
    }
}

複雜度分析

時間複雜度: O(N log k)

N 是所有節點的總數，k 是鏈表數量。每個節點都會被加入和取出堆積一次，每次操作 O(log k)。

空間複雜度: O(k)

堆積中最多同時存儲 k 個節點（每個鏈表的當前節點）。

重點提示

  • 為什麼用堆積：在 k 個數中找最小值，堆積是最優選擇
  • 堆積維護：
    • 初始：將 k 個鏈表的頭節點全部加入堆積
    • 每次取出最小節點後，將其 next 加入堆積
    • 堆積大小始終 ≤ k
  • Python 注意事項：使用索引避免節點值相同時的比較問題
  • C++ 注意事項：使用 lambda 自定義比較器創建最小堆積
  • Java 注意事項：PriorityQueue 默認是最小堆積
  • 邊界情況：
    • lists 為空 → 返回 null
    • lists 中有空鏈表 → 過濾掉
    • 只有一個鏈表 → 直接返回
    • 所有鏈表都為空 → 返回 null
  • 對比暴力法：
    • 暴力：每次線性掃描 k 個鏈表找最小值 → O(Nk)
    • 堆積：每次 O(log k) 找最小值 → O(N log k) ⭐
  • 兩種方法對比：
    • 堆積法：容易理解，代碼簡潔，需要 O(k) 額外空間
    • 分治法：空間優化，但代碼較複雜
  • 面試技巧：
    • 先說暴力法 O(Nk)，展現思考過程
    • 優化到堆積法 O(N log k)
    • 可選：提到分治法作為替代方案
    • 說明為什麼堆積是合適的數據結構
