283. Move Zeroes
Easy

題目描述

給定一個整數陣列 nums，將所有 0 移動到陣列的末尾，同時保持非零元素的相對順序。

注意：必須在原陣列上操作，不能複製陣列。

範例 1:
輸入: nums = [0,1,0,3,12]
輸出: [1,3,12,0,0]
範例 2:
輸入: nums = [0]
輸出: [0]
限制條件:
1 <= nums.length <= 10^4
-2^31 <= nums[i] <= 2^31 - 1
進階:

你能盡量減少操作次數嗎？

解題思路

方法：雙指標（快慢指標）⭐

使用慢指標記錄非零元素應該放置的位置，快指標遍歷陣列尋找非零元素。

核心思想：

慢指標指向下一個非零元素應該放置的位置，快指標遍歷陣列找非零元素並移動到慢指標位置。

步驟：

  1. 初始化慢指標 slow = 0（記錄非零元素位置）
  2. 快指標 fast 遍歷陣列：
    • 如果 nums[fast] != 0：
      • 將 nums[fast] 移到 nums[slow]
      • slow++
  3. 遍歷完成後，slow 到末尾的位置填充 0

視覺化範例：nums = [0,1,0,3,12]

初始:    0  1  0  3  12
        ↑  ↑
      slow fast

fast=0: nums[0]=0，跳過
       0  1  0  3  12
       ↑     ↑
     slow  fast

fast=1: nums[1]=1≠0，移動到 slow
       1  1  0  3  12
          ↑     ↑
        slow  fast

fast=2: nums[2]=0，跳過
       1  1  0  3  12
          ↑        ↑
        slow     fast

fast=3: nums[3]=3≠0，移動到 slow
       1  3  0  3  12
             ↑     ↑
           slow  fast

fast=4: nums[4]=12≠0，移動到 slow
       1  3  12  3  12
                ↑     ↑
              slow  fast

填充 0: slow=3，從 3 到末尾填 0
       1  3  12  0  0

最終: [1,3,12,0,0]
優化：減少賦值次數

可以使用交換代替賦值，避免最後填充 0 的步驟：

當 nums[fast] != 0 時:
    交換 nums[slow] 和 nums[fast]
    slow++

這樣可以在一次遍歷中完成，減少操作次數。

程式碼實現

class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        方法一：快慢指標 + 賦值
        時間複雜度: O(n)
        空間複雜度: O(1)
        
        Do not return anything, modify nums in-place instead.
        """
        slow = 0  # 慢指標：記錄非零元素位置
        
        # 將所有非零元素移到前面
        for fast in range(len(nums)):
            if nums[fast] != 0:
                nums[slow] = nums[fast]
                slow += 1
        
        # 將 slow 到末尾的位置填充 0
        for i in range(slow, len(nums)):
            nums[i] = 0
    
    def moveZeroes_swap(self, nums: List[int]) -> None:
        """
        方法二：快慢指標 + 交換（推薦，減少操作）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        slow = 0
        
        for fast in range(len(nums)):
            if nums[fast] != 0:
                # 交換非零元素到前面
                nums[slow], nums[fast] = nums[fast], nums[slow]
                slow += 1
#include <vector>
#include <algorithm>

class Solution {
public:
    /**
     * 方法一：快慢指標 + 賦值
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    void moveZeroes(std::vector& nums) {
        int slow = 0;
        
        // 將所有非零元素移到前面
        for (int fast = 0; fast < nums.size(); fast++) {
            if (nums[fast] != 0) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        
        // 將 slow 到末尾的位置填充 0
        for (int i = slow; i < nums.size(); i++) {
            nums[i] = 0;
        }
    }
    
    /**
     * 方法二：快慢指標 + 交換（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    void moveZeroes_swap(std::vector& nums) {
        int slow = 0;
        
        for (int fast = 0; fast < nums.size(); fast++) {
            if (nums[fast] != 0) {
                std::swap(nums[slow], nums[fast]);
                slow++;
            }
        }
    }
};
class Solution {
    /**
     * 方法一：快慢指標 + 賦值
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public void moveZeroes(int[] nums) {
        int slow = 0;
        
        // 將所有非零元素移到前面
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        
        // 將 slow 到末尾的位置填充 0
        for (int i = slow; i < nums.length; i++) {
            nums[i] = 0;
        }
    }
    
    /**
     * 方法二：快慢指標 + 交換（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public void moveZeroes_swap(int[] nums) {
        int slow = 0;
        
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                // 交換
                int temp = nums[slow];
                nums[slow] = nums[fast];
                nums[fast] = temp;
                slow++;
            }
        }
    }
}

複雜度分析

時間複雜度: O(n)

只需遍歷陣列一次（或兩次），其中 n 是陣列長度。

空間複雜度: O(1)

只使用兩個指標變數，原地修改陣列。

重點提示

  • 快慢指標經典應用：slow 記錄位置，fast 遍歷陣列
  • 保持相對順序：非零元素的順序不能改變
  • 原地操作：不能創建新陣列
  • 兩種方法對比：
    • 賦值法：需要兩次遍歷，邏輯清晰
    • 交換法：一次遍歷完成，操作更少 ⭐
  • 邊界情況：
    • 全是 0 → 不需要移動
    • 沒有 0 → slow 會到達末尾
    • 只有一個元素 → 不需要操作
  • 面試技巧：
    • 先說賦值法（容易理解）
    • 再優化到交換法（減少操作）
    • 強調保持相對順序的重要性
