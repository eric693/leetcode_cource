121. Best Time to Buy and Sell Stock
Easy

題目描述

給定一個陣列 prices，其中 prices[i] 是一支股票第 i 天的價格。

你想要通過選擇某一天購買這支股票，並選擇在未來的某一天出售該股票來最大化你的利潤。

返回你可以從這筆交易中獲取的最大利潤。如果你不能獲取任何利潤，返回 0。

範例 1:
輸入: prices = [7,1,5,3,6,4]
輸出: 5
解釋: 在第 2 天（股票價格 = 1）買入，在第 5 天（股票價格 = 6）賣出
最大利潤 = 6 - 1 = 5
注意不能在第 2 天買入然後在第 1 天賣出（因為買入必須在賣出之前）
範例 2:
輸入: prices = [7,6,4,3,1]
輸出: 0
解釋: 在這種情況下，沒有交易完成，最大利潤為 0
限制條件:
1 <= prices.length <= 10^5
0 <= prices[i] <= 10^4

解題思路

方法一：一次遍歷（推薦）

維護歷史最低價格，同時計算如果今天賣出的最大利潤。

核心思想：

遍歷價格陣列，對於每一天：

  • 如果今天的價格更低，更新最低價格
  • 否則，計算今天賣出的利潤，更新最大利潤

步驟：

  1. 初始化 min_price = ∞，max_profit = 0
  2. 遍歷每天的價格：
    • 如果 price < min_price，更新 min_price
    • 否則，計算 profit = price - min_price
    • 更新 max_profit = max(max_profit, profit)
  3. 返回 max_profit

視覺化範例：prices = [7,1,5,3,6,4]

Day 0: price=7, min_price=7, max_profit=0
Day 1: price=1, min_price=1, max_profit=0 (更新最低價)
Day 2: price=5, min_price=1, max_profit=4 (5-1=4)
Day 3: price=3, min_price=1, max_profit=4 (3-1=2 < 4)
Day 4: price=6, min_price=1, max_profit=5 (6-1=5) ⭐
Day 5: price=4, min_price=1, max_profit=5 (4-1=3 < 5)

答案: 5 (在第1天買入,第4天賣出)
方法二：動態規劃思路

將問題視為求最大子陣列和的變形（與 Maximum Subarray 類似）。

轉換思路：

將價格陣列轉換為相鄰日期的價格差：[0, -6, 4, -2, 3, -2]

問題變成：找到價格差陣列的最大子陣列和

程式碼實現

class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        """
        方法一：一次遍歷（推薦）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        if not prices:
            return 0
        
        min_price = float('inf')  # 歷史最低價格
        max_profit = 0            # 最大利潤
        
        for price in prices:
            if price < min_price:
                # 更新最低價格
                min_price = price
            else:
                # 計算今天賣出的利潤
                profit = price - min_price
                max_profit = max(max_profit, profit)
        
        return max_profit
    
    def maxProfit_clean(self, prices: List[int]) -> int:
        """
        方法一：簡潔版本
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        min_price = float('inf')
        max_profit = 0
        
        for price in prices:
            min_price = min(min_price, price)
            max_profit = max(max_profit, price - min_price)
        
        return max_profit
    
    def maxProfit_dp(self, prices: List[int]) -> int:
        """
        方法二：動態規劃（轉換為最大子陣列和）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        max_profit = 0
        current_profit = 0
        
        for i in range(1, len(prices)):
            # 計算價格變化
            diff = prices[i] - prices[i-1]
            # 類似 Kadane's Algorithm
            current_profit = max(0, current_profit + diff)
            max_profit = max(max_profit, current_profit)
        
        return max_profit

複雜度分析

時間複雜度: O(n)

只需要遍歷一次價格陣列，其中 n 是陣列長度。

空間複雜度: O(1)

只使用常數個額外變數（min_price 和 max_profit）。

重點提示

  • 貪心思想：始終在歷史最低點買入，在當前價格賣出
  • 只需一次交易：這題只允許一次買入和一次賣出
  • 時間順序：必須先買入再賣出，不能在同一天完成
  • 邊界情況：
    • 價格持續下跌 → 返回 0
    • 只有一天 → 返回 0
    • 所有價格相同 → 返回 0
  • 不要想複雜：不需要記錄買入賣出的具體日期，只需要最大利潤
  • 面試技巧：
    • 先確認只能進行一次交易
    • 說明暴力解法 O(n²) 後優化到 O(n)
    • 解釋為什麼要維護歷史最低價
  • 擴展問題：
    • Best Time to Buy and Sell Stock II (#122) - 可以多次交易
    • Best Time to Buy and Sell Stock III (#123) - 最多兩次交易
    • Best Time to Buy and Sell Stock IV (#188) - 最多 k 次交易
