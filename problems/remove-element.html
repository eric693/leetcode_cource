27. Remove Element
Easy

題目描述

給你一個陣列 nums 和一個值 val，你需要原地移除所有數值等於 val 的元素，並返回移除後陣列的新長度。

不要使用額外的陣列空間，你必須僅使用 O(1) 額外空間並原地修改輸入陣列。

元素的順序可以改變。你不需要考慮陣列中超出新長度後面的元素。

說明:

為什麼返回數值是整數，但輸出的答案是陣列呢？

請注意，輸入陣列是以「引用」方式傳遞的，這意味著在函數裡修改輸入陣列對於調用者是可見的。

範例 1:
輸入: nums = [3,2,2,3], val = 3
輸出: 2, nums = [2,2,_,_]
解釋: 函數應該返回新的長度 2, 並且 nums 中的前兩個元素均為 2。
你不需要考慮陣列中超出新長度後面的元素。
例如，函數返回的新長度為 2，而 nums = [2,2,3,3] 或 nums = [2,2,0,0]，也會被視為正確答案。
範例 2:
輸入: nums = [0,1,2,2,3,0,4,2], val = 2
輸出: 5, nums = [0,1,4,0,3,_,_,_]
解釋: 函數應該返回新的長度 5，並且 nums 中的前五個元素為 0,1,3,0,4。
注意這五個元素可為任意順序。你不需要考慮陣列中超出新長度後面的元素。
限制條件:
0 <= nums.length <= 100
0 <= nums[i] <= 50
0 <= val <= 100

解題思路

核心觀察

這道題的關鍵特性：

  • 需要原地操作 - O(1) 空間
  • 元素順序可以改變 - 這很重要！
  • 不需要考慮新長度後的元素
  • 只需保留不等於 val 的元素

關鍵：與 #26 類似，使用雙指標技巧

方法一：雙指標（快慢指標）⭐⭐⭐

使用兩個指標來實現原地移除。

核心思想：

  • 慢指標（slow）：指向下一個要寫入的位置
  • 快指標（fast）：遍歷整個陣列
  • 當 nums[fast] != val 時，保留這個元素
  • 將 nums[fast] 寫入 nums[slow]，slow++

視覺化範例：nums = [3,2,2,3], val = 3

初始狀態:
nums = [3, 2, 2, 3], val = 3
        ↑
      slow=0

fast=0: nums[0]=3 == val，跳過（不保留）
nums = [3, 2, 2, 3]
        ↑  ↑
      slow fast

fast=1: nums[1]=2 != val，保留！
        nums[slow] = nums[fast] = 2
        slow++
nums = [2, 2, 2, 3]
           ↑  ↑
         slow fast

fast=2: nums[2]=2 != val，保留！
        nums[slow] = nums[fast] = 2
        slow++
nums = [2, 2, 2, 3]
              ↑  ↑
            slow fast

fast=3: nums[3]=3 == val，跳過（不保留）
nums = [2, 2, 2, 3]
              ↑     ↑
            slow   fast

最終結果:
nums = [2, 2, _, _]
返回 slow = 2

更複雜的範例：nums = [0,1,2,2,3,0,4,2], val = 2

初始: nums = [0, 1, 2, 2, 3, 0, 4, 2], val = 2

fast=0: nums[0]=0 != 2 → 保留 → nums[0]=0, slow=1
fast=1: nums[1]=1 != 2 → 保留 → nums[1]=1, slow=2
fast=2: nums[2]=2 == 2 → 跳過
fast=3: nums[3]=2 == 2 → 跳過
fast=4: nums[4]=3 != 2 → 保留 → nums[2]=3, slow=3
fast=5: nums[5]=0 != 2 → 保留 → nums[3]=0, slow=4
fast=6: nums[6]=4 != 2 → 保留 → nums[4]=4, slow=5
fast=7: nums[7]=2 == 2 → 跳過

結果: nums = [0, 1, 3, 0, 4, _, _, _]
返回 slow = 5
方法二：雙指標優化（交換法）⭐⭐⭐

當要刪除的元素很少時，這個方法更高效。

核心思想：

  • 從左右兩端同時開始
  • 左指標遇到 val，與右指標交換
  • 右指標左移，左指標不動
  • 左指標遇到非 val，左移繼續

優勢：

  • 減少不必要的賦值操作
  • 當 val 元素少時更高效
  • 避免覆蓋有用的元素

視覺化範例：nums = [3,2,2,3], val = 3

初始: nums = [3, 2, 2, 3]
              ↑        ↑
            left     right

left=0: nums[0]=3 == val
        交換 nums[left] 和 nums[right]
        right--
nums = [3, 2, 2, 3]
        ↑     ↑
      left  right

left=0: nums[0]=3 == val
        交換 nums[left] 和 nums[right]
        right--
nums = [2, 2, 2, 3]
        ↑  ↑
      left right

left=0: nums[0]=2 != val
        left++
nums = [2, 2, 2, 3]
           ↑  ↑
         left right

left=1: nums[1]=2 != val
        left++
nums = [2, 2, 2, 3]
              ↑
          left,right

left > right，結束
返回 right + 1 = 2

程式碼實現

class Solution:
    def removeElement(self, nums: List[int], val: int) -> int:
        """
        方法一：雙指標（快慢指標）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        slow = 0
        
        for fast in range(len(nums)):
            # 保留所有不等於 val 的元素
            if nums[fast] != val:
                nums[slow] = nums[fast]
                slow += 1
        
        return slow
    
    def removeElement_swap(self, nums: List[int], val: int) -> int:
        """
        方法二：雙指標優化（交換法）
        時間複雜度: O(n)
        空間複雜度: O(1)
        
        優勢：當要移除的元素較少時，減少賦值次數
        """
        left = 0
        right = len(nums) - 1
        
        while left <= right:
            if nums[left] == val:
                # 與末尾元素交換
                nums[left] = nums[right]
                right -= 1
            else:
                left += 1
        
        return left
#include <vector>
using namespace std;

class Solution {
public:
    /**
     * 方法一：雙指標（快慢指標）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int removeElement(vector& nums, int val) {
        int slow = 0;
        
        for (int fast = 0; fast < nums.size(); fast++) {
            if (nums[fast] != val) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        
        return slow;
    }
    
    /**
     * 方法二：雙指標優化（交換法）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int removeElement_swap(vector& nums, int val) {
        int left = 0;
        int right = nums.size() - 1;
        
        while (left <= right) {
            if (nums[left] == val) {
                nums[left] = nums[right];
                right--;
            } else {
                left++;
            }
        }
        
        return left;
    }
};
class Solution {
    /**
     * 方法一：雙指標（快慢指標）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int removeElement(int[] nums, int val) {
        int slow = 0;
        
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != val) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        
        return slow;
    }
    
    /**
     * 方法二：雙指標優化（交換法）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int removeElement_swap(int[] nums, int val) {
        int left = 0;
        int right = nums.length - 1;
        
        while (left <= right) {
            if (nums[left] == val) {
                nums[left] = nums[right];
                right--;
            } else {
                left++;
            }
        }
        
        return left;
    }
}

複雜度分析

時間複雜度: O(n)

兩種方法都只需遍歷陣列一次（最多）。

空間複雜度: O(1)

只使用常數額外空間，原地修改。

重點提示

  • 兩種方法比較：
    • 方法一（快慢指標）：穩定，適合大多數情況
    • 方法二（交換法）：當要移除的元素很少時更優
    • 方法一：賦值次數 = n - val_count
    • 方法二：賦值次數 = val_count
  • 與 #26 的區別：
    • #26：移除重複元素（已排序）
    • #27：移除指定值（任意順序）
    • 兩題都用雙指標，思路相似
  • 為什麼順序可以改變很重要？
    • 允許我們使用交換法
    • 不需要保持相對順序
    • 可以更靈活地優化
  • 交換法的優勢：
    • 假設陣列 [1,2,3,4,5]，val = 5
    • 快慢指標：需要複製 4 次（1,2,3,4）
    • 交換法：只需交換 1 次（5 和 5）
  • 何時使用哪種方法？
    • 不確定 val 數量 → 用方法一
    • val 元素很少 → 用方法二
    • 需要穩定順序 → 只能用方法一
  • 邊界情況：
    • 空陣列 → 返回 0
    • 所有元素都是 val → 返回 0
    • 沒有元素是 val → 返回原長度
    • 只有一個元素
  • 常見錯誤：
    • 忘記初始化 slow = 0
    • 交換法中 left <= right 寫成 left < right
    • 沒有處理空陣列（Python 不需要）
  • 面試技巧：
    • 先問是否需要保持順序
    • 提出兩種解法
    • 分析各自優劣
    • 畫圖展示過程
  • 與 Move Zeroes (#283) 的關係：
    • #283 是本題的變形
    • 移除 0 後，需要填充 0
    • 核心思想完全一樣
  • 延伸思考：
    • 如果要移除多個不同的值？
    • 如果需要統計移除了多少個？
    • 如果需要返回被移除的元素？
  • 實際應用場景：
    • 數據過濾
    • 清除無效值
    • 陣列壓縮
  • 變形題：
    • Remove Duplicates from Sorted Array (#26)
    • Move Zeroes (#283)
    • Remove All Occurrences of a Substring (#1910)
  • 記憶口訣：「快讀慢寫留不同，交換優化少移動」
