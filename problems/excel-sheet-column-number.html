171. Excel Sheet Column Number
Easy

題目描述

給你一個字串 columnTitle，表示 Excel 表格中的列名稱。返回該列名稱對應的列序號。

對照表:
A -> 1
B -> 2
C -> 3
...
Z -> 26
AA -> 27
AB -> 28
...
ZY -> 701
ZZ -> 702
AAA -> 703
...
範例 1:
輸入: columnTitle = "A"
輸出: 1
範例 2:
輸入: columnTitle = "AB"
輸出: 28
範例 3:
輸入: columnTitle = "ZY"
輸出: 701
限制條件:
1 <= columnTitle.length <= 7
columnTitle 僅由大寫英文字母組成
columnTitle 在範圍 ["A", "FXSHRXW"] 內

解題思路

核心觀察

這是一個26 進制轉換問題（類似）：

  • A-Z 代表 1-26（注意不是 0-25）
  • 類似十進制：123 = 1×100 + 2×10 + 3×1
  • Excel 列：AB = A×26 + B = 1×26 + 2 = 28
  • 從左到右，每個字母權重是 26 的次方

關鍵：這不是標準 26 進制（因為沒有 0，從 1 開始）

方法：進制轉換 ⭐⭐⭐

將 Excel 列名轉換為數字。

核心思想：

  • 從左到右遍歷字串
  • 每次結果 × 26
  • 加上當前字母的值（A=1, B=2, ..., Z=26）
  • 累積得到最終結果

公式：

result = result × 26 + (當前字母 - 'A' + 1)

視覺化範例 1：columnTitle = "AB"

AB = A×26¹ + B×26⁰ = 1×26 + 2×1 = 28

逐步計算:
初始: result = 0

處理 'A':
  A 的值 = 'A' - 'A' + 1 = 1
  result = 0 × 26 + 1 = 1

處理 'B':
  B 的值 = 'B' - 'A' + 1 = 2
  result = 1 × 26 + 2 = 28 ✓

視覺化範例 2：columnTitle = "ZY"

ZY = Z×26¹ + Y×26⁰ = 26×26 + 25×1 = 701

逐步計算:
初始: result = 0

處理 'Z':
  Z 的值 = 'Z' - 'A' + 1 = 26
  result = 0 × 26 + 26 = 26

處理 'Y':
  Y 的值 = 'Y' - 'A' + 1 = 25
  result = 26 × 26 + 25 = 676 + 25 = 701 ✓

與十進制類比：

十進制數 "123":
1×10² + 2×10¹ + 3×10⁰ = 100 + 20 + 3 = 123

逐步計算（從左到右）:
result = 0
result = 0 × 10 + 1 = 1
result = 1 × 10 + 2 = 12
result = 12 × 10 + 3 = 123

Excel 列名 "ABC":
result = 0
result = 0 × 26 + 1 = 1      (A)
result = 1 × 26 + 2 = 28     (B)
result = 28 × 26 + 3 = 731   (C)

ABC = 731

為什麼是 +1？

  • 因為 A 代表 1，不是 0
  • 'A' - 'A' = 0，需要 +1 變成 1
  • 'Z' - 'A' = 25，需要 +1 變成 26

程式碼實現

class Solution:
    def titleToNumber(self, columnTitle: str) -> int:
        """
        進制轉換
        時間複雜度: O(n)，n 是字串長度
        空間複雜度: O(1)
        """
        result = 0
        
        for char in columnTitle:
            # 當前字母的值 (A=1, B=2, ..., Z=26)
            value = ord(char) - ord('A') + 1
            # 累積計算
            result = result * 26 + value
        
        return result
    
    def titleToNumber_v2(self, columnTitle: str) -> int:
        """
        另一種寫法：從右到左計算
        """
        result = 0
        power = 0
        
        # 從右到左遍歷
        for i in range(len(columnTitle) - 1, -1, -1):
            value = ord(columnTitle[i]) - ord('A') + 1
            result += value * (26 ** power)
            power += 1
        
        return result
    
    def titleToNumber_oneliner(self, columnTitle: str) -> int:
        """
        Python 一行解法
        """
        return sum((ord(c) - ord('A') + 1) * (26 ** i) 
                   for i, c in enumerate(reversed(columnTitle)))
#include <string>
using namespace std;

class Solution {
public:
    /**
     * 進制轉換
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int titleToNumber(string columnTitle) {
        int result = 0;
        
        for (char c : columnTitle) {
            int value = c - 'A' + 1;
            result = result * 26 + value;
        }
        
        return result;
    }
};
class Solution {
    /**
     * 進制轉換
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int titleToNumber(String columnTitle) {
        int result = 0;
        
        for (char c : columnTitle.toCharArray()) {
            int value = c - 'A' + 1;
            result = result * 26 + value;
        }
        
        return result;
    }
}

複雜度分析

時間複雜度: O(n)

需要遍歷字串一次，n 是字串長度。

空間複雜度: O(1)

只使用常數額外空間。

重點提示

  • 進制轉換核心：
    • 從左到右：result = result × 26 + value
    • 從右到左：result += value × 26^power
    • 兩種方法等價，左到右更簡潔
  • 字母到數字的映射：
    • A → 1：'A' - 'A' + 1 = 0 + 1 = 1
    • B → 2：'B' - 'A' + 1 = 1 + 1 = 2
    • Z → 26：'Z' - 'A' + 1 = 25 + 1 = 26
  • 為什麼不是標準 26 進制？
    • 標準 26 進制：0-25（有 0）
    • Excel 列名：1-26（沒有 0）
    • 這導致沒有 "0" 對應的字母
  • 常見錯誤：
    • 忘記 +1（A 應該是 1 不是 0）
    • 溢位問題（最長 7 位，不會溢位 int）
    • 字母大小寫（題目保證大寫）
  • 驗證範例：
    • A = 1 ✓
    • Z = 26 ✓
    • AA = 1×26 + 1 = 27 ✓
    • AZ = 1×26 + 26 = 52 ✓
    • BA = 2×26 + 1 = 53 ✓
  • 與標準進制的區別：
    • 十進制：0-9（10 個符號）
    • 二進制：0-1（2 個符號）
    • Excel：1-26（26 個符號，無 0）
  • 邊界情況：
    • 單字母：A-Z（1-26）
    • 雙字母：AA-ZZ（27-702）
    • 最長 7 位不會溢位 32 位整數

    • ord(c) 獲取字符 ASCII 值
    • chr(n) 將數字轉回字符
    • 可以用 reduce 寫一行
