64. Minimum Path Sum
Medium

題目描述

給定一個包含非負整數的 m x n 網格 grid，請找出一條從左上角到右下角的路徑，使得路徑上的數字總和為最小。

說明：每次只能向下或者向右移動一步。

範例 1:
輸入: grid = [[1,3,1],[1,5,1],[4,2,1]]
輸出: 7
解釋: 因為路徑 1→3→1→1→1 的總和最小。
網格:        路徑:
1  3  1      1→ 3→ 1
1  5  1      ↓     ↓
4  2  1             1→ 1

路徑和: 1+3+1+1+1 = 7
範例 2:
輸入: grid = [[1,2,3],[4,5,6]]
輸出: 12
限制條件:
m == grid.length
n == grid[i].length
1 <= m, n <= 200
0 <= grid[i][j] <= 200

解題思路

核心觀察

這是一個經典的最短路徑動態規劃問題：

  • 只能向右或向下移動
  • 到達某個格子的最小路徑和 = min(從上方來, 從左方來) + 當前格子值
  • 第一行只能從左邊來，第一列只能從上面來

關鍵：dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]

方法一：二維動態規劃（推薦理解）⭐

使用二維陣列記錄到達每個格子的最小路徑和。

狀態定義：

  • dp[i][j]：從左上角到達位置 (i, j) 的最小路徑和
  • 初始狀態：
    • dp[0][0] = grid[0][0]（起點）
    • dp[i][0] = dp[i-1][0] + grid[i][0]（第一列，累加）
    • dp[0][j] = dp[0][j-1] + grid[0][j]（第一行，累加）
  • 轉移方程：
    • dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]
    • （選擇從上方或左方來的較小值）

視覺化範例：grid = [[1,3,1],[1,5,1],[4,2,1]]

原始網格:
1  3  1
1  5  1
4  2  1

初始化第一行和第一列:
dp[0][0] = 1
dp[0][1] = 1 + 3 = 4
dp[0][2] = 4 + 1 = 5
dp[1][0] = 1 + 1 = 2
dp[2][0] = 2 + 4 = 6

DP 表格初始狀態:
1  4  5
2  ?  ?
6  ?  ?

計算 dp[1][1]:
  從上: dp[0][1] + grid[1][1] = 4 + 5 = 9
  從左: dp[1][0] + grid[1][1] = 2 + 5 = 7
  dp[1][1] = min(9, 7) = 7

DP 表格:
1  4  5
2  7  ?
6  ?  ?

計算 dp[1][2]:
  從上: dp[0][2] + grid[1][2] = 5 + 1 = 6
  從左: dp[1][1] + grid[1][2] = 7 + 1 = 8
  dp[1][2] = min(6, 8) = 6

DP 表格:
1  4  5
2  7  6
6  ?  ?

計算 dp[2][1]:
  從上: dp[1][1] + grid[2][1] = 7 + 2 = 9
  從左: dp[2][0] + grid[2][1] = 6 + 2 = 8
  dp[2][1] = min(9, 8) = 8

DP 表格:
1  4  5
2  7  6
6  8  ?

計算 dp[2][2]:
  從上: dp[1][2] + grid[2][2] = 6 + 1 = 7
  從左: dp[2][1] + grid[2][2] = 8 + 1 = 9
  dp[2][2] = min(7, 9) = 7

最終 DP 表格:
1  4  5
2  7  6
6  8  7

結果: 7
方法二：原地修改（空間優化）⭐⭐

直接在原始 grid 上修改，不需要額外空間。

  • 修改第一行和第一列為累加值
  • 其他格子：grid[i][j] += min(grid[i-1][j], grid[i][j-1])
  • 空間複雜度：O(1)

注意：會修改原始陣列，如果不允許修改需要複製

方法三：一維動態規劃⭐⭐⭐

使用一維陣列滾動更新，節省空間。

一維 DP 示意：

初始: dp = [1, 4, 5] (第一行)

處理第 1 行 [1,5,1]:
  dp[0] = dp[0] + grid[1][0] = 1 + 1 = 2
  dp[1] = min(dp[0], dp[1]) + grid[1][1] = min(2,4) + 5 = 7
  dp[2] = min(dp[1], dp[2]) + grid[1][2] = min(7,5) + 1 = 6
  dp = [2, 7, 6]

處理第 2 行 [4,2,1]:
  dp[0] = dp[0] + grid[2][0] = 2 + 4 = 6
  dp[1] = min(dp[0], dp[1]) + grid[2][1] = min(6,7) + 2 = 8
  dp[2] = min(dp[1], dp[2]) + grid[2][2] = min(8,6) + 1 = 7
  dp = [6, 8, 7]

結果: dp[2] = 7

程式碼實現

class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        """
        方法一：二維動態規劃
        時間複雜度: O(m * n)
        空間複雜度: O(m * n)
        """
        m, n = len(grid), len(grid[0])
        dp = [[0] * n for _ in range(m)]
        
        # 初始化起點
        dp[0][0] = grid[0][0]
        
        # 初始化第一行
        for j in range(1, n):
            dp[0][j] = dp[0][j-1] + grid[0][j]
        
        # 初始化第一列
        for i in range(1, m):
            dp[i][0] = dp[i-1][0] + grid[i][0]
        
        # 填充 DP 表格
        for i in range(1, m):
            for j in range(1, n):
                dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]
        
        return dp[m-1][n-1]
    
    def minPathSum_inplace(self, grid: List[List[int]]) -> int:
        """
        方法二：原地修改（推薦，如果允許修改）
        時間複雜度: O(m * n)
        空間複雜度: O(1)
        """
        m, n = len(grid), len(grid[0])
        
        # 初始化第一行
        for j in range(1, n):
            grid[0][j] += grid[0][j-1]
        
        # 初始化第一列
        for i in range(1, m):
            grid[i][0] += grid[i-1][0]
        
        # 填充網格
        for i in range(1, m):
            for j in range(1, n):
                grid[i][j] += min(grid[i-1][j], grid[i][j-1])
        
        return grid[m-1][n-1]
    
    def minPathSum_1d(self, grid: List[List[int]]) -> int:
        """
        方法三：一維動態規劃（最優空間）
        時間複雜度: O(m * n)
        空間複雜度: O(n)
        """
        m, n = len(grid), len(grid[0])
        dp = [0] * n
        
        # 初始化第一行
        dp[0] = grid[0][0]
        for j in range(1, n):
            dp[j] = dp[j-1] + grid[0][j]
        
        # 逐行更新
        for i in range(1, m):
            dp[0] += grid[i][0]  # 第一列
            for j in range(1, n):
                dp[j] = min(dp[j], dp[j-1]) + grid[i][j]
        
        return dp[n-1]

複雜度分析

時間複雜度: O(m × n)

需要遍歷整個網格一次，m 是行數，n 是列數。

空間複雜度: O(1) ~ O(m × n)

原地修改：O(1)，一維 DP：O(n)，二維 DP：O(m × n)。

重點提示

  • DP 思路：每個格子的最小路徑和 = min(上方, 左方) + 當前值
  • 初始化技巧：
    • 第一行：只能從左邊來，累加
    • 第一列：只能從上面來，累加
    • 起點：就是 grid[0][0]
  • 三種實現比較：
    • 二維 DP：最直觀，易理解
    • 原地修改：空間最優 O(1)，但會改變原陣列
    • 一維 DP：平衡，不改原陣列且省空間
  • 與 Unique Paths 的區別：
    • Unique Paths：計算路徑「數量」（相加）
    • Minimum Path Sum：找「最小和」（取 min）
    • 狀態轉移方程不同但思路相似
  • 邊界情況：
    • 只有一個格子 (1×1) → 返回該格子值
    • 只有一行或一列 → 直接累加
    • 所有格子值相同
    • 包含 0 的格子
  • 常見錯誤：
    • 忘記初始化第一行/第一列
    • min 函數參數順序錯誤
    • 索引越界 (i-1, j-1)
    • 原地修改但需要保留原陣列
  • 面試技巧：
    • 先畫小網格示意 (3×3)
    • 展示 DP 表格填充過程
    • 說明三種實現的優劣
    • 討論是否允許修改原陣列
  • 延伸思考：
    • 如何追蹤實際路徑？（記錄選擇方向）
    • 如果可以向四個方向移動？（Dijkstra）
    • 如果有負數？（仍然可以用 DP）
    • 如果要找最大路徑和？（改 min 為 max）
