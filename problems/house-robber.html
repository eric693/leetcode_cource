198. House Robber
Medium

題目描述

你是一個專業的小偷，計劃偷竊沿街的房屋。每間房內都藏有一定的現金，影響你偷竊的唯一制約因素就是相鄰的房屋裝有相互連通的防盜系統，如果兩間相鄰的房屋在同一晚上被小偷闖入，系統會自動報警。

給定一個代表每個房屋存放金額的非負整數陣列，計算你不觸動警報裝置的情況下，一夜之內能夠偷竊到的最高金額。

範例 1:
輸入: nums = [1,2,3,1]
輸出: 4
解釋: 偷竊 1 號房屋 (金額 = 1) ，然後偷竊 3 號房屋 (金額 = 3)。
     偷竊到的最高金額 = 1 + 3 = 4。
範例 2:
輸入: nums = [2,7,9,3,1]
輸出: 12
解釋: 偷竊 1 號房屋 (金額 = 2), 偷竊 3 號房屋 (金額 = 9)，接著偷竊 5 號房屋 (金額 = 1)。
     偷竊到的最高金額 = 2 + 9 + 1 = 12。
限制條件:
1 <= nums.length <= 100
0 <= nums[i] <= 400

解題思路

核心觀察

這是一個經典的動態規劃問題：

  • 每個房屋面臨兩個選擇：偷或不偷
  • 如果偷第 i 間房，就不能偷第 i-1 間
  • 如果不偷第 i 間房，最大金額就是前 i-1 間的最大值

關鍵：對於每間房，比較「偷」和「不偷」哪個收益更大

方法一：動態規劃（推薦）⭐⭐

使用 DP 記錄到每個位置時的最大收益。

狀態定義：

  • dp[i]：偷竊前 i 間房屋能獲得的最大金額
  • 初始狀態：
    • dp[0] = 0（沒有房屋）
    • dp[1] = nums[0]（只有一間房，直接偷）
  • 轉移方程：
    • 偷第 i 間房：dp[i] = dp[i-2] + nums[i-1]
    • 不偷第 i 間房：dp[i] = dp[i-1]
    • 取最大值：dp[i] = max(dp[i-1], dp[i-2] + nums[i-1])

視覺化範例：nums = [2,7,9,3,1]

房屋:    2    7    9    3    1
索引:    0    1    2    3    4

初始化:
dp[0] = 0  (沒有房屋)
dp[1] = 2  (偷第 0 間)

i=2 (房屋1，金額7):
  偷: dp[0] + 7 = 0 + 7 = 7
  不偷: dp[1] = 2
  dp[2] = max(7, 2) = 7

i=3 (房屋2，金額9):
  偷: dp[1] + 9 = 2 + 9 = 11
  不偷: dp[2] = 7
  dp[3] = max(11, 7) = 11

i=4 (房屋3，金額3):
  偷: dp[2] + 3 = 7 + 3 = 10
  不偷: dp[3] = 11
  dp[4] = max(10, 11) = 11

i=5 (房屋4，金額1):
  偷: dp[3] + 1 = 11 + 1 = 12
  不偷: dp[4] = 11
  dp[5] = max(12, 11) = 12

結果: 12
方案: 偷房屋 0(2) + 2(9) + 4(1) = 12
方法二：空間優化（O(1) 空間）⭐⭐⭐

觀察到 dp[i] 只依賴 dp[i-1] 和 dp[i-2]，可以用兩個變數優化。

滾動變數示意：nums = [2,7,9,3,1]

初始: prev2=0, prev1=2

處理 7:
  current = max(prev1, prev2 + 7)
          = max(2, 0 + 7) = 7
  更新: prev2=2, prev1=7

處理 9:
  current = max(7, 2 + 9) = 11
  更新: prev2=7, prev1=11

處理 3:
  current = max(11, 7 + 3) = 11
  更新: prev2=11, prev1=11

處理 1:
  current = max(11, 11 + 1) = 12
  更新: prev2=11, prev1=12

結果: 12
方法三：記憶化遞迴（易理解但效率稍低）

使用遞迴 + 記憶化的方式思考問題。

  • 定義 rob(i) = 從第 i 間房開始到最後的最大收益
  • rob(i) = max(nums[i] + rob(i+2), rob(i+1))
  • 使用 memo 避免重複計算

程式碼實現

class Solution:
    def rob(self, nums: List[int]) -> int:
        """
        方法一：動態規劃
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        if not nums:
            return 0
        if len(nums) == 1:
            return nums[0]
        
        n = len(nums)
        dp = [0] * (n + 1)
        dp[1] = nums[0]
        
        for i in range(2, n + 1):
            # 偷第 i 間 vs 不偷第 i 間
            dp[i] = max(dp[i-1], dp[i-2] + nums[i-1])
        
        return dp[n]
    
    def rob_optimized(self, nums: List[int]) -> int:
        """
        方法二：空間優化（推薦）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        if not nums:
            return 0
        if len(nums) == 1:
            return nums[0]
        
        # prev2 = dp[i-2], prev1 = dp[i-1]
        prev2, prev1 = 0, nums[0]
        
        for i in range(1, len(nums)):
            current = max(prev1, prev2 + nums[i])
            prev2, prev1 = prev1, current
        
        return prev1
    
    def rob_clean(self, nums: List[int]) -> int:
        """
        最簡潔版本
        """
        prev, curr = 0, 0
        
        for num in nums:
            prev, curr = curr, max(curr, prev + num)
        
        return curr
    
    def rob_recursive(self, nums: List[int]) -> int:
        """
        方法三：記憶化遞迴
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        memo = {}
        
        def dp(i):
            # 越界
            if i >= len(nums):
                return 0
            
            # 已計算過
            if i in memo:
                return memo[i]
            
            # 偷第 i 間 vs 不偷第 i 間
            result = max(nums[i] + dp(i + 2), dp(i + 1))
            memo[i] = result
            return result
        
        return dp(0)
#include <vector>
#include <algorithm>
using namespace std;

class Solution {
public:
    /**
     * 方法一：動態規劃
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    int rob(vector& nums) {
        if (nums.empty()) return 0;
        if (nums.size() == 1) return nums[0];
        
        int n = nums.size();
        vector dp(n + 1, 0);
        dp[1] = nums[0];
        
        for (int i = 2; i <= n; i++) {
            dp[i] = max(dp[i-1], dp[i-2] + nums[i-1]);
        }
        
        return dp[n];
    }
    
    /**
     * 方法二：空間優化（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int rob_optimized(vector& nums) {
        if (nums.empty()) return 0;
        if (nums.size() == 1) return nums[0];
        
        int prev2 = 0, prev1 = nums[0];
        
        for (int i = 1; i < nums.size(); i++) {
            int current = max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = current;
        }
        
        return prev1;
    }
    
    /**
     * 最簡潔版本
     */
    int rob_clean(vector& nums) {
        int prev = 0, curr = 0;
        
        for (int num : nums) {
            int temp = curr;
            curr = max(curr, prev + num);
            prev = temp;
        }
        
        return curr;
    }
};

    /**
     * 方法一：動態規劃
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)

    public int rob(int[] nums) {
        if (nums == null || nums.length == 0) return 0;
        if (nums.length == 1) return nums[0];

        }
        
        return dp[n];
    }
    
    /**
     * 方法二：空間優化（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int rob_optimized(int[] nums) {
        if (nums == null || nums.length == 0) return 0;

複雜度分析

時間複雜度: O(n)

只需遍歷陣列一次，n 是房屋數量。

空間複雜度: O(1)

優化版本只用兩個變數。 基本 DP 版本需要 O(n) 陣列空間。

重點提示

  • DP 思路：每個房屋面臨「偷」或「不偷」的選擇
  • 狀態轉移：
    • 偷第 i 間：收益 = dp[i-2] + nums[i]
    • 不偷第 i 間：收益 = dp[i-1]
    • 取最大值
  • 空間優化：
    • 只需要前兩個狀態
    • 使用 prev2 和 prev1 滾動更新
    • 空間從 O(n) 降到 O(1)
  • 理解轉移方程：
    • max(不偷當前房, 偷當前房)
    • max(dp[i-1], dp[i-2] + nums[i])
    • 核心：不能偷相鄰房屋
  • 邊界情況：
    • 空陣列 → 0
    • 只有一間房 → nums[0]
    • 兩間房 → max(nums[0], nums[1])
    • 所有房屋金額相同
  • 常見錯誤：
    • 忘記處理只有一間房的情況
    • 索引混亂（dp 索引 vs nums 索引）
    • 初始值設置錯誤
  • 面試技巧：
    • 先說明問題本質：相鄰不能偷
    • 畫圖展示 dp 陣列變化
    • 從 O(n) 空間優化到 O(1)
    • 可以討論遞迴解法
  • 延伸思考：
