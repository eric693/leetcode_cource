41. First Missing Positive
Hard

題目描述

給定一個未排序的整數陣列 nums，找出其中沒有出現的最小的正整數。

進階：你能實現時間複雜度為 O(n) 並且只使用常數級別額外空間的解決方案嗎？

範例 1:
輸入: nums = [1,2,0]
輸出: 3
解釋: 範圍 [1,2] 中的數字都出現了，所以最小的缺失正整數是 3。
範例 2:
輸入: nums = [3,4,-1,1]
輸出: 2
解釋: 1 在陣列中，但 2 沒有。
範例 3:
輸入: nums = [7,8,9,11,12]
輸出: 1
解釋: 最小的正整數 1 沒有出現。
限制條件:
1 <= nums.length <= 10^5
-2^31 <= nums[i] <= 2^31 - 1

解題思路

核心洞察

關鍵觀察：

對於長度為 n 的陣列，答案必定在 [1, n+1] 範圍內。

  • 如果 1~n 都出現了，答案是 n+1
  • 否則，答案是 1~n 中缺失的最小正整數

思路：將陣列當作 Hash Table 使用！

將數字 i 放到索引 i-1 的位置，然後遍歷找第一個不匹配的位置。

方法：原地哈希（推薦）⭐

利用陣列本身作為 Hash Table，實現 O(1) 空間。

核心思想：

將數字 i 放到索引 i-1 的位置（如果 i 在 [1, n] 範圍內）。

步驟：

  1. 遍歷陣列，對每個元素：
    • 如果 1 ≤ nums[i] ≤ n 且 nums[i] 不在正確位置
    • 交換 nums[i] 到索引 nums[i]-1
    • 重複直到當前位置無法交換
  2. 再次遍歷陣列，找第一個 nums[i] != i+1 的位置
  3. 返回 i+1（或 n+1 如果都匹配）

視覺化範例：nums = [3,4,-1,1]

初始: [3, 4, -1, 1]
索引:  0  1   2  3

目標: 讓 1 在索引 0，2 在索引 1，3 在索引 2...

步驟 1: i=0, nums[0]=3
  3 應該在索引 2
  交換 nums[0] 和 nums[2]: [-1, 4, 3, 1]

步驟 2: i=0, nums[0]=-1
  -1 不在 [1,n] 範圍，跳過

步驟 3: i=1, nums[1]=4
  4 應該在索引 3
  交換 nums[1] 和 nums[3]: [-1, 1, 3, 4]

步驟 4: i=1, nums[1]=1
  1 應該在索引 0
  交換 nums[1] 和 nums[0]: [1, -1, 3, 4]

步驟 5: i=1, nums[1]=-1
  -1 不在 [1,n] 範圍，跳過

步驟 6: i=2, nums[2]=3
  3 已在索引 2（正確位置）

步驟 7: i=3, nums[3]=4
  4 已在索引 3（正確位置）

最終: [1, -1, 3, 4]
      ↑   ↑
    索引 0 正確（1）
    索引 1 錯誤（-1 != 2）

答案: 2
為什麼這個方法有效？
  • 每個數字最多被交換一次：時間 O(n)
  • 原地操作：空間 O(1)
  • 關鍵：忽略不在 [1, n] 範圍的數字

程式碼實現

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        """
        方法：原地哈希
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        n = len(nums)
        
        # 步驟 1: 將每個數字放到正確位置
        for i in range(n):
            # 持續交換直到當前位置無法交換
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                # 將 nums[i] 放到索引 nums[i]-1
                correct_idx = nums[i] - 1
                nums[i], nums[correct_idx] = nums[correct_idx], nums[i]
        
        # 步驟 2: 找第一個不匹配的位置
        for i in range(n):
            if nums[i] != i + 1:
                return i + 1
        
        # 所有位置都匹配，答案是 n+1
        return n + 1
#include <vector>

class Solution {
public:
    /**
     * 方法：原地哈希
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int firstMissingPositive(vector& nums) {
        int n = nums.size();
        
        // 步驟 1: 將每個數字放到正確位置
        for (int i = 0; i < n; i++) {
            // 持續交換直到當前位置無法交換
            while (nums[i] >= 1 && nums[i] <= n && 
                   nums[nums[i] - 1] != nums[i]) {
                // 將 nums[i] 放到索引 nums[i]-1
                int correct_idx = nums[i] - 1;
                swap(nums[i], nums[correct_idx]);
            }
        }
        
        // 步驟 2: 找第一個不匹配的位置
        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }
        
        // 所有位置都匹配，答案是 n+1
        return n + 1;
    }
};
class Solution {
    /**
     * 方法：原地哈希
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int firstMissingPositive(int[] nums) {
        int n = nums.length;
        
        // 步驟 1: 將每個數字放到正確位置
        for (int i = 0; i < n; i++) {
            // 持續交換直到當前位置無法交換
            while (nums[i] >= 1 && nums[i] <= n && 
                   nums[nums[i] - 1] != nums[i]) {
                // 將 nums[i] 放到索引 nums[i]-1
                int correctIdx = nums[i] - 1;
                int temp = nums[i];
                nums[i] = nums[correctIdx];
                nums[correctIdx] = temp;
            }
        }
        
        // 步驟 2: 找第一個不匹配的位置
        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }
        
        // 所有位置都匹配，答案是 n+1
        return n + 1;
    }
}

複雜度分析

時間複雜度: O(n)

雖然有雙重循環，但每個元素最多被交換一次到正確位置。
總共最多 n 次交換 + n 次遍歷 = O(n)。

空間複雜度: O(1)

原地修改陣列，只使用常數個變數。

重點提示

  • 核心洞察：答案在 [1, n+1] 範圍內
  • 原地哈希：將陣列當作 Hash Table
  • 交換策略：將數字 i 放到索引 i-1
  • 時間分析：
    • 每個數字最多被交換一次
    • 雖然是 while 循環，總複雜度仍是 O(n)
  • 邊界條件：
    • 忽略 ≤ 0 的數字
    • 忽略 > n 的數字
    • 避免死循環：nums[nums[i]-1] != nums[i]
  • 常見錯誤：
    • 忘記檢查重複數字（死循環）
    • 交換條件錯誤
    • 索引越界
  • 面試技巧：
    • 先說明答案範圍在 [1, n+1]
    • 提出原地哈希思路
    • 用例子演示交換過程
    • 說明為什麼時間是 O(n)
  • 其他方法對比：
    • 排序：O(n log n) 時間
    • Set：O(n) 空間
    • 原地哈希：O(n) 時間 + O(1) 空間 ⭐
