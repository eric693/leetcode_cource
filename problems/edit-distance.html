72. Edit Distance
Hard

題目描述

給你兩個單詞 word1 和 word2，請返回將 word1 轉換成 word2 所使用的最少操作數。

你可以對一個單詞進行如下三種操作：

  • 插入一個字符
  • 刪除一個字符
  • 替換一個字符
範例 1:
輸入: word1 = "horse", word2 = "ros"
輸出: 3
解釋: 
horse -> rorse (將 'h' 替換為 'r')
rorse -> rose (刪除 'r')
rose -> ros (刪除 'e')
範例 2:
輸入: word1 = "intention", word2 = "execution"
輸出: 5
解釋: 
intention -> inention (刪除 't')
inention -> enention (將 'i' 替換為 'e')
enention -> exention (將 'n' 替換為 'x')
exention -> exection (將 'n' 替換為 'c')
exection -> execution (插入 'u')
限制條件:
0 <= word1.length, word2.length <= 500
word1 和 word2 由小寫英文字母組成

解題思路

核心觀察

這是一個經典的編輯距離（Levenshtein Distance）問題：

  • 三種操作：插入、刪除、替換
  • 需要找到最少操作次數
  • 使用動態規劃求解

關鍵：dp[i][j] 表示 word1 前 i 個字符轉換為 word2 前 j 個字符的最少操作數

方法：動態規劃（推薦）⭐⭐⭐

使用二維 DP 記錄每個子問題的最優解。

狀態定義：

  • dp[i][j]：word1 前 i 個字符轉換為 word2 前 j 個字符的最少操作數
  • 初始狀態：
    • dp[0][j] = j（空字串轉換為 word2 前 j 個字符，需要插入 j 次）
    • dp[i][0] = i（word1 前 i 個字符轉換為空字串，需要刪除 i 次）
  • 轉移方程：
    • 如果 word1[i-1] == word2[j-1]：
      • dp[i][j] = dp[i-1][j-1]（不需要操作）
    • 否則，取三種操作的最小值：
      • 插入：dp[i][j-1] + 1
      • 刪除：dp[i-1][j] + 1
      • 替換：dp[i-1][j-1] + 1
      • dp[i][j] = min(插入, 刪除, 替換)

視覺化範例：word1 = "horse", word2 = "ros"

DP 表格（行=word1, 列=word2）:

       ""  r   o   s
    ┌───┬───┬───┬───┐
""  │ 0 │ 1 │ 2 │ 3 │
    ├───┼───┼───┼───┤
h   │ 1 │ 1 │ 2 │ 3 │
    ├───┼───┼───┼───┤
o   │ 2 │ 2 │ 1 │ 2 │
    ├───┼───┼───┼───┤
r   │ 3 │ 2 │ 2 │ 2 │
    ├───┼───┼───┼───┤
s   │ 4 │ 3 │ 3 │ 2 │
    ├───┼───┼───┼───┤
e   │ 5 │ 4 │ 4 │ 3 │
    └───┴───┴───┴───┘

計算過程：

dp[1][1]: h vs r
  不相等，取 min:
  插入: dp[1][0] + 1 = 2
  刪除: dp[0][1] + 1 = 2
  替換: dp[0][0] + 1 = 1 ✓
  dp[1][1] = 1

dp[2][2]: ho vs ro
  o == o，相等！
  dp[2][2] = dp[1][1] = 1

dp[3][1]: hor vs r
  r == r，相等！
  dp[3][1] = dp[2][0] = 2

...繼續填充表格...

最終答案: dp[5][3] = 3

三種操作的理解：

假設目前比較 word1[i] 和 word2[j]

1. 插入操作：dp[i][j-1] + 1
   - 將 word1[0..i] 轉換為 word2[0..j-1]
   - 然後插入 word2[j]

2. 刪除操作：dp[i-1][j] + 1
   - 先刪除 word1[i]
   - 然後將 word1[0..i-1] 轉換為 word2[0..j]

3. 替換操作：dp[i-1][j-1] + 1
   - 將 word1[0..i-1] 轉換為 word2[0..j-1]
   - 然後將 word1[i] 替換為 word2[j]
空間優化

觀察到 dp[i][j] 只依賴 dp[i-1][...] 和 dp[i][...]，可以用一維陣列優化。

  • 只需保存當前行和上一行
  • 空間從 O(m×n) 降到 O(n)
  • 需要額外變數記錄左上角的值

程式碼實現

class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        """
        方法：動態規劃
        時間複雜度: O(m * n)
        空間複雜度: O(m * n)
        """
        m, n = len(word1), len(word2)
        
        # 創建 DP 表格
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        
        # 初始化：空字串轉換
        for i in range(m + 1):
            dp[i][0] = i  # 刪除 i 個字符
        for j in range(n + 1):
            dp[0][j] = j  # 插入 j 個字符
        
        # 填充 DP 表格
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    # 字符相同，不需要操作
                    dp[i][j] = dp[i-1][j-1]
                else:
                    # 取三種操作的最小值
                    insert = dp[i][j-1] + 1      # 插入
                    delete = dp[i-1][j] + 1      # 刪除
                    replace = dp[i-1][j-1] + 1   # 替換
                    dp[i][j] = min(insert, delete, replace)
        
        return dp[m][n]
    
    def minDistance_optimized(self, word1: str, word2: str) -> int:
        """
        空間優化版本
        時間複雜度: O(m * n)
        空間複雜度: O(n)
        """
        m, n = len(word1), len(word2)
        
        # 只保存一行
        prev = list(range(n + 1))
        
        for i in range(1, m + 1):
            curr = [i]  # 當前行的第一個元素
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    curr.append(prev[j-1])
                else:
                    curr.append(min(
                        curr[j-1] + 1,    # 插入
                        prev[j] + 1,      # 刪除
                        prev[j-1] + 1     # 替換
                    ))
            prev = curr
        
        return prev[n]
    
    def minDistance_1d(self, word1: str, word2: str) -> int:
        """
        進一步優化：原地更新
        """
        m, n = len(word1), len(word2)
        dp = list(range(n + 1))
        
        for i in range(1, m + 1):
            prev = dp[0]
            dp[0] = i
            for j in range(1, n + 1):
                temp = dp[j]
                if word1[i-1] == word2[j-1]:
                    dp[j] = prev
                else:
                    dp[j] = min(dp[j-1], dp[j], prev) + 1
                prev = temp
        
        return dp[n]

class Solution {
    /**
     * 方法：動態規劃
     * 時間複雜度: O(m * n)
     * 空間複雜度: O(m * n)
     */
    public int minDistance(String word1, String word2) {
        int m = word1.length(), n = word2.length();
        
        int[][] dp = new int[m + 1][n + 1];
        
        // 初始化
        for (int i = 0; i <= m; i++) {
            dp[i][0] = i;
        }
        for (int j = 0; j <= n; j++) {
            dp[0][j] = j;
        }
        
        // 填充表格
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i-1) == word2.charAt(j-1)) {
                    dp[i][j] = dp[i-1][j-1];
                } else {
                    dp[i][j] = Math.min(
                        Math.min(dp[i][j-1], dp[i-1][j]),
                        dp[i-1][j-1]
                    ) + 1;
                }
            }
        }
        
        return dp[m][n];
    }
    
    /**
     * 空間優化版本
     * 時間複雜度: O(m * n)
     * 空間複雜度: O(n)
     */
    public int minDistance_optimized(String word1, String word2) {
        int m = word1.length(), n = word2.length();
        int[] dp = new int[n + 1];
        
        for (int j = 0; j <= n; j++) {
            dp[j] = j;
        }
        
        for (int i = 1; i <= m; i++) {
            int prev = dp[0];
            dp[0] = i;
            for (int j = 1; j <= n; j++) {
                int temp = dp[j];
                if (word1.charAt(i-1) == word2.charAt(j-1)) {
                    dp[j] = prev;
                } else {
                    dp[j] = Math.min(Math.min(dp[j-1], dp[j]), prev) + 1;
                }
                prev = temp;
            }
        }
        
        return dp[n];
    }
}

複雜度分析

時間複雜度: O(m × n)

需要填充 m × n 的 DP 表格，每個格子的計算是 O(1)。

空間複雜度: O(m × n) 或 O(n)

基本版本：O(m × n) 二維陣列。 優化版本：O(n) 一維陣列。

重點提示

  • 經典 DP 問題：編輯距離是字串 DP 的代表題
  • 三種操作的理解：
    • 插入：在 word1 中插入 word2[j]
    • 刪除：刪除 word1[i]
    • 替換：將 word1[i] 替換為 word2[j]
    • 關鍵：理解每種操作對應的子問題
  • 狀態轉移關鍵：
    • 字符相同：dp[i][j] = dp[i-1][j-1]
    • 字符不同：取三種操作的最小值 + 1
    • 初始化很重要（邊界條件）
  • DP 表格理解：
    • 行代表 word1 的前綴
    • 列代表 word2 的前綴
    • dp[i][j] = word1[0..i-1] 轉換為 word2[0..j-1]
  • 初始化邊界：
    • dp[0][j] = j（空串 → word2[0..j-1]，插入 j 次）
    • dp[i][0] = i（word1[0..i-1] → 空串，刪除 i 次）
    • dp[0][0] = 0（空串 → 空串，0 次操作）
  • 空間優化技巧：
    • 只需當前行和上一行
    • 使用滾動陣列
    • 需要額外變數保存左上角值（prev）
  • 邊界情況：
    • 兩個空字串 → 0
    • 一個空字串 → 另一個的長度
    • 兩個相同字串 → 0
    • 完全不同的字串 → max(m, n)
  • 常見錯誤：
    • 索引混亂（dp 索引 vs 字串索引）
    • 初始化邊界錯誤
    • 狀態轉移方程寫錯
    • 空間優化時 prev 變數處理錯誤
  • 面試技巧：
    • 先畫 DP 表格示意圖
    • 說明三種操作的含義
    • 展示狀態轉移過程
    • 可以提到空間優化
    • 舉小例子驗證邏輯
  • 延伸思考：
    • 如何追蹤實際的操作序列？（記錄選擇）
    • 如果不同操作有不同代價？（加權編輯距離）
