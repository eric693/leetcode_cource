350. Intersection of Two Arrays II
Easy

題目描述

給定兩個整數陣列 nums1 和 nums2，返回它們的交集。輸出結果中的每個元素出現的次數，應與元素在兩個陣列中都出現的次數一致（如果出現次數不一致，則考慮較小的次數）。可以不考慮輸出結果的順序。

範例 1:
輸入: nums1 = [1,2,2,1], nums2 = [2,2]
輸出: [2,2]
範例 2:
輸入: nums1 = [4,9,5], nums2 = [9,4,9,8,4]
輸出: [4,9] 或 [9,4]
限制條件:
1 <= nums1.length, nums2.length <= 1000
0 <= nums1[i], nums2[i] <= 1000
進階問題:
  • 如果給定的陣列已經排序，你將如何優化你的算法？
  • 如果 nums1 的大小比 nums2 小，哪種方法更好？
  • 如果 nums2 的元素存儲在磁盤上，內存是有限的，不能一次加載所有的元素到內存中，你該怎麼辦？

解題思路

方法一：哈希表計數（推薦）⭐

使用哈希表記錄一個陣列中每個元素的出現次數，然後遍歷另一個陣列找交集。

核心思想：

統計 nums1 中每個數字的頻率，然後遍歷 nums2，如果數字在哈希表中且計數 > 0，就加入結果。

步驟：

  1. 創建哈希表，統計 nums1 中每個數字的出現次數
  2. 遍歷 nums2：
    • 如果當前數字在哈希表中且計數 > 0：
      • 將該數字加入結果
      • 哈希表中該數字計數減 1
  3. 返回結果陣列

視覺化範例：nums1 = [1,2,2,1], nums2 = [2,2]

步驟 1: 統計 nums1
       {1: 2, 2: 2}

步驟 2: 遍歷 nums2
       檢查 2 → 在哈希表中且計數 = 2 > 0
       加入結果: [2]
       更新哈希表: {1: 2, 2: 1}

       檢查 2 → 在哈希表中且計數 = 1 > 0
       加入結果: [2, 2]
       更新哈希表: {1: 2, 2: 0}

結果: [2, 2]
方法二：雙指標（適用於已排序陣列）

如果兩個陣列已經排序，可以使用雙指標同時遍歷兩個陣列。

步驟：

  1. 先對兩個陣列排序（如果未排序）
  2. 使用兩個指標 i, j 分別指向兩個陣列的開始
  3. 比較 nums1[i] 和 nums2[j]：
    • 如果相等 → 加入結果，i++, j++
    • 如果 nums1[i] < nums2[j] → i++
    • 如果 nums1[i] > nums2[j] → j++

適用場景：陣列已排序，或排序開銷可接受

程式碼實現

from collections import Counter

class Solution:
    def intersect(self, nums1: List[int], nums2: List[int]) -> List[int]:
        """
        方法一：哈希表計數（推薦）
        時間複雜度: O(m + n)
        空間複雜度: O(min(m, n))
        """
        # 使用較小的陣列建立哈希表，節省空間
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        
        # 統計 nums1 中每個數字的頻率
        count = Counter(nums1)
        result = []
        
        # 遍歷 nums2
        for num in nums2:
            if count[num] > 0:
                result.append(num)
                count[num] -= 1
        
        return result
    
    def intersect_twopointers(self, nums1: List[int], nums2: List[int]) -> List[int]:
        """
        方法二：雙指標（適用於已排序陣列）
        時間複雜度: O(m log m + n log n) - 排序
        空間複雜度: O(1) - 不考慮輸出空間
        """
        nums1.sort()
        nums2.sort()
        
        i, j = 0, 0
        result = []
        
        while i < len(nums1) and j < len(nums2):
            if nums1[i] == nums2[j]:
                result.append(nums1[i])
                i += 1
                j += 1
            elif nums1[i] < nums2[j]:
                i += 1
            else:
                j += 1
        
        return result
#include <vector>
#include <unordered_map>
#include <algorithm>

class Solution {
public:
    /**
     * 方法一：哈希表計數（推薦）
     * 時間複雜度: O(m + n)
     * 空間複雜度: O(min(m, n))
     */
    vector intersect(vector& nums1, vector& nums2) {
        // 使用較小的陣列建立哈希表
        if (nums1.size() > nums2.size()) {
            return intersect(nums2, nums1);
        }
        
        // 統計 nums1 中每個數字的頻率
        unordered_map count;
        for (int num : nums1) {
            count[num]++;
        }
        
        vector result;
        
        // 遍歷 nums2
        for (int num : nums2) {
            if (count[num] > 0) {
                result.push_back(num);
                count[num]--;
            }
        }
        
        return result;
    }
    
    /**
     * 方法二：雙指標（適用於已排序陣列）
     * 時間複雜度: O(m log m + n log n)
     * 空間複雜度: O(1)
     */
    vector intersect_twopointers(vector& nums1, vector& nums2) {
        sort(nums1.begin(), nums1.end());
        sort(nums2.begin(), nums2.end());
        
        int i = 0, j = 0;
        vector result;
        
        while (i < nums1.size() && j < nums2.size()) {
            if (nums1[i] == nums2[j]) {
                result.push_back(nums1[i]);
                i++;
                j++;
            } else if (nums1[i] < nums2[j]) {
                i++;
            } else {
                j++;
            }
        }
        
        return result;
    }
};
import java.util.*;

class Solution {
    /**
     * 方法一：哈希表計數（推薦）
     * 時間複雜度: O(m + n)
     * 空間複雜度: O(min(m, n))
     */
    public int[] intersect(int[] nums1, int[] nums2) {
        // 使用較小的陣列建立哈希表
        if (nums1.length > nums2.length) {
            return intersect(nums2, nums1);
        }
        
        // 統計 nums1 中每個數字的頻率
        Map count = new HashMap<>();
        for (int num : nums1) {
            count.put(num, count.getOrDefault(num, 0) + 1);
        }
        
        List result = new ArrayList<>();
        
        // 遍歷 nums2
        for (int num : nums2) {
            if (count.getOrDefault(num, 0) > 0) {
                result.add(num);
                count.put(num, count.get(num) - 1);
            }
        }
        
        // 轉換為陣列
        int[] answer = new int[result.size()];
        for (int i = 0; i < result.size(); i++) {
            answer[i] = result.get(i);
        }
        
        return answer;
    }
    
    /**
     * 方法二：雙指標（適用於已排序陣列）
     * 時間複雜度: O(m log m + n log n)
     * 空間複雜度: O(1)
     */
    public int[] intersect_twopointers(int[] nums1, int[] nums2) {
        Arrays.sort(nums1);
        Arrays.sort(nums2);
        
        int i = 0, j = 0;
        List result = new ArrayList<>();
        
        while (i < nums1.length && j < nums2.length) {
            if (nums1[i] == nums2[j]) {
                result.add(nums1[i]);
                i++;
                j++;
            } else if (nums1[i] < nums2[j]) {
                i++;
            } else {
                j++;
            }
        }
        
        int[] answer = new int[result.size()];
        for (int k = 0; k < result.size(); k++) {
            answer[k] = result.get(k);
        }
        
        return answer;
    }
}

複雜度分析

方法一：哈希表
  • 時間複雜度：O(m + n) - m、n 分別是兩個陣列的長度
  • 空間複雜度：O(min(m, n)) - 使用較小陣列建立哈希表
方法二：雙指標
  • 時間複雜度：O(m log m + n log n) - 排序開銷
  • 空間複雜度：O(1) - 不考慮輸出空間

重點提示

  • 與 Intersection I 的區別：
    • Intersection I (#349)：每個元素只出現一次
    • Intersection II (#350)：考慮元素出現次數 ⭐ 本題
  • 優化技巧：用較小的陣列建立哈希表，節省空間
  • 方法選擇：
    • 未排序 → 哈希表 O(m+n) ⭐
    • 已排序 → 雙指標 O(m+n)
    • 需排序 → 看情況（排序 vs 哈希表）
  • 進階問題解答：
    • 已排序 → 使用雙指標，空間 O(1)
    • nums1 較小 → 用 nums1 建立哈希表
    • nums2 在磁盤 → 分批讀取 nums2，用 nums1 的哈希表匹配
  • 邊界情況：
    • 空陣列 → 返回空
    • 無交集 → 返回空
    • 完全相同 → 返回整個陣列
  • 面試技巧：
    • 先問陣列是否已排序
    • 討論空間和時間的權衡
    • 提到優化：用較小陣列建哈希表
    • 回答進階問題展現深度
