15. 3Sum
Medium

題目描述

給定一個整數陣列 nums，返回所有和為 0 的三元組 [nums[i], nums[j], nums[k]]，且滿足 i != j、i != k 且 j != k。

注意：答案中不可以包含重複的三元組。

範例 1:
輸入: nums = [-1,0,1,2,-1,-4]
輸出: [[-1,-1,2],[-1,0,1]]
解釋:
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0
去重後的不同三元組是 [-1,0,1] 和 [-1,-1,2]
注意，輸出的順序和三元組的順序並不重要
範例 2:
輸入: nums = [0,1,1]
輸出: []
解釋: 唯一可能的三元組和不為 0
範例 3:
輸入: nums = [0,0,0]
輸出: [[0,0,0]]
解釋: 唯一可能的三元組和為 0
限制條件:
3 <= nums.length <= 3000
-10^5 <= nums[i] <= 10^5

解題思路

⚠️ 關鍵難點

本題的難點在於去重！必須避免產生重複的三元組。

方法：排序 + 雙指標

先排序陣列，然後固定一個數，用雙指標找另外兩個數。

核心思想：

3Sum 可以轉換為 2Sum：固定第一個數 nums[i]，問題變成在剩餘陣列中找兩個數，使得和為 -nums[i]。

步驟：

  1. 將陣列排序
  2. 遍歷陣列，固定第一個數 nums[i]：
    • 如果 nums[i] > 0，後面的數都大於 0，不可能和為 0，直接 break
    • 使用雙指標在 i+1 到 n-1 範圍內找另外兩個數
    • 左指標 left = i+1，右指標 right = n-1
    • 計算三數之和：
      • 如果 sum = 0，記錄結果，移動兩個指標並跳過重複元素
      • 如果 sum < 0，left++（需要更大的數）
      • 如果 sum > 0，right--（需要更小的數）
  3. 跳過重複的第一個數

視覺化範例：nums = [-1,0,1,2,-1,-4]

排序後: [-4, -1, -1, 0, 1, 2]

i=0, nums[i]=-4, target=4
     left=1(-1), right=5(2): -1+2=1 < 4, left++
     left=2(-1), right=5(2): -1+2=1 < 4, left++
     left=3(0), right=5(2): 0+2=2 < 4, left++
     left=4(1), right=5(2): 1+2=3 < 4, left++
     沒找到

i=1, nums[i]=-1, target=1
     left=2(-1), right=5(2): -1+2=1 = 1 ✓
     找到: [-1, -1, 2]
     跳過重複的 -1

i=2, nums[i]=-1 (重複,跳過)

i=3, nums[i]=0, target=0
     left=4(1), right=5(2): 1+2=3 > 0, right--
     沒找到更多

i=4, nums[i]=1, target=-1
     left=5(2), right=5: left >= right, 結束

答案: [[-1,-1,2], [-1,0,1]]
去重策略（重要！）
  1. 第一個數去重： 
    if i > 0 and nums[i] == nums[i-1]:
    continue
  2. 第二個數去重： 
    while left < right and nums[left] == nums[left+1]:
    left += 1
  3. 第三個數去重： 
    while left < right and nums[right] == nums[right-1]:
    right -= 1

程式碼實現

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        """
        排序 + 雙指標
        時間複雜度: O(n²)
        空間複雜度: O(log n) 到 O(n)，取決於排序算法
        """
        nums.sort()
        result = []
        n = len(nums)
        
        for i in range(n - 2):
            # 如果第一個數已經大於 0，後面不可能找到和為 0 的三元組
            if nums[i] > 0:
                break
            
            # 跳過重複的第一個數
            if i > 0 and nums[i] == nums[i-1]:
                continue
            
            # 雙指標找另外兩個數
            left, right = i + 1, n - 1
            
            while left < right:
                total = nums[i] + nums[left] + nums[right]
                
                if total == 0:
                    # 找到一個解
                    result.append([nums[i], nums[left], nums[right]])
                    
                    # 跳過重複的第二個數
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    # 跳過重複的第三個數
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    
                    # 移動指標繼續尋找
                    left += 1
                    right -= 1
                    
                elif total < 0:
                    # 和太小，需要更大的數
                    left += 1
                else:
                    # 和太大，需要更小的數
                    right -= 1
        
        return result

#include <algorithm>

class Solution {
public:
    /**
     * 排序 + 雙指標
     * 時間複雜度: O(n²)
     * 空間複雜度: O(log n) 到 O(n)
     */
    std::vector> threeSum(std::vector& nums) {
        std::sort(nums.begin(), nums.end());
        std::vector> result;
        int n = nums.size();
        
        for (int i = 0; i < n - 2; i++) {
            // 如果第一個數已經大於 0，後面不可能找到和為 0 的三元組
            if (nums[i] > 0) {
                break;
            }
            
            // 跳過重複的第一個數
            if (i > 0 && nums[i] == nums[i-1]) {
                continue;
            }
            
            // 雙指標找另外兩個數
            int left = i + 1, right = n - 1;
            
            while (left < right) {
                int total = nums[i] + nums[left] + nums[right];
                
                if (total == 0) {
                    // 找到一個解
                    result.push_back({nums[i], nums[left], nums[right]});
                    
                    // 跳過重複的第二個數
                    while (left < right && nums[left] == nums[left + 1]) {
                        left++;
                    }
                    // 跳過重複的第三個數
                    while (left < right && nums[right] == nums[right - 1]) {
                        right--;
                    }
                    
                    // 移動指標繼續尋找
                    left++;
                    right--;
                    
                } else if (total < 0) {
                    // 和太小，需要更大的數
                    left++;
                } else {
                    // 和太大，需要更小的數
                    right--;
                }
            }
        }
        
        return result;
    }
};

複雜度分析

時間複雜度: O(n²)

排序需要 O(n log n)，外層循環 O(n)，內層雙指標 O(n)，總時間複雜度為 O(n²)。

空間複雜度: O(log n) 到 O(n)

取決於排序算法使用的空間。如果不計算結果陣列，空間複雜度為 O(log n)（快速排序）到 O(n)（歸併排序）。

重點提示

  • 去重是關鍵：必須正確處理重複元素，避免產生重複的三元組
  • 排序的作用：
    • 讓相同元素相鄰，便於去重
    • 使用雙指標策略
    • 提前剪枝（nums[i] > 0 可以 break）
  • 雙指標移動規則：
    • sum < 0 → left++（需要更大的數）
    • sum > 0 → right--（需要更小的數）
    • sum = 0 → 記錄結果，同時移動兩個指標
  • 邊界情況：
    • 陣列長度 < 3 → 返回空
    • 所有元素相同 → 檢查是否為 [0,0,0]
    • 沒有解 → 返回空陣列
  • 為什麼不用 HashSet：雙指標在已排序陣列上更高效，且去重更簡單
  • 面試技巧：
    • 先確認可以修改原陣列（排序）
    • 解釋 3Sum 轉換為 2Sum 的思路
    • 特別強調去重邏輯
  • 擴展問題：
    • 3Sum Closest (#16) - 找最接近目標值的三數之和
    • 4Sum (#18) - 四個數之和
    • 2Sum (#1) - 兩數之和
