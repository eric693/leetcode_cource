62. Unique Paths
Medium

題目描述

一個機器人位於一個 m x n 網格的左上角（起始點在下圖中標記為 "Start"）。

機器人每次只能向下或者向右移動一步。機器人試圖達到網格的右下角（在下圖中標記為 "Finish"）。

問總共有多少條不同的路徑？

範例 1:
輸入: m = 3, n = 7
輸出: 28
範例 2:
輸入: m = 3, n = 2
輸出: 3
解釋: 從左上角開始，總共有 3 條路徑可以到達右下角。
1. 向右 -> 向下 -> 向下
2. 向下 -> 向下 -> 向右
3. 向下 -> 向右 -> 向下
限制條件:
1 <= m, n <= 100

解題思路

核心觀察

這是一個經典的二維動態規劃問題：

  • 機器人只能向右或向下移動
  • 到達某個格子的路徑數 = 從上方來的路徑數 + 從左方來的路徑數
  • 第一行和第一列的路徑數都是 1（只有一條路）

關鍵：dp[i][j] = dp[i-1][j] + dp[i][j-1]

方法一：二維動態規劃（推薦理解）⭐

使用二維陣列記錄到達每個格子的路徑數。

狀態定義：

  • dp[i][j]：到達位置 (i, j) 的不同路徑數
  • 初始狀態：
    • dp[0][0] = 1（起點）
    • dp[i][0] = 1（第一列，只能向下）
    • dp[0][j] = 1（第一行，只能向右）
  • 轉移方程：
    • dp[i][j] = dp[i-1][j] + dp[i][j-1]
    • （從上方來 + 從左方來）

方法二：一維動態規劃（空間優化）⭐⭐

觀察到只需要當前行和上一行的數據，可以優化為一維陣列。

一維 DP 示意：m = 3, n = 3

初始: dp = [1, 1, 1]

處理第 1 行:
  j=1: dp[1] = dp[1] + dp[0] = 1 + 1 = 2
  j=2: dp[2] = dp[2] + dp[1] = 1 + 2 = 3
  dp = [1, 2, 3]

處理第 2 行:
  j=1: dp[1] = dp[1] + dp[0] = 2 + 1 = 3
  j=2: dp[2] = dp[2] + dp[1] = 3 + 3 = 6
  dp = [1, 3, 6]

結果: dp[2] = 6
方法三：數學公式（組合數學）⭐⭐⭐

從數學角度，這是一個組合問題。

  • 總共需要移動 (m-1) + (n-1) = m+n-2 步
  • 其中向下 (m-1) 步，向右 (n-1) 步
  • 答案 = C(m+n-2, m-1) = (m+n-2)! / ((m-1)! × (n-1)!)

時間複雜度: O(m+n)，空間複雜度: O(1)

程式碼實現

class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        """
        方法一：二維動態規劃
        時間複雜度: O(m * n)
        空間複雜度: O(m * n)
        """
        # 創建 DP 表格
        dp = [[0] * n for _ in range(m)]
        
        # 初始化第一行和第一列
        for i in range(m):
            dp[i][0] = 1
        for j in range(n):
            dp[0][j] = 1
        
        # 填充 DP 表格
        for i in range(1, m):
            for j in range(1, n):
                dp[i][j] = dp[i-1][j] + dp[i][j-1]
        
        return dp[m-1][n-1]
    
    def uniquePaths_1d(self, m: int, n: int) -> int:
        """
        方法二：一維動態規劃（空間優化，推薦）
        時間複雜度: O(m * n)
        空間複雜度: O(n)
        """
        # 只需要一行的空間
        dp = [1] * n
        
        # 逐行更新
        for i in range(1, m):
            for j in range(1, n):
                dp[j] += dp[j-1]
        
        return dp[n-1]
    
    def uniquePaths_math(self, m: int, n: int) -> int:
        """
        方法三：數學公式（組合數）
        時間複雜度: O(m + n)
        空間複雜度: O(1)
        """
        from math import comb
        
        # 總共需要走 m+n-2 步，選擇其中 m-1 步向下（或 n-1 步向右）
        return comb(m + n - 2, m - 1)
    
    def uniquePaths_math_manual(self, m: int, n: int) -> int:
        """
        方法三：手動計算組合數（避免大數溢出）
        """
        # 計算 C(m+n-2, min(m-1, n-1))
        # 選較小的數計算，減少運算量
        total_steps = m + n - 2
        down_steps = m - 1
        
        # 優化：選較小的數
        if down_steps > total_steps - down_steps:
            down_steps = total_steps - down_steps
        
        # 逐步計算，避免溢出
        result = 1
        for i in range(down_steps):
            result = result * (total_steps - i) // (i + 1)
        
        return result

     * 時間複雜度: O(m + n)

    int uniquePaths_math(int m, int n) {
        long long result = 1;
        int total_steps = m + n - 2;
        int down_steps = m - 1;
        
        // 選較小的數計算
        if (down_steps > total_steps - down_steps) {
            down_steps = total_steps - down_steps;
        }
        
        // 逐步計算組合數
        for (int i = 0; i < down_steps; i++) {
            result = result * (total_steps - i) / (i + 1);
        }
        
        return (int)result;
    }
};

    /**
     * 方法一：二維動態規劃
     * 時間複雜度: O(m * n)
     * 空間複雜度: O(m * n)
     */
    public int uniquePaths(int m, int n) {
        int[][] dp = new int[m][n];
        
        // 初始化
        for (int i = 0; i < m; i++) {
            dp[i][0] = 1;
        }
        for (int j = 0; j < n; j++) {
            dp[0][j] = 1;
        }
        
        // 填充

                dp[i][j] = dp[i-1][j] + dp[i][j-1];
            }
        }

    }
    
    /**
     * 方法二：一維動態規劃（推薦）
     * 時間複雜度: O(m * n)
     * 空間複雜度: O(n)
     */
    public int uniquePaths_1d(int m, int n) {
        int[] dp = new int[n];
        
        // 初始化
        for (int j = 0; j < n; j++) {
            dp[j] = 1;
        }

        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                dp[j] += dp[j-1];
            }
        }
        
        return dp[n-1];
    }
    
    /**
     * 方法三：數學公式
     * 時間複雜度: O(m + n)
     * 空間複雜度: O(1)

    public int uniquePaths_math(int m, int n) {
        long result = 1;

複雜度分析

方法一、二：動態規劃
  • 時間複雜度：O(m × n) - 需要填充整個 DP 表格
  • 空間複雜度：O(m × n) 或 O(n) - 取決於是否優化
方法三：數學公式
  • 時間複雜度：O(min(m, n)) - 計算組合數
  • 空間複雜度：O(1) - 只用常數空間

重點提示

  • DP 思路：每個格子的路徑數 = 上方格子 + 左方格子
  • 初始化很重要：
    • 第一行全是 1（只能向右）
    • 第一列全是 1（只能向下）
    • 忘記初始化是常見錯誤
  • 空間優化技巧：
    • 二維 → 一維：只需保留當前行
    • 原地更新：dp[j] += dp[j-1]
    • 節省空間從 O(m×n) 到 O(n)
  • 數學解法：
    • 本質是組合問題：C(m+n-2, m-1)
    • 最快但需要注意大數溢出
    • 面試可以提及但不一定要實現
  • 邊界情況：
    • m = 1 或 n = 1 → 只有 1 條路徑
    • m = n = 1 → 1 條路徑（起點即終點）
    • 正方形網格 (m = n)
  • 常見錯誤：
    • 忘記初始化第一行/第一列
    • 索引越界（i-1, j-1）
    • 一維 DP 更新順序錯誤
  • 面試技巧：
    • 先畫小格子圖示說明（如 3×3）
    • 展示 DP 表格的填充過程
    • 說明空間優化的思路
    • 提到數學解法（加分項）
