240. Search a 2D Matrix II
Medium

題目描述

編寫一個高效的算法來搜尋 m x n 矩陣 matrix 中的一個目標值 target。該矩陣具有以下特性：

  • 每行的元素從左到右升序排列。
  • 每列的元素從上到下升序排列。
範例 1:
輸入: matrix = [[1,4,7,11,15],[2,5,8,12,19],[3,6,9,16,22],[10,13,14,17,24],[18,21,23,26,30]], target = 5
輸出: true
矩陣:
 1   4   7  11  15
 2  [5]  8  12  19
 3   6   9  16  22
10  13  14  17  24
18  21  23  26  30

找到 5 ✓
範例 2:
輸入: matrix = [[1,4,7,11,15],[2,5,8,12,19],[3,6,9,16,22],[10,13,14,17,24],[18,21,23,26,30]], target = 20
輸出: false
限制條件:
m == matrix.length
n == matrix[i].length
1 <= n, m <= 300
-10^9 <= matrix[i][j] <= 10^9
每行的所有元素從左到右升序排列
每列的所有元素從上到下升序排列
-10^9 <= target <= 10^9

解題思路

核心觀察

這個矩陣有特殊的性質：

  • 每行從左到右遞增
  • 每列從上到下遞增
  • 關鍵：右上角或左下角的元素有特殊性質！

從右上角開始：比目標大就往左，比目標小就往下

方法一：從右上角開始搜尋（推薦）⭐⭐⭐

從矩陣的右上角開始搜尋，利用排序特性進行二維搜尋。

核心思想：

從右上角 (0, n-1) 開始：

  • 如果當前值 == target → 找到，返回 true
  • 如果當前值 > target → 往左移動（排除當前列）
  • 如果當前值 < target → 往下移動（排除當前行）

為什麼從右上角？

  • 右上角的左邊都比它小（同一行）
  • 右上角的下邊都比它大（同一列）
  • 可以明確地排除一整行或一整列

視覺化範例：尋找 target = 5

矩陣:

方法二：從左下角開始搜尋⭐⭐

同樣的思路，但從左下角開始也可以。

  • 如果當前值 > target → 往上移動
  • 如果當前值 < target → 往右移動
方法三：暴力法（不推薦）

遍歷整個矩陣尋找目標值。

時間複雜度 O(m×n)，沒有利用矩陣的排序特性。

方法四：對每一行進行二分搜尋

對每一行使用二分搜尋。

時間複雜度 O(m log n)，比方法一慢。

程式碼實現

class Solution:
    def searchMatrix(self, matrix: List[List[int]], target: int) -> bool:
        """
        方法一：從右上角開始搜尋（推薦）
        時間複雜度: O(m + n)
        空間複雜度: O(1)
        """
        if not matrix or not matrix[0]:
            return False
        
        m, n = len(matrix), len(matrix[0])
        row, col = 0, n - 1  # 從右上角開始
        
        while row < m and col >= 0:
            current = matrix[row][col]
            
            if current == target:
                return True
            elif current > target:
                col -= 1  # 往左移動
            else:
                row += 1  # 往下移動
        
        return False
    
    def searchMatrix_leftBottom(self, matrix: List[List[int]], target: int) -> bool:
        """
        方法二：從左下角開始搜尋
        時間複雜度: O(m + n)
        空間複雜度: O(1)
        """
        if not matrix or not matrix[0]:
            return False
        
        m, n = len(matrix), len(matrix[0])
        row, col = m - 1, 0  # 從左下角開始
        
        while row >= 0 and col < n:
            current = matrix[row][col]
            
            if current == target:
                return True
            elif current > target:
                row -= 1  # 往上移動
            else:
                col += 1  # 往右移動
        
        return False
    
    def searchMatrix_binarySearch(self, matrix: List[List[int]], target: int) -> bool:
        """
        方法三：對每一行進行二分搜尋
        時間複雜度: O(m log n)
        空間複雜度: O(1)
        """
        if not matrix or not matrix[0]:
            return False
        
        for row in matrix:
            # 檢查 target 是否可能在這一行
            if row[0] <= target <= row[-1]:
                # 二分搜尋
                left, right = 0, len(row) - 1
                while left <= right:
                    mid = (left + right) // 2
                    if row[mid] == target:
                        return True
                    elif row[mid] < target:
                        left = mid + 1
                    else:
                        right = mid - 1
        
        return False
#include <vector>
using namespace std;

class Solution {
public:
    /**
     * 方法一：從右上角開始搜尋（推薦）
     * 時間複雜度: O(m + n)
     * 空間複雜度: O(1)
     */
    bool searchMatrix(vector>& matrix, int target) {
        if (matrix.empty() || matrix[0].empty()) {
            return false;
        }
        
        int m = matrix.size(), n = matrix[0].size();
        int row = 0, col = n - 1;  // 從右上角開始
        
        while (row < m && col >= 0) {
            int current = matrix[row][col];
            
            if (current == target) {
                return true;
            } else if (current > target) {
                col--;  // 往左移動
            } else {
                row++;  // 往下移動
            }
        }
        
        return false;
    }
    
    /**
     * 方法二：從左下角開始搜尋
     * 時間複雜度: O(m + n)
     * 空間複雜度: O(1)
     */
    bool searchMatrix_leftBottom(vector>& matrix, int target) {
        if (matrix.empty() || matrix[0].empty()) {
            return false;
        }
        
        int m = matrix.size(), n = matrix[0].size();
        int row = m - 1, col = 0;  // 從左下角開始
        
        while (row >= 0 && col < n) {
            int current = matrix[row][col];
            
            if (current == target) {
                return true;
            } else if (current > target) {
                row--;  // 往上移動
            } else {
                col++;  // 往右移動
            }
        }
        
        return false;
    }
};

複雜度分析

時間複雜度: O(m + n)

最壞情況下，從右上角走到左下角，需要 m + n 步。 m 是行數，n 是列數。

空間複雜度: O(1)

只使用了兩個變數記錄位置，沒有使用額外空間。

重點提示

  • 核心技巧：從右上角或左下角開始，利用排序特性
  • 為什麼不從左上或右下？
    • 左上角：右邊和下邊都比它大，無法確定方向
    • 右下角：左邊和上邊都比它小，無法確定方向
    • 右上/左下：有明確的大小關係，可以排除
  • 搜尋路徑：
    • 右上角：大了往左，小了往下
    • 左下角：大了往上，小了往右
    • 每次排除一整行或一整列
  • 時間複雜度分析：
    • 最好：O(1)（第一次就找到）
    • 最壞：O(m + n)（走完 L 型路徑）
    • 平均：O(m + n)
  • 與 #74 Search a 2D Matrix 的區別：
    • #74：每行首元素 > 前一行尾元素（更強的條件）
    • #240：只保證行內和列內有序
    • #74 可以視為一維陣列用二分搜尋 O(log(mn))
  • 邊界情況：
    • 空矩陣 → false
    • 只有一個元素 → 比較該元素
    • target 小於左上角 → false
    • target 大於右下角 → false
  • 常見錯誤：
    • 從錯誤的角落開始（左上或右下）
    • 移動方向搞反
    • 邊界條件判斷錯誤
  • 面試技巧：
    • 先畫小矩陣示意圖
    • 說明為什麼選右上角/左下角
    • 展示搜尋路徑過程
    • 分析時間複雜度優於暴力法
  • 延伸思考：
    • 如果要找所有出現位置？（記錄所有匹配）
    • 如果矩陣不保證有序？（只能暴力）
    • 如果可以預處理矩陣？（建索引）
