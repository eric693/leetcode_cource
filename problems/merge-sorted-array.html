88. Merge Sorted Array
Easy

題目描述

給你兩個按非遞減順序排列的整數陣列 nums1 和 nums2，另有兩個整數 m 和 n，分別表示 nums1 和 nums2 中的元素數目。

請你合併 nums2 到 nums1 中，使合併後的陣列同樣按非遞減順序排列。

注意：最終，合併後陣列不應由函數返回，而是存儲在陣列 nums1 中。為了應對這種情況，nums1 的初始長度為 m + n，其中前 m 個元素表示應合併的元素，後 n 個元素為 0，應忽略。nums2 的長度為 n。

範例 1:
輸入: nums1 = [1,2,3,0,0,0], m = 3, nums2 = [2,5,6], n = 3
輸出: [1,2,2,3,5,6]
解釋: 需要合併 [1,2,3] 和 [2,5,6]
合併結果是 [1,2,2,3,5,6]
範例 2:
輸入: nums1 = [1], m = 1, nums2 = [], n = 0
輸出: [1]
解釋: 需要合併 [1] 和 []，合併結果是 [1]
範例 3:
輸入: nums1 = [0], m = 0, nums2 = [1], n = 1
輸出: [1]
解釋: 需要合併的陣列是 [] 和 [1]
注意，因為 m = 0，所以 nums1 中沒有元素，0 只是為了確保合併結果可以順利存放到 nums1 中
限制條件:
nums1.length == m + n
nums2.length == n
0 <= m, n <= 200
1 <= m + n <= 200
-10⁹ <= nums1[i], nums2[j] <= 10⁹
進階:

你可以設計一個時間複雜度為 O(m + n) 的算法來解決這個問題嗎？

解題思路

核心觀察

這道題的關鍵在於：

  • nums1 有足夠的空間容納所有元素
  • 兩個陣列都已排序
  • 需要原地操作，不能用額外陣列

關鍵：從後往前填充，避免覆蓋

方法一：從後往前合併（最優解）⭐⭐⭐

利用 nums1 尾部的空間，從後往前填充。

為什麼從後往前？

  • 從前往後會覆蓋 nums1 的元素
  • 從後往前，尾部是空的，不會覆蓋
  • 每次比較兩個陣列的最大值

算法步驟：

  1. 設置三個指標：i (nums1 最後元素)、j (nums2 最後元素)、k (合併後最後位置)
  2. 比較 nums1[i] 和 nums2[j]
  3. 較大的放到 nums1[k]
  4. 移動對應指標
  5. 重複直到處理完所有元素

視覺化範例：nums1=[1,2,3,0,0,0], m=3, nums2=[2,5,6], n=3

初始狀態:
nums1: [1, 2, 3, 0, 0, 0]
        ↑i          ↑k
nums2: [2, 5, 6]
              ↑j

步驟 1: 比較 nums1[i]=3 vs nums2[j]=6
       6 較大 → nums1[k]=6
nums1: [1, 2, 3, 0, 0, 6]
        ↑i       ↑k
nums2: [2, 5, 6]
           ↑j

步驟 2: 比較 nums1[i]=3 vs nums2[j]=5
       5 較大 → nums1[k]=5
nums1: [1, 2, 3, 0, 5, 6]
        ↑i    ↑k
nums2: [2, 5, 6]
        ↑j

步驟 3: 比較 nums1[i]=3 vs nums2[j]=2
       3 較大 → nums1[k]=3
nums1: [1, 2, 3, 3, 5, 6]
     ↑i    ↑k
nums2: [2, 5, 6]
        ↑j

步驟 4: 比較 nums1[i]=2 vs nums2[j]=2
       相等，任選一個 → nums1[k]=2
nums1: [1, 2, 2, 3, 5, 6]
     ↑i ↑k
nums2: [2, 5, 6]
     ↑j

步驟 5: 比較 nums1[i]=2 vs nums2[j]=2
       (已處理) → nums1[k]=2
nums1: [1, 2, 2, 3, 5, 6]
  ↑i↑k
nums2: [2, 5, 6]
     (j < 0)

步驟 6: nums1[i]=1 直接放入
nums1: [1, 2, 2, 3, 5, 6] ✓
↑i,k

完成！
方法二：合併後排序

簡單但不是最優解。

  • 將 nums2 複製到 nums1 後面
  • 對整個 nums1 排序
  • 時間複雜度：O((m+n) log(m+n))

程式碼實現

class Solution:
    def merge(self, nums1: List[int], m: int, nums2: List[int], n: int) -> None:
        """
        方法一：從後往前合併（最優解）
        時間複雜度: O(m + n)
        空間複雜度: O(1)
        """
        # 三個指標：i 指向 nums1 最後元素，j 指向 nums2 最後元素，k 指向合併後最後位置
        i = m - 1
        j = n - 1
        k = m + n - 1
        
        # 從後往前合併
        while i >= 0 and j >= 0:
            if nums1[i] > nums2[j]:
                nums1[k] = nums1[i]
                i -= 1
            else:
                nums1[k] = nums2[j]
                j -= 1
            k -= 1
        
        # 如果 nums2 還有剩餘，複製到 nums1
        # 注意：如果 nums1 有剩餘，已經在正確位置了
        while j >= 0:
            nums1[k] = nums2[j]
            j -= 1
            k -= 1
    
    def merge_v2(self, nums1: List[int], m: int, nums2: List[int], n: int) -> None:
        """
        方法二：合併後排序
        時間複雜度: O((m+n) log(m+n))
        空間複雜度: O(1)
        """
        # 複製 nums2 到 nums1 後面
        nums1[m:] = nums2
        # 排序
        nums1.sort()

using namespace std;

class Solution {
public:
    void merge(vector& nums1, int m, vector& nums2, int n) {
        int i = m - 1;  // nums1 最後元素
        int j = n - 1;  // nums2 最後元素
        int k = m + n - 1;  // 合併後最後位置
        
        // 從後往前合併
        while (i >= 0 && j >= 0) {
            if (nums1[i] > nums2[j]) {
                nums1[k] = nums1[i];
                i--;
            } else {
                nums1[k] = nums2[j];
                j--;
            }
            k--;
        }
        
        // 如果 nums2 還有剩餘
        while (j >= 0) {
            nums1[k] = nums2[j];
            j--;
            k--;
        }
    }
};

複雜度分析

時間複雜度: O(m + n)

每個元素最多被處理一次

空間複雜度: O(1)

只使用常數個變數，原地操作

重點提示

  • 為什麼從後往前？避免覆蓋 nums1 的元素
  • 三個指標：i (nums1)、j (nums2)、k (結果位置)
  • 處理剩餘：只需處理 nums2 的剩餘（nums1 已在正確位置）
  • 初始化：i=m-1, j=n-1, k=m+n-1
  • 比較方向：從大到小
  • 邊界情況：m=0 或 n=0
  • 常見錯誤：忘記處理 nums2 的剩餘元素
