1. Two Sum
Easy

題目描述

給定一個整數陣列 nums 和一個整數目標值 target，請你在該陣列中找出和為目標值 target 的那兩個整數，並返回它們的陣列索引。

你可以假設每種輸入只會對應一個答案。但是，陣列中同一個元素在答案裡不能重複出現。

你可以按任意順序返回答案。

範例 1:
輸入: nums = [2,7,11,15], target = 9
輸出: [0,1]
解釋: 因為 nums[0] + nums[1] == 9，返回 [0, 1]
範例 2:
輸入: nums = [3,2,4], target = 6
輸出: [1,2]
範例 3:
輸入: nums = [3,3], target = 6
輸出: [0,1]

解題思路

方法一：暴力解法

使用兩層迴圈遍歷所有可能的組合，檢查是否存在兩數之和等於目標值。

步驟：

  1. 外層迴圈遍歷每個元素
  2. 內層迴圈從當前元素的下一個位置開始
  3. 檢查兩數之和是否等於目標值
  4. 如果相等，返回兩個索引

時間複雜度：O(n²)

空間複雜度：O(1)

方法二：雜湊表（推薦）

使用雜湊表（Hash Table）儲存已遍歷過的數字及其索引。這樣可以在 O(1) 時間內查找補數是否存在。

核心思想：

對於每個數字 num，我們需要找到 target - num 是否在陣列中已經出現過。

步驟：

  1. 建立一個空的雜湊表，用於存儲數字和對應的索引
  2. 遍歷陣列中的每個元素 nums[i]
  3. 計算補數：complement = target - nums[i]
  4. 檢查補數是否在雜湊表中：
    • 如果存在，返回 [hashMap[complement], i]
    • 如果不存在，將當前數字和索引存入雜湊表

時間複雜度：O(n) - 只需遍歷一次陣列

空間複雜度：O(n) - 最壞情況下需要存儲 n 個元素

程式碼實現

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        """
        使用雜湊表找出兩數之和
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        # 建立雜湊表，鍵為數字，值為索引
        hash_map = {}
        
        # 遍歷陣列
        for i, num in enumerate(nums):
            # 計算需要的補數
            complement = target - num
            
            # 如果補數在雜湊表中，找到答案
            if complement in hash_map:
                return [hash_map[complement], i]
            
            # 將當前數字和索引存入雜湊表
            hash_map[num] = i
        
        # 題目保證有解，不會執行到這裡
        return []
#include <vector>
#include <unordered_map>

class Solution {
public:
    /**
     * 使用雜湊表找出兩數之和
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    std::vector<int> twoSum(std::vector<int>& nums, int target) {
        // 建立雜湊表，鍵為數字，值為索引
        std::unordered_map<int, int> hashMap;
        
        // 遍歷陣列
        for (int i = 0; i < nums.size(); i++) {
            // 計算補數
            int complement = target - nums[i];
            
            // 如果補數在雜湊表中，找到答案
            if (hashMap.find(complement) != hashMap.end()) {
                return {hashMap[complement], i};
            }
            
            // 將當前數字和索引存入雜湊表
            hashMap[nums[i]] = i;
        }
        
        return {};
    }
};
import java.util.HashMap;
import java.util.Map;

class Solution {
    /**
     * 使用雜湊表找出兩數之和
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    public int[] twoSum(int[] nums, int target) {
        // 建立雜湊表，鍵為數字，值為索引
        Map<Integer, Integer> hashMap = new HashMap<>();
        
        // 遍歷陣列
        for (int i = 0; i < nums.length; i++) {
            // 計算補數
            int complement = target - nums[i];
            
            // 如果補數在雜湊表中，找到答案
            if (hashMap.containsKey(complement)) {
                return new int[] {hashMap.get(complement), i};
            }
            
            // 將當前數字和索引存入雜湊表
            hashMap.put(nums[i], i);
        }
        
        return new int[] {};
    }
}

複雜度分析

時間複雜度: O(n)

我們只需要遍歷陣列一次。對於每個元素，在雜湊表中查找補數的時間為 O(1)。

空間複雜度: O(n)

最壞情況下，雜湊表需要存儲 n 個元素（當答案是最後兩個元素時）。

重點提示

  • 雜湊表的優勢：提供 O(1) 的查找時間，是解決「查找配對」類問題的經典模式
  • 為什麼可以邊遍歷邊查找：因為我們在找的是兩個不同索引的數字，當前數字的補數一定在之前已經遍歷過的數字中
  • 處理重複數字：雖然可能有重複數字，但我們使用索引作為答案，確保不會使用同一個位置的數字兩次
  • 數據結構選擇：Python 使用 dict，C++ 使用 unordered_map，Java 使用 HashMap
