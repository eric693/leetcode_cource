75. Sort Colors
Medium

題目描述

給定一個包含紅色、白色和藍色，一共 n 個元素的陣列 nums，原地對它們進行排序，使得相同顏色的元素相鄰，並按照紅色、白色、藍色順序排列。

我們使用整數 0、1 和 2 分別表示紅色、白色和藍色。

必須在不使用庫的 sort 函數的情況下解決這個問題。

範例 1:
輸入: nums = [2,0,2,1,1,0]
輸出: [0,0,1,1,2,2]
範例 2:
輸入: nums = [2,0,1]
輸出: [0,1,2]
限制條件:
n == nums.length
1 <= n <= 300
nums[i] 為 0、1 或 2
進階挑戰:

你能想出一個僅使用常數空間的一趟掃描算法嗎？

解題思路

核心觀察

這道題也被稱為「荷蘭國旗問題」（Dutch National Flag Problem）：

  • 只有三種顏色：0（紅）、1（白）、2（藍）
  • 需要原地排序
  • 要求 O(1) 空間
  • 最好一次遍歷（O(n) 時間）

關鍵：使用三指標將陣列分成三個區域

方法一：三指標（荷蘭國旗）⭐⭐⭐

使用三個指標將陣列分成三個區域。

核心思想：

  • left：指向下一個 0 應該放置的位置
  • right：指向下一個 2 應該放置的位置
  • current：當前遍歷的位置
  • 維護三個區域：[0...left-1] 全是 0，[left...current-1] 全是 1，[right+1...n-1] 全是 2

區域劃分：

[0,0,0 | 1,1,1 | ?,?,? | 2,2,2]
       ↑       ↑       ↑
      left  current  right

已處理的 0 | 已處理的 1 | 未處理 | 已處理的 2

操作規則：

  1. 如果 nums[current] = 0：
    • 交換 nums[left] 和 nums[current]
    • left++, current++
  2. 如果 nums[current] = 1：
    • 不需要交換
    • current++
  3. 如果 nums[current] = 2：
    • 交換 nums[current] 和 nums[right]
    • right--
    • current 不動（因為交換來的元素還沒檢查）

視覺化範例：nums = [2,0,2,1,1,0]

初始狀態:
nums = [2, 0, 2, 1, 1, 0]
        ↑              ↑
      left,cur       right

步驟 1: nums[cur]=2，與 right 交換
交換 nums[0] 和 nums[5]: [0, 0, 2, 1, 1, 2]
right--
nums = [0, 0, 2, 1, 1, 2]
        ↑           ↑
      left,cur    right

步驟 2: nums[cur]=0，與 left 交換
交換 nums[0] 和 nums[0]: [0, 0, 2, 1, 1, 2]
left++, cur++
nums = [0, 0, 2, 1, 1, 2]
           ↑        ↑
         left,cur right

步驟 3: nums[cur]=0，與 left 交換
交換 nums[1] 和 nums[1]: [0, 0, 2, 1, 1, 2]
left++, cur++
nums = [0, 0, 2, 1, 1, 2]
              ↑     ↑
            left,  right
              cur

步驟 4: nums[cur]=2，與 right 交換
交換 nums[2] 和 nums[4]: [0, 0, 1, 1, 2, 2]
right--
nums = [0, 0, 1, 1, 2, 2]
              ↑  ↑
            left,right
              cur

步驟 5: nums[cur]=1，直接移動
cur++
nums = [0, 0, 1, 1, 2, 2]
              ↑  ↑
            left right
                 cur

步驟 6: nums[cur]=1，直接移動
cur++
nums = [0, 0, 1, 1, 2, 2]
              ↑  ↑
            left cur,right

cur > right，結束！
最終結果: [0, 0, 1, 1, 2, 2] ✓

為什麼 current 遇到 2 不移動？

  • 與 right 交換後，current 位置得到的是未檢查的元素
  • 可能是 0、1 或 2
  • 需要再次檢查
  • 而與 left 交換後，得到的一定是已處理的 1（或自己）
方法二：計數排序

統計 0、1、2 的數量，然後重建陣列。

步驟：

  1. 第一次遍歷：統計 0、1、2 的數量
  2. 第二次遍歷：按照數量填充陣列
  • 時間複雜度：O(n)，但需要兩次遍歷
  • 空間複雜度：O(1)
  • 不符合「一趟掃描」的要求

程式碼實現

class Solution:
    def sortColors(self, nums: List[int]) -> None:
        """
        方法一：三指標（荷蘭國旗）
        時間複雜度: O(n)，一次遍歷
        空間複雜度: O(1)
        
        Do not return anything, modify nums in-place instead.
        """
        left = 0  # 下一個 0 的位置
        right = len(nums) - 1  # 下一個 2 的位置
        current = 0  # 當前檢查的位置
        
        while current <= right:
            if nums[current] == 0:
                # 將 0 交換到左邊
                nums[left], nums[current] = nums[current], nums[left]
                left += 1
                current += 1
            elif nums[current] == 1:
                # 1 在中間，不需要移動
                current += 1
            else:  # nums[current] == 2
                # 將 2 交換到右邊
                nums[current], nums[right] = nums[right], nums[current]
                right -= 1
                # current 不移動，因為交換來的元素還沒檢查
    
    def sortColors_counting(self, nums: List[int]) -> None:
        """
        方法二：計數排序
        時間複雜度: O(n)，兩次遍歷
        空間複雜度: O(1)
        """
        count = [0, 0, 0]
        
        # 統計數量
        for num in nums:
            count[num] += 1
        
        # 重建陣列
        index = 0
        for color in range(3):
            for _ in range(count[color]):
                nums[index] = color
                index += 1
#include <vector>
using namespace std;

class Solution {
public:
    /**
     * 方法一：三指標（荷蘭國旗）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    void sortColors(vector& nums) {
        int left = 0;
        int right = nums.size() - 1;
        int current = 0;
        
        while (current <= right) {
            if (nums[current] == 0) {
                swap(nums[left], nums[current]);
                left++;
                current++;
            } else if (nums[current] == 1) {
                current++;
            } else {  // nums[current] == 2
                swap(nums[current], nums[right]);
                right--;
            }
        }
    }
    
    /**
     * 方法二：計數排序
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    void sortColors_counting(vector& nums) {
        int count[3] = {0};
        
        for (int num : nums) {
            count[num]++;
        }
        
        int index = 0;
        for (int color = 0; color < 3; color++) {
            for (int i = 0; i < count[color]; i++) {
                nums[index++] = color;
            }
        }
    }
};
class Solution {
    /**
     * 方法一：三指標（荷蘭國旗）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public void sortColors(int[] nums) {
        int left = 0;
        int right = nums.length - 1;
        int current = 0;
        
        while (current <= right) {
            if (nums[current] == 0) {
                swap(nums, left, current);
                left++;
                current++;
            } else if (nums[current] == 1) {
                current++;
            } else {  // nums[current] == 2
                swap(nums, current, right);
                right--;
            }
        }
    }
    
    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
    
    /**
     * 方法二：計數排序
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public void sortColors_counting(int[] nums) {
        int[] count = new int[3];
        
        for (int num : nums) {
            count[num]++;
        }
        
        int index = 0;
        for (int color = 0; color < 3; color++) {
            for (int i = 0; i < count[color]; i++) {
                nums[index++] = color;
            }
        }
    }
}

複雜度分析

時間複雜度: O(n)

三指標法只需一次遍歷，每個元素最多被訪問一次。

空間複雜度: O(1)

只使用常數額外空間（三個指標變數）。

重點提示

  • 荷蘭國旗問題：
    • 由荷蘭科學家 Edsger Dijkstra 提出
    • 荷蘭國旗有三種顏色（紅白藍）
    • 經典的三路快排（3-way partition）
  • 三指標的精髓：
    • left 左邊全是 0
    • right 右邊全是 2
    • current 掃描未處理區域
    • 中間自然就是 1
  • 為什麼遇到 2 不移動 current？
    • 交換來的元素未知
    • 可能是 0、1 或 2
    • 需要重新判斷
    • 這是最容易出錯的地方
  • 為什麼遇到 0 可以移動 current？
    • left 指向的一定是已處理區域
    • 交換來的只可能是 1（或自己的 0）
    • 不需要重新判斷
  • 循環不變式：
    • [0, left) 全是 0
    • [left, current) 全是 1
    • (right, n-1] 全是 2
    • [current, right] 是未處理區域
  • 兩種方法比較：
    • 三指標：一次遍歷，符合進階要求
    • 計數排序：兩次遍歷，更直觀
    • 面試時優先說三指標
  • 邊界情況：
    • 只有一個元素
    • 全是同一種顏色
    • 已經排序好
    • 逆序排列
  • 常見錯誤：
    • 遇到 2 時 current++ （錯誤）
    • 循環條件寫成 current < right
    • 忘記交換後移動指標
  • 與快速排序的關係：
    • 這是三路快排的核心思想
    • 快排：< pivot, = pivot, > pivot
    • 本題：= 0, = 1, = 2
  • 面試技巧：
    • 先提計數排序（暴力解）
    • 說明兩次遍歷的缺點
    • 提出三指標優化
    • 畫圖展示指標移動
    • 強調循環不變式
  • 擴展問題：
    • 如果有 4 種顏色怎麼辦？
    • 如果有 k 種顏色？
    • 如何推廣到一般情況？
  • 實際應用：
    • 數據分類
    • 快速排序優化
    • 三路快排
  • 變形題：
    • Move Zeroes (#283) - 類似思想
    • Partition Array - 相同模式
  • 記憶口訣：「左指 0，右指 2，中掃未知別忘檢」
