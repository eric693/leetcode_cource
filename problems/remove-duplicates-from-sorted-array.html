26. Remove Duplicates from Sorted Array
Easy

題目描述

給你一個已排序的陣列 nums，請你原地刪除重複出現的元素，使每個元素只出現一次，返回刪除後陣列的新長度。元素的相對順序應該保持一致。

不要使用額外的空間，你必須在原地修改輸入陣列並在使用 O(1) 額外空間的條件下完成。

澄清說明:

為什麼返回數值是整數，但輸出的答案是陣列呢？

請注意，輸入陣列是以「引用」方式傳遞的，這意味著在函數裡修改輸入陣列對於調用者是可見的。

// nums 是以「引用」方式傳遞的。也就是說，不對實參做任何拷貝
int len = removeDuplicates(nums);

// 在函數裡修改輸入陣列對於調用者是可見的。
// 根據你的函數返回的長度, 它會打印出陣列中該長度範圍內的所有元素。
for (int i = 0; i < len; i++) {
    print(nums[i]);
}
範例 1:
輸入: nums = [1,1,2]
輸出: 2, nums = [1,2,_]
解釋: 函數應該返回新的長度 2，並且原陣列 nums 的前兩個元素被修改為 1, 2。
不需要考慮陣列中超出新長度後面的元素。
範例 2:
輸入: nums = [0,0,1,1,1,2,2,3,3,4]
輸出: 5, nums = [0,1,2,3,4,_,_,_,_,_]
解釋: 函數應該返回新的長度 5，並且原陣列 nums 的前五個元素被修改為 0, 1, 2, 3, 4。
不需要考慮陣列中超出新長度後面的元素。
限制條件:
1 <= nums.length <= 3 * 10^4
-100 <= nums[i] <= 100
nums 已按升序排列

解題思路

核心觀察

這道題的關鍵特性：

  • 陣列已排序 - 重複元素會相鄰
  • 需要原地操作 - O(1) 空間
  • 保持相對順序
  • 只需處理前 k 個元素，後面的可以忽略

關鍵：使用雙指標技巧，一個讀取，一個寫入

方法：雙指標（快慢指標）⭐⭐⭐

使用兩個指標來實現原地去重。

核心思想：

  • 慢指標（slow）：指向下一個要寫入的位置
  • 快指標（fast）：遍歷整個陣列
  • 當 nums[fast] != nums[slow-1] 時，說明找到新元素
  • 將 nums[fast] 寫入 nums[slow]，slow++

算法步驟：

  1. 如果陣列為空或只有一個元素，直接返回長度
  2. 初始化 slow = 1（第一個元素一定保留）
  3. 從 fast = 1 開始遍歷：
    • 如果 nums[fast] != nums[fast-1]（找到新元素）
    • 將 nums[fast] 複製到 nums[slow]
    • slow++
  4. 返回 slow（新陣列長度）

視覺化範例：nums = [0,0,1,1,1,2,2,3,3,4]

最終結果:
nums = [0, 1, 2, 3, 4, _, _, _, _, _]
返回 slow = 5

指標移動規則：

slow: 只在找到新元素時移動（寫入指標）
fast: 每次都移動（讀取指標）

比喻：
- fast 是「探險家」，不斷前進探索
- slow 是「記錄員」，只記錄新發現

結果：
- slow 之前的元素都是唯一的
- fast 繼續探索尋找新元素

為什麼這個方法可行？

  • 陣列已排序，相同元素一定相鄰
  • 只需比較相鄰元素就能檢測重複
  • slow 之前的元素保證都是唯一的
  • fast 在前面探路，找到新元素就告訴 slow

程式碼實現

class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        """
        雙指標法
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        if not nums:
            return 0
        
        # slow 指向下一個要寫入的位置
        slow = 1
        
        # fast 遍歷陣列
        for fast in range(1, len(nums)):
            # 找到新元素（與前一個不同）
            if nums[fast] != nums[fast - 1]:
                nums[slow] = nums[fast]
                slow += 1
        
        return slow
    
    def removeDuplicates_v2(self, nums: List[int]) -> int:
        """
        另一種寫法：比較 nums[fast] 和 nums[slow-1]
        """
        if not nums:
            return 0
        
        slow = 1
        
        for fast in range(1, len(nums)):
            if nums[fast] != nums[slow - 1]:
                nums[slow] = nums[fast]
                slow += 1
        
        return slow
#include <vector>
using namespace std;

class Solution {
public:
    /**
     * 雙指標法
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int removeDuplicates(vector& nums) {
        if (nums.empty()) {
            return 0;
        }
        
        int slow = 1;
        
        for (int fast = 1; fast < nums.size(); fast++) {
            if (nums[fast] != nums[fast - 1]) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        
        return slow;
    }
};
class Solution {
    /**
     * 雙指標法
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int removeDuplicates(int[] nums) {
        if (nums.length == 0) {
            return 0;
        }
        
        int slow = 1;
        
        for (int fast = 1; fast < nums.length; fast++) {
            if (nums[fast] != nums[fast - 1]) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        
        return slow;
    }
}

複雜度分析

時間複雜度: O(n)

只需遍歷陣列一次。

空間複雜度: O(1)

只使用兩個指標變數，原地修改。

重點提示

  • 雙指標模式：
    • 快指標：負責探索和讀取
    • 慢指標：負責寫入和記錄
    • 這是處理陣列原地修改的常用技巧
  • 為什麼從索引 1 開始？
    • 第一個元素（索引 0）一定保留
    • 它不可能是重複的
    • 所以 slow 從 1 開始
  • 比較策略：
    • 方法一：nums[fast] != nums[fast-1]
    • 方法二：nums[fast] != nums[slow-1]
    • 兩種方法都可以，效果相同
  • 已排序的重要性：
    • 如果未排序，需要先排序或用 HashSet
    • 已排序使得重複元素相鄰
    • 只需比較相鄰元素即可
  • 原地修改技巧：
    • 不需要創建新陣列
    • 直接在原陣列上修改
    • 後面的元素可以被覆蓋
  • 邊界情況：
    • 空陣列 → 返回 0
    • 只有一個元素 → 返回 1
    • 沒有重複 → 返回原長度
    • 全部重複 → 返回 1
  • 常見錯誤：
    • 忘記處理空陣列
    • slow 初始化為 0 而不是 1
    • 比較錯誤的元素
    • 忘記移動指標
  • 為什麼不用 Set？
    • Set 需要 O(n) 空間
    • 不符合原地修改的要求
    • 雙指標更優雅，O(1) 空間
  • 面試技巧：
    • 先確認陣列是否已排序
    • 畫圖展示指標移動過程
    • 說明為什麼從 1 開始
    • 測試邊界情況
  • 指標命名：
    • slow/fast 清楚表達速度差異
    • 也可以用 i/j 或 write/read
    • 好的命名幫助理解
