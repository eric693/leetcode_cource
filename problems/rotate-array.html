189. Rotate Array
Medium

題目描述

給定一個陣列，將陣列中的元素向右輪轉 k 個位置，其中 k 是非負數。

範例 1:
輸入: nums = [1,2,3,4,5,6,7], k = 3
輸出: [5,6,7,1,2,3,4]
解釋:
向右輪轉 1 步: [7,1,2,3,4,5,6]
向右輪轉 2 步: [6,7,1,2,3,4,5]
向右輪轉 3 步: [5,6,7,1,2,3,4]
範例 2:
輸入: nums = [-1,-100,3,99], k = 2
輸出: [3,99,-1,-100]
解釋:
向右輪轉 1 步: [99,-1,-100,3]
向右輪轉 2 步: [3,99,-1,-100]
限制條件:
1 <= nums.length <= 10^5
-2^31 <= nums[i] <= 2^31 - 1
0 <= k <= 10^5
進階:

• 盡可能想出更多的解決方案，至少有三種不同的方法可以解決這個問題。
• 你能使用空間複雜度為 O(1) 的原地算法解決嗎？

解題思路

方法一：三次反轉（推薦）⭐

最優解！使用三次反轉操作實現輪轉，空間複雜度 O(1)。

核心思想：

反轉操作的巧妙組合可以實現陣列輪轉。

關鍵步驟（向右輪轉 k 步）：

  1. 先對整個陣列進行反轉
  2. 再對前 k 個元素進行反轉
  3. 最後對剩餘的 n-k 個元素進行反轉

視覺化範例：nums = [1,2,3,4,5,6,7], k = 3

原始陣列: [1, 2, 3, 4, 5, 6, 7]

步驟 1: 反轉整個陣列
       [7, 6, 5, 4, 3, 2, 1]

步驟 2: 反轉前 k=3 個元素
       [5, 6, 7, 4, 3, 2, 1]
        ↑-------↑

步驟 3: 反轉剩餘的 n-k=4 個元素
       [5, 6, 7, 1, 2, 3, 4]
                 ↑---------↑

結果: [5, 6, 7, 1, 2, 3, 4] ✓

為什麼有效？

  • 第一次反轉：將後 k 個元素移到前面（但順序相反）
  • 第二次反轉：將前 k 個元素恢復正確順序
  • 第三次反轉：將剩餘元素恢復正確順序
方法二：使用額外陣列

最直觀的方法，使用新陣列存放輪轉後的結果。

步驟：

  1. 創建新陣列 result
  2. 將 nums[i] 放到 result[(i + k) % n]
  3. 將 result 複製回 nums

優缺點：

  • 優點：簡單易懂
  • 缺點：需要 O(n) 額外空間
重要細節：處理 k > n

當 k 大於陣列長度時，需要取模：k = k % n

因為輪轉 n 步後陣列回到原位，所以只需輪轉 k % n 步。

程式碼實現

class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        方法一：三次反轉（推薦）
        時間複雜度: O(n)
        空間複雜度: O(1)
        
        Do not return anything, modify nums in-place instead.
        """
        n = len(nums)
        k = k % n  # 處理 k > n 的情況
        
        # 輔助函數：反轉陣列的 [left, right] 區間
        def reverse(left: int, right: int) -> None:
            while left < right:
                nums[left], nums[right] = nums[right], nums[left]
                left += 1
                right -= 1
        
        # 三次反轉
        reverse(0, n - 1)      # 反轉整個陣列
        reverse(0, k - 1)      # 反轉前 k 個元素
        reverse(k, n - 1)      # 反轉剩餘元素
    
    def rotate_extra_space(self, nums: List[int], k: int) -> None:
        """
        方法二：使用額外陣列
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        n = len(nums)
        k = k % n
        
        # 創建新陣列
        result = [0] * n
        
        # 將元素放到新位置
        for i in range(n):
            result[(i + k) % n] = nums[i]
        
        # 複製回原陣列
        nums[:] = result
#include <vector>
#include <algorithm>

class Solution {
public:
    /**
     * 方法一：三次反轉（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    void rotate(vector& nums, int k) {
        int n = nums.size();
        k = k % n;  // 處理 k > n 的情況
        
        // 三次反轉
        reverse(nums.begin(), nums.end());           // 反轉整個陣列
        reverse(nums.begin(), nums.begin() + k);     // 反轉前 k 個元素
        reverse(nums.begin() + k, nums.end());       // 反轉剩餘元素
    }
    
    /**
     * 方法二：使用額外陣列
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    void rotate_extra_space(vector& nums, int k) {
        int n = nums.size();
        k = k % n;
        
        vector result(n);
        
        // 將元素放到新位置
        for (int i = 0; i < n; i++) {
            result[(i + k) % n] = nums[i];
        }
        
        // 複製回原陣列
        nums = result;
    }
};
class Solution {
    /**
     * 方法一：三次反轉（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public void rotate(int[] nums, int k) {
        int n = nums.length;
        k = k % n;  // 處理 k > n 的情況
        
        // 三次反轉
        reverse(nums, 0, n - 1);      // 反轉整個陣列
        reverse(nums, 0, k - 1);      // 反轉前 k 個元素
        reverse(nums, k, n - 1);      // 反轉剩餘元素
    }
    
    // 輔助函數：反轉陣列的 [left, right] 區間
    private void reverse(int[] nums, int left, int right) {
        while (left < right) {
            int temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            left++;
            right--;
        }
    }
    
    /**
     * 方法二：使用額外陣列
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    public void rotate_extra_space(int[] nums, int k) {
        int n = nums.length;
        k = k % n;
        
        int[] result = new int[n];
        
        // 將元素放到新位置
        for (int i = 0; i < n; i++) {
            result[(i + k) % n] = nums[i];
        }
        
        // 複製回原陣列
        for (int i = 0; i < n; i++) {
            nums[i] = result[i];
        }
    }
}

複雜度分析

方法一：三次反轉
  • 時間複雜度：O(n) - 三次反轉，每次 O(n)
  • 空間複雜度：O(1) - 原地操作
方法二：額外陣列
  • 時間複雜度：O(n)
  • 空間複雜度：O(n) - 需要新陣列

重點提示

  • 三次反轉技巧：這是最優解，必須掌握 ⭐
  • k % n 處理：避免不必要的輪轉，k > n 時必須取模
  • 原地操作：三次反轉法滿足 O(1) 空間要求
  • 反轉邏輯：
    • 整體反轉 → 後 k 個到前面（逆序）
    • 局部反轉 → 恢復正確順序
  • 邊界情況：
    • k = 0 → 不需輪轉
    • k = n → 等同於 k = 0
    • k > n → 取模後處理
    • n = 1 → 不需輪轉
  • 常見錯誤：
    • 忘記處理 k > n 的情況
    • 反轉順序錯誤
    • 邊界索引計算錯誤
  • 面試技巧：
    • 先說額外陣列法（展現思路）
    • 立即優化到三次反轉（展現能力）
    • 用例子驗證三次反轉的正確性
    • 說明時間和空間複雜度都是最優
