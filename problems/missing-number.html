268. Missing Number
Easy

題目描述

給定一個包含 n 個不同數字的陣列 nums，數字範圍在 [0, n] 之間。請找出這個範圍內沒有出現在陣列中的那個數字。

範例 1:
輸入: nums = [3,0,1]
輸出: 2
解釋: n = 3，因為有 3 個數字，所以所有數字都在範圍 [0,3] 內。
2 是沒有出現在陣列中的數字。
範例 2:
輸入: nums = [0,1]
輸出: 2
解釋: n = 2，因為有 2 個數字，所以所有數字都在範圍 [0,2] 內。
2 是沒有出現在陣列中的數字。
範例 3:
輸入: nums = [9,6,4,2,3,5,7,0,1]
輸出: 8
解釋: n = 9，因為有 9 個數字，所以所有數字都在範圍 [0,9] 內。
8 是沒有出現在陣列中的數字。
限制條件:
n == nums.length
1 <= n <= 10^4
0 <= nums[i] <= n
nums 中的所有數字都是唯一的
進階挑戰:

你能否實現線性時間複雜度、僅使用額外常數空間的算法？

解題思路

核心觀察

這道題有多種解法，關鍵是理解數字的特性：

  • 陣列長度為 n，包含 n 個數字
  • 數字範圍是 [0, n]，共 n+1 個數字
  • 所以恰好缺少一個數字
  • 數字都是唯一的
方法一：數學求和（推薦）⭐⭐⭐

使用數學公式計算期望總和，與實際總和的差就是缺失的數字。

核心思想：

  • 0 到 n 的總和公式：sum = n × (n + 1) / 2
  • 計算陣列實際總和
  • 期望總和 - 實際總和 = 缺失數字

視覺化範例：nums = [3,0,1]

n = 3（陣列長度）
範圍: [0, 1, 2, 3]

期望總和 = 0 + 1 + 2 + 3 = 6
         = 3 × (3 + 1) / 2 = 6

實際總和 = 3 + 0 + 1 = 4

缺失數字 = 6 - 4 = 2 ✓

優點：

  • 時間複雜度：O(n)
  • 空間複雜度：O(1)
  • 思路清晰，易於理解

注意：需要注意整數溢位問題（對於大的 n 值）

方法二：位元運算 XOR（最優）⭐⭐⭐

利用 XOR 的特性：相同數字 XOR 結果為 0。

XOR 特性：

  • a ⊕ a = 0（相同數字 XOR 為 0）
  • a ⊕ 0 = a（任何數與 0 XOR 還是自己）
  • XOR 滿足交換律和結合律

步驟：

  1. 將 0 到 n 的所有數字與陣列元素一起 XOR
  2. 相同的數字會相互抵消（XOR 為 0）
  3. 最後剩下的就是缺失的數字

視覺化範例：nums = [3,0,1]

陣列元素: 3, 0, 1
完整範圍: 0, 1, 2, 3

計算過程:
result = 0 ⊕ 1 ⊕ 2 ⊕ 3 ⊕ 3 ⊕ 0 ⊕ 1

重新排列（利用交換律）:
= (0 ⊕ 0) ⊕ (1 ⊕ 1) ⊕ (3 ⊕ 3) ⊕ 2
= 0 ⊕ 0 ⊕ 0 ⊕ 2
= 2 ✓

解釋：0、1、3 都出現兩次，XOR 後為 0
     只有 2 出現一次，所以結果是 2

優點：

  • 不會溢位
  • 空間 O(1)
  • 時間 O(n)
  • 最優雅的解法
方法三：HashSet

使用 HashSet 存儲所有數字，然後檢查哪個數字缺失。

  • 時間複雜度：O(n)
  • 空間複雜度：O(n) - 不是最優
  • 思路直觀但空間不夠優
方法四：排序

先排序，然後檢查哪個位置的數字不對。

  • 時間複雜度：O(n log n) - 不是最優
  • 空間複雜度：O(1)
  • 效率較低

程式碼實現

class Solution:
    def missingNumber(self, nums: List[int]) -> int:
        """
        方法一：數學求和
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        n = len(nums)
        expected_sum = n * (n + 1) // 2
        actual_sum = sum(nums)
        return expected_sum - actual_sum
    
    def missingNumber_xor(self, nums: List[int]) -> int:
        """
        方法二：XOR 位元運算（最優）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        result = len(nums)  # 先 XOR n
        
        for i, num in enumerate(nums):
            result ^= i ^ num
        
        return result
    
    def missingNumber_xor_v2(self, nums: List[int]) -> int:
        """
        方法二：XOR 位元運算（另一種寫法）
        """
        result = 0
        
        # XOR 所有索引
        for i in range(len(nums) + 1):
            result ^= i
        
        # XOR 所有元素
        for num in nums:
            result ^= num
        
        return result
    
    def missingNumber_set(self, nums: List[int]) -> int:
        """
        方法三：HashSet
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        num_set = set(nums)
        n = len(nums)
        
        for i in range(n + 1):
            if i not in num_set:
                return i
        
        return -1  # 不會執行到這裡
#include <vector>
#include <numeric>
using namespace std;

class Solution {
public:
    /**
     * 方法一：數學求和
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int missingNumber(vector& nums) {
        int n = nums.size();
        int expected_sum = n * (n + 1) / 2;
        int actual_sum = accumulate(nums.begin(), nums.end(), 0);
        return expected_sum - actual_sum;
    }
    
    /**
     * 方法二：XOR 位元運算
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int missingNumber_xor(vector& nums) {
        int result = nums.size();
        
        for (int i = 0; i < nums.size(); i++) {
            result ^= i ^ nums[i];
        }
        
        return result;
    }
};
class Solution {
    /**
     * 方法一：數學求和
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int missingNumber(int[] nums) {
        int n = nums.length;
        int expectedSum = n * (n + 1) / 2;
        int actualSum = 0;
        
        for (int num : nums) {
            actualSum += num;
        }
        
        return expectedSum - actualSum;
    }
    
    /**
     * 方法二：XOR 位元運算
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public int missingNumber_xor(int[] nums) {
        int result = nums.length;
        
        for (int i = 0; i < nums.length; i++) {
            result ^= i ^ nums[i];
        }
        
        return result;
    }
}

複雜度分析

時間複雜度: O(n)

需要遍歷陣列一次。

空間複雜度: O(1)

只使用常數額外空間（數學求和和 XOR 方法）。

重點提示

  • 數學求和法優點：
    • 思路最直觀
    • 易於理解和實現
    • 面試時可以先提出這個方案
  • 數學求和法注意事項：
    • 可能有整數溢位風險
    • 對於 n = 10^4，總和可達 5 × 10^7，通常沒問題
    • 但如果 n 更大，需要考慮使用 long 型別
  • XOR 方法優勢：
    • 不會溢位
    • 位元運算效率高
    • 最優雅的解法
    • 展示對位元運算的理解
  • XOR 核心原理：
    • a ⊕ a = 0
    • a ⊕ 0 = a
    • XOR 滿足交換律：a ⊕ b = b ⊕ a
    • XOR 滿足結合律：(a ⊕ b) ⊕ c = a ⊕ (b ⊕ c)
  • 為什麼 XOR 可行？
    • 把 0 到 n 的所有數字都 XOR 一遍
    • 再把陣列中的所有數字都 XOR 一遍
    • 出現兩次的數字會抵消為 0
    • 只出現一次的（缺失的）數字會保留
  • HashSet 方法：
    • 最直觀但空間不優
    • 適合先想到的第一個方案
    • 可以作為暴力解提出
  • 邊界情況：
    • nums = [0] → 缺失 1
    • nums = [1] → 缺失 0
    • 缺失的數字可能是 0 或 n
  • 常見錯誤：
    • 忘記 n 的範圍是 [0, n] 而不是 [0, n-1]
    • 數學求和公式寫錯
    • XOR 的初始值設錯
  • 面試技巧：
    • 先說 HashSet 方法（暴力解）
    • 然後優化到數學求和
    • 最後提出 XOR 方法（展示深度）
    • 分析各方法的優缺點
  • 公式推導：
    • 0 + 1 + 2 + ... + n = n(n+1)/2
    • 這是等差數列求和公式
    • 首項 a₁ = 0，末項 aₙ = n，項數 = n+1
    • 和 = (首項 + 末項) × 項數 / 2
  • 延伸思考：
    • 如果缺失兩個數字怎麼辦？
    • 如果有重複數字怎麼辦？
    • 如何找到缺失和重複的數字？
