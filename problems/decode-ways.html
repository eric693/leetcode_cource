91. Decode Ways
Medium

題目描述

一條包含字母 A-Z 的消息通過以下映射進行編碼：

'A' -> "1"
'B' -> "2"
...
'Z' -> "26"

要解碼已編碼的消息，所有數字必須基於上述映射的方法，反向映射回字母（可能有多種方法）。例如，"11106" 可以解碼為：

  • "AAJF" (1 1 10 6)
  • "KJF" (11 10 6)

注意，消息不能分組為 (1 11 06)，因為 "06" 不能映射為 "F"（因為 "6" 和 "06" 並不相同）。

給你一個只含數字的非空字串 s，請計算並返回解碼方法的總數。

範例 1:
輸入: s = "12"
輸出: 2
解釋: 它可以解碼為 "AB"（1 2）或者 "L"（12）。
範例 2:
輸入: s = "226"
輸出: 3
解釋: 它可以解碼為 "BZ" (2 26), "VF" (22 6), 或者 "BBF" (2 2 6)。
範例 3:
輸入: s = "06"
輸出: 0
解釋: "06" 無法映射到 "F"，因為有前導零（"6" 和 "06" 並不相同）。
限制條件:
1 <= s.length <= 100
s 只包含數字，並且可能包含前導零

解題思路

核心觀察

這是一個典型的動態規劃問題，類似爬樓梯：

  • 每次可以解碼 1 位數字（1-9）
  • 或解碼 2 位數字（10-26）
  • 需要統計所有可能的解碼方式

關鍵：dp[i] 表示解碼前 i 個字符的方法數

方法一：動態規劃（推薦）⭐⭐

使用 DP 記錄每個位置的解碼方法數。

狀態定義：

  • dp[i]：字串前 i 個字符的解碼方法數
  • 初始狀態：
    • dp[0] = 1（空字串，一種方法）
    • dp[1] = 1 如果 s[0] != '0'，否則 0
  • 轉移方程：
    • 單獨解碼第 i 個字符：如果 s[i-1] 是 '1'-'9'，dp[i] += dp[i-1]
    • 和前一個字符一起解碼：如果 s[i-2:i] 是 '10'-'26'，dp[i] += dp[i-2]

視覺化範例：s = "226"

字串: 2  2  6
索引: 0  1  2

初始化:
dp[0] = 1 (空字串)
dp[1] = 1 (s[0]='2' 可以解碼為 'B')

計算 dp[2] (前2個字符 "22"):
  - 單獨 '2': s[1]='2' ✓ → dp[2] += dp[1] = 1
  - 組合 '22': "22" 在 [10,26] ✓ → dp[2] += dp[0] = 1
  dp[2] = 2 (可以是 "BB" 或 "V")

計算 dp[3] (前3個字符 "226"):
  - 單獨 '6': s[2]='6' ✓ → dp[3] += dp[2] = 2
  - 組合 '26': "26" 在 [10,26] ✓ → dp[3] += dp[1] = 1
  dp[3] = 3

結果: 3 種解碼方式
  1. "BBF" (2, 2, 6)
  2. "VF"  (22, 6)
  3. "BZ"  (2, 26)
方法二：空間優化（O(1) 空間）⭐

觀察到 dp[i] 只依賴 dp[i-1] 和 dp[i-2]，可以用兩個變數優化空間。

滾動變數示意：s = "226"

初始: prev2=1, prev1=1

處理 s[1]='2':
  current = 0
  單獨: '2' ✓ → current += prev1 = 1
  組合: '22' ✓ → current += prev2 = 1
  current = 2
  更新: prev2=1, prev1=2

處理 s[2]='6':
  current = 0
  單獨: '6' ✓ → current += prev1 = 2
  組合: '26' ✓ → current += prev2 = 1
  current = 3
  
結果: 3
邊界情況處理
  • '0' 的處理：
    • '0' 不能單獨解碼
    • '10' 和 '20' 可以解碼
    • '30', '40'... 無法解碼 → 返回 0
    • 前導零：如 "06" → 返回 0
  • 兩位數判斷：必須在 [10, 26] 範圍內
  • 連續的 '0'：如 "100" 需要特別處理

程式碼實現

class Solution:
    def numDecodings(self, s: str) -> int:
        """
        方法一：動態規劃
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        if not s or s[0] == '0':
            return 0
        
        n = len(s)
        dp = [0] * (n + 1)
        dp[0] = 1  # 空字串
        dp[1] = 1  # 第一個字符（已確認不是 '0'）
        
        for i in range(2, n + 1):
            # 檢查單個字符（第 i 個字符，索引 i-1）
            one_digit = int(s[i-1])
            if 1 <= one_digit <= 9:
                dp[i] += dp[i-1]
            
            # 檢查兩個字符
            two_digits = int(s[i-2:i])
            if 10 <= two_digits <= 26:
                dp[i] += dp[i-2]
        
        return dp[n]
    
    def numDecodings_optimized(self, s: str) -> int:
        """
        方法二：空間優化（O(1) 空間）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        if not s or s[0] == '0':
            return 0
        
        n = len(s)
        # prev2 = dp[i-2], prev1 = dp[i-1]
        prev2, prev1 = 1, 1
        
        for i in range(2, n + 1):
            current = 0
            
            # 單個字符
            one_digit = int(s[i-1])
            if 1 <= one_digit <= 9:
                current += prev1
            
            # 兩個字符
            two_digits = int(s[i-2:i])
            if 10 <= two_digits <= 26:
                current += prev2
            
            # 滾動更新
            prev2, prev1 = prev1, current
        
        return prev1
    
    def numDecodings_clean(self, s: str) -> int:
        """
        簡潔版本
        """
        if not s or s[0] == '0':
            return 0
        
        prev2, prev1 = 1, 1
        
        for i in range(1, len(s)):
            current = 0
            
            # 單個字符
            if s[i] != '0':
                current += prev1
            
            # 兩個字符
            two_digits = int(s[i-1:i+1])
            if 10 <= two_digits <= 26:
                current += prev2
            
            # 如果 current 仍為 0，說明無法解碼
            if current == 0:
                return 0
            
            prev2, prev1 = prev1, current
        
        return prev1
#include <string>
#include <vector>
using namespace std;

class Solution {
public:
    /**
     * 方法一：動態規劃
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    int numDecodings(string s) {
        if (s.empty() || s[0] == '0') {
            return 0;
        }
        
        int n = s.length();
        vector dp(n + 1, 0);
        dp[0] = 1;
        dp[1] = 1;
        
        for (int i = 2; i <= n; i++) {
            // 單個字符
            int oneDigit = s[i-1] - '0';
            if (oneDigit >= 1 && oneDigit <= 9) {
                dp[i] += dp[i-1];
            }
            
            // 兩個字符
            int twoDigits = stoi(s.substr(i-2, 2));
            if (twoDigits >= 10 && twoDigits <= 26) {
                dp[i] += dp[i-2];
            }
        }
        
        return dp[n];
    }
    
    /**
     * 方法二：空間優化
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    int numDecodings_optimized(string s) {
        if (s.empty() || s[0] == '0') {
            return 0;
        }
        
        int n = s.length();
        int prev2 = 1, prev1 = 1;
        
        for (int i = 2; i <= n; i++) {
            int current = 0;
            
            // 單個字符
            int oneDigit = s[i-1] - '0';
            if (oneDigit >= 1 && oneDigit <= 9) {
                current += prev1;
            }
            
            // 兩個字符
            int twoDigits = stoi(s.substr(i-2, 2));
            if (twoDigits >= 10 && twoDigits <= 26) {
                current += prev2;
            }
            
            prev2 = prev1;
            prev1 = current;
        }
        
        return prev1;
    }
};

複雜度分析

時間複雜度: O(n)

只需遍歷字串一次，n 是字串長度。

空間複雜度: O(n) 或 O(1)

基本 DP：O(n) 陣列空間。 優化版本：O(1) 只用兩個變數。

重點提示

  • DP 思路：類似爬樓梯，每次可以走 1 步或 2 步
  • 關鍵判斷：
    • 單個字符：必須是 '1'-'9'（不能是 '0'）
    • 兩個字符：必須在 [10, 26] 範圍內
  • '0' 的陷阱：
    • '0' 不能單獨解碼
    • '10' 和 '20' 可以，其他如 '30' 不行
    • 前導零直接返回 0
    • 連續零如 "100" 要小心處理
  • 邊界情況：
    • 空字串或首字符為 '0' → 0
    • 只有一個字符 → 1（如果不是 '0'）
    • 全是 '1' → 斐波那契數列
    • "27"、"91" 等超過 26 的兩位數
  • 常見錯誤：
    • 忘記檢查 '0' 的特殊情況
    • 兩位數範圍寫成 [1, 26] 而非 [10, 26]
    • 字串索引和 dp 索引搞混

    • 先舉例說明：如 "226" 的三種解碼
    • 說明 dp[i] 的定義
