35. Search Insert Position
Easy

題目描述

給定一個排序陣列和一個目標值，在陣列中找到目標值，並返回其索引。如果目標值不存在於陣列中，返回它將會被按順序插入的位置。

請必須使用時間複雜度為 O(log n) 的演算法。

範例 1:
輸入: nums = [1,3,5,6], target = 5
輸出: 2
範例 2:
輸入: nums = [1,3,5,6], target = 2
輸出: 1
範例 3:
輸入: nums = [1,3,5,6], target = 7
輸出: 4
限制條件:
1 <= nums.length <= 10^4
-10^4 <= nums[i] <= 10^4
nums 為無重複元素的升序排列陣列
-10^4 <= target <= 10^4

解題思路

核心觀察

這道題的關鍵特性：

  • 陣列已排序 - 提示使用二分搜尋
  • 要求 O(log n) - 必須二分搜尋
  • 找不到時返回插入位置
  • 無重複元素

關鍵：這是二分搜尋的變形 - 找「第一個 >= target 的位置」

方法：二分搜尋 ⭐⭐⭐

使用二分搜尋找到目標值或插入位置。

核心思想：

  • 標準二分搜尋模板
  • 找到目標值 → 返回索引
  • 找不到 → left 就是插入位置

為什麼 left 是插入位置？

  • 循環結束時：left > right
  • left 指向第一個 >= target 的位置
  • right 指向最後一個 < target 的位置
  • 插入位置就是 left

視覺化範例 1：nums = [1,3,5,6], target = 5（找到）

初始: [1, 3, 5, 6], target = 5
       ↑     ↑  ↑
      left  mid right

mid = (0+3)/2 = 1
nums[1] = 3 < 5
left = mid + 1 = 2

       [1, 3, 5, 6]
             ↑  ↑
           left,mid,right

mid = (2+3)/2 = 2
nums[2] = 5 == 5
找到了！返回 2 ✓

視覺化範例 2：nums = [1,3,5,6], target = 2（未找到）

初始: [1, 3, 5, 6], target = 2
       ↑     ↑  ↑
      left  mid right

mid = (0+3)/2 = 1
nums[1] = 3 > 2
right = mid - 1 = 0

       [1, 3, 5, 6]
        ↑
      left,mid,right

mid = (0+0)/2 = 0
nums[0] = 1 < 2
left = mid + 1 = 1

       [1, 3, 5, 6]
           ↑
          left
        ↑
       right

left > right，循環結束
返回 left = 1 ✓（應該插入到索引 1）

mid = (0+3)/2 = 1
nums[1] = 3 < 7
left = mid + 1 = 2

left > right，循環結束

視覺化範例 4：nums = [1,3,5,6], target = 0（插入開頭）

初始: [1, 3, 5, 6], target = 0
       ↑     ↑  ↑
      left  mid right

mid = (0+3)/2 = 1
nums[1] = 3 > 0
right = mid - 1 = 0

        ↑

nums[0] = 1 > 0

     ↑ [1, 3, 5, 6]

left > right，循環結束
返回 left = 0 ✓（應該插入到開頭）

關鍵洞察：

  • 找到目標：直接返回索引
  • 目標太小：left 停在開頭（索引 0）
  • 目標太大：left 停在末尾（索引 n）
  • 目標在中間：left 停在應插入的位置

程式碼實現

class Solution:
    def searchInsert(self, nums: List[int], target: int) -> int:
        """
        二分搜尋
        時間複雜度: O(log n)
        空間複雜度: O(1)
        """
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = left + (right - left) // 2
            
            if nums[mid] == target:
                return mid
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
        
        # 循環結束時，left 就是插入位置
        return left
    
    def searchInsert_builtin(self, nums: List[int], target: int) -> int:
        """
        Python 內建 bisect 模組
        bisect_left 返回第一個 >= target 的位置
        """
        import bisect
        return bisect.bisect_left(nums, target)
#include <vector>
using namespace std;

class Solution {
public:
    /**
     * 二分搜尋
     * 時間複雜度: O(log n)
     * 空間複雜度: O(1)
     */
    int searchInsert(vector& nums, int target) {
        int left = 0;
        int right = nums.size() - 1;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;
            
            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        return left;
    }
};
class Solution {
    /**
     * 二分搜尋
     * 時間複雜度: O(log n)
     * 空間複雜度: O(1)
     */
    public int searchInsert(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;
            
            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        return left;
    }
}

複雜度分析

時間複雜度: O(log n)

每次循環將搜尋範圍減半。

空間複雜度: O(1)

只使用常數額外空間。

重點提示

  • 二分搜尋的核心：
    • 每次排除一半的搜尋空間
    • 需要陣列已排序
    • 時間複雜度 O(log n)
  • 為什麼用 left + (right - left) / 2？
    • 避免 (left + right) 溢位
    • 等價於 (left + right) / 2
    • 更安全的寫法
  • 循環結束條件：
    • left <= right（包含等於）
    • 結束時：left = right + 1
    • left 指向插入位置
    • right 指向小於 target 的最後位置
  • 為什麼返回 left？
    • left 是第一個 >= target 的位置
    • 這正是插入位置的定義
    • 無論找到與否，left 都是答案
  • 三種情況統一處理：
    • 找到目標 → 返回 mid
    • 未找到 → 返回 left（插入位置）
    • 不需要特殊判斷
  • 邊界情況：
    • target 小於所有元素 → left = 0
    • target 大於所有元素 → left = n
    • target 在中間 → left = 正確位置
    • 只有一個元素
  • 常見錯誤：
    • 循環條件寫成 left < right
    • mid 計算溢位
    • 忘記返回 left
    • left/right 更新錯誤（忘記 ±1）
  • 二分搜尋模板：
    • 初始化：left = 0, right = n - 1
    • 循環：while left <= right
    • mid = left + (right - left) / 2
    • 比較後更新 left 或 right
  • 變形理解：
    • 這題是「lower_bound」問題
    • 找第一個 >= target 的位置
    • C++ STL: lower_bound()
    • Python: bisect.bisect_left()
  • 面試技巧：
    • 先畫圖展示二分過程
    • 說明為什麼返回 left
    • 測試邊界情況
    • 提到溢位問題
  • Python bisect 模組：
    • bisect_left: 第一個 >= target
    • bisect_right: 第一個 > target
    • 本題用 bisect_left
  • 延伸思考：
    • 如果有重複元素？
    • 如何找最後一個 <= target？
    • 如何找第一個 > target？
