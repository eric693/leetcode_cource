300. Longest Increasing Subsequence
Medium

題目描述

給定一個整數陣列 nums，找出其中最長嚴格遞增子序列的長度。

子序列是由陣列派生而來的序列，刪除（或不刪除）陣列中的元素而不改變其餘元素的順序。例如，[3,6,2,7] 是陣列 [0,3,1,6,2,2,7] 的子序列。

範例 1:
輸入: nums = [10,9,2,5,3,7,101,18]
輸出: 4
解釋: 最長遞增子序列是 [2,3,7,101]，長度為 4
範例 2:
輸入: nums = [0,1,0,3,2,3]
輸出: 4
解釋: 最長遞增子序列是 [0,1,2,3]，長度為 4
範例 3:
輸入: nums = [7,7,7,7,7,7,7]
輸出: 1
解釋: 最長遞增子序列只能是單個元素，長度為 1
限制條件:
1 <= nums.length <= 2500
-10^4 <= nums[i] <= 10^4
進階:

你能將演算法的時間複雜度降低到 O(n log n) 嗎？

解題思路

方法一：動態規劃 O(n²)

使用動態規劃記錄以每個位置結尾的最長遞增子序列長度。

狀態定義：

dp[i] = 以 nums[i] 結尾的最長遞增子序列的長度

狀態轉移方程：

dp[i] = max(dp[j] + 1) 對所有 j < i 且 nums[j] < nums[i]

步驟：

  1. 初始化 dp 陣列，所有值設為 1（每個元素本身是長度為 1 的子序列）
  2. 對於每個位置 i：
    • 檢查所有 j < i
    • 如果 nums[j] < nums[i]，更新 dp[i] = max(dp[i], dp[j] + 1)
  3. 返回 dp 陣列中的最大值

視覺化範例：nums = [10,9,2,5,3,7,101,18]

索引:  0   1  2  3  4  5   6   7
nums: 10   9  2  5  3  7 101  18
dp:    1   1  1  2  2  3   4   4

計算過程:
i=0: dp[0]=1 (10本身)
i=1: dp[1]=1 (9本身，9<10無法形成遞增)
i=2: dp[2]=1 (2本身)
i=3: dp[3]=2 (2→5，因為2<5，dp[2]+1=2)
i=4: dp[4]=2 (2→3，因為2<3，dp[2]+1=2)
i=5: dp[5]=3 (2→3→7 或 2→5→7)
i=6: dp[6]=4 (2→3→7→101 或 2→5→7→101)
i=7: dp[7]=4 (2→3→7→18 或 2→5→7→18)

答案: max(dp) = 4
方法二：二分搜尋 + 貪心 O(n log n) ⭐

維護一個遞增陣列 tails，其中 tails[i] 表示長度為 i+1 的遞增子序列的最小結尾元素。

核心思想：

對於相同長度的遞增子序列，結尾元素越小，越有利於後續擴展。因此我們維護每個長度對應的最小結尾元素。

步驟：

  1. 創建空陣列 tails
  2. 對於 nums 中的每個數字 num：
    • 如果 tails 為空或 num > tails[-1]，將 num 添加到 tails 末尾
    • 否則，用二分搜尋找到 tails 中第一個 >= num 的位置，將其替換為 num
  3. 返回 tails 的長度

視覺化範例：nums = [10,9,2,5,3,7,101,18]

處理 10: tails = [10]
處理  9: tails = [9]   (替換10，因為9<10且長度相同)
處理  2: tails = [2]   (替換9，更小的結尾更好)
處理  5: tails = [2,5] (5>2，可以擴展)
處理  3: tails = [2,3] (替換5，因為3<5且長度相同)
處理  7: tails = [2,3,7] (7>3，可以擴展)
處理101: tails = [2,3,7,101] (101>7，可以擴展)
處理 18: tails = [2,3,7,18] (替換101，18<101且長度相同)

答案: tails.length = 4

注意: tails 不一定是實際的 LIS，但長度正確！
實際 LIS 可能是 [2,3,7,101] 或 [2,5,7,101] 等

程式碼實現

class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        """
        方法一：動態規劃 O(n²)
        時間複雜度: O(n²)
        空間複雜度: O(n)
        """
        if not nums:
            return 0
        
        n = len(nums)
        # dp[i] 表示以 nums[i] 結尾的最長遞增子序列長度
        dp = [1] * n
        
        for i in range(1, n):
            for j in range(i):
                # 如果 nums[j] < nums[i]，可以將 nums[i] 接在以 nums[j] 結尾的序列後
                if nums[j] < nums[i]:
                    dp[i] = max(dp[i], dp[j] + 1)
        
        return max(dp)
    
    def lengthOfLIS_binary(self, nums: List[int]) -> int:
        """
        方法二：二分搜尋 + 貪心 O(n log n) ⭐ 推薦
        時間複雜度: O(n log n)
        空間複雜度: O(n)
        """
        if not nums:
            return 0
        
        # tails[i] 表示長度為 i+1 的遞增子序列的最小結尾元素
        tails = []
        
        for num in nums:
            # 如果 num 比 tails 中所有元素都大，直接添加
            if not tails or num > tails[-1]:
                tails.append(num)
            else:
                # 二分搜尋找到第一個 >= num 的位置並替換
                left, right = 0, len(tails) - 1
                while left < right:
                    mid = (left + right) // 2
                    if tails[mid] < num:
                        left = mid + 1
                    else:
                        right = mid
                tails[left] = num
        
        return len(tails)
    
    def lengthOfLIS_bisect(self, nums: List[int]) -> int:
        """
        方法三：使用 Python bisect 模組（最簡潔）
        時間複雜度: O(n log n)
        空間複雜度: O(n)
        """
        from bisect import bisect_left
        
        tails = []
        for num in nums:
            pos = bisect_left(tails, num)
            if pos == len(tails):
                tails.append(num)
            else:
                tails[pos] = num
        
        return len(tails)
#include <vector>
#include <algorithm>

class Solution {
public:
    /**
     * 方法一：動態規劃 O(n²)
     * 時間複雜度: O(n²)
     * 空間複雜度: O(n)
     */
    int lengthOfLIS(std::vector& nums) {
        if (nums.empty()) {
            return 0;
        }
        
        int n = nums.size();
        // dp[i] 表示以 nums[i] 結尾的最長遞增子序列長度
        std::vector dp(n, 1);
        
        for (int i = 1; i < n; i++) {
            for (int j = 0; j < i; j++) {
                // 如果 nums[j] < nums[i]，可以將 nums[i] 接在以 nums[j] 結尾的序列後
                if (nums[j] < nums[i]) {
                    dp[i] = std::max(dp[i], dp[j] + 1);
                }
            }
        }
        
        return *std::max_element(dp.begin(), dp.end());
    }
    
    /**
     * 方法二：二分搜尋 + 貪心 O(n log n) ⭐ 推薦
     * 時間複雜度: O(n log n)
     * 空間複雜度: O(n)
     */
    int lengthOfLIS_binary(std::vector& nums) {
        if (nums.empty()) {
            return 0;
        }
        
        // tails[i] 表示長度為 i+1 的遞增子序列的最小結尾元素
        std::vector tails;
        
        for (int num : nums) {
            // 使用 lower_bound 找到第一個 >= num 的位置
            auto it = std::lower_bound(tails.begin(), tails.end(), num);
            
            if (it == tails.end()) {
                // num 比所有元素都大，添加到末尾
                tails.push_back(num);
            } else {
                // 替換找到的位置
                *it = num;
            }
        }
        
        return tails.size();
    }
};
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

class Solution {
    /**
     * 方法一：動態規劃 O(n²)
     * 時間複雜度: O(n²)
     * 空間複雜度: O(n)
     */
    public int lengthOfLIS(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }
        
        int n = nums.length;
        // dp[i] 表示以 nums[i] 結尾的最長遞增子序列長度
        int[] dp = new int[n];
        Arrays.fill(dp, 1);
        
        for (int i = 1; i < n; i++) {
            for (int j = 0; j < i; j++) {
                // 如果 nums[j] < nums[i]，可以將 nums[i] 接在以 nums[j] 結尾的序列後
                if (nums[j] < nums[i]) {
                    dp[i] = Math.max(dp[i], dp[j] + 1);
                }
            }
        }
        
        int maxLen = 0;
        for (int len : dp) {
            maxLen = Math.max(maxLen, len);
        }
        return maxLen;
    }
    
    /**
     * 方法二：二分搜尋 + 貪心 O(n log n) ⭐ 推薦
     * 時間複雜度: O(n log n)
     * 空間複雜度: O(n)
     */
    public int lengthOfLIS_binary(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }
        
        // tails[i] 表示長度為 i+1 的遞增子序列的最小結尾元素
        List tails = new ArrayList<>();
        
        for (int num : nums) {
            // 二分搜尋找到第一個 >= num 的位置
            int pos = binarySearch(tails, num);
            
            if (pos == tails.size()) {
                // num 比所有元素都大，添加到末尾
                tails.add(num);
            } else {
                // 替換找到的位置
                tails.set(pos, num);
            }
        }
        
        return tails.size();
    }
    
    // 二分搜尋：找到第一個 >= target 的位置
    private int binarySearch(List tails, int target) {
        int left = 0, right = tails.size();
        
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (tails.get(mid) < target) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        
        return left;
    }
}

複雜度分析

時間複雜度:
  • 方法一（動態規劃）：O(n²) - 需要雙重循環
  • 方法二（二分搜尋）：O(n log n) - 每個元素進行一次二分搜尋
空間複雜度: O(n)

兩種方法都需要 O(n) 的額外空間來存儲 dp 陣列或 tails 陣列。

重點提示

  • 經典 DP 題：這是動態規劃的經典問題，務必掌握兩種解法
  • 子序列 vs 子陣列：子序列不要求連續，子陣列要求連續
  • 嚴格遞增：注意是「嚴格」遞增（不包含相等），所以用 < 而非 ≤
  • DP 狀態定義：dp[i] 是以 nums[i] 結尾的最長遞增子序列長度
  • 二分搜尋優化：
    • 維護 tails 陣列保持遞增性質
    • tails 陣列不一定是實際的 LIS，但長度一定正確
    • 使用 lower_bound 找第一個 >= num 的位置
  • 面試建議：
    • 先說 O(n²) DP 解法（容易理解）
    • 再優化到 O(n log n)（展現進階能力）
  • 擴展問題：
    • 求 LIS 的具體序列（需要額外記錄路徑）
    • 求最長遞減子序列（反向思考）
    • 求最長連續遞增子序列（子陣列版本）
