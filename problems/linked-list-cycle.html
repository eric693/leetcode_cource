141. Linked List Cycle
Easy

題目描述

給定一個鏈表的頭節點 head，判斷鏈表中是否有環（cycle）。

如果鏈表中有某個節點可以通過連續跟蹤 next 指針再次到達，則鏈表中存在環。

如果鏈表中存在環，則返回 true。否則，返回 false。

範例 1:
輸入: head = [3,2,0,-4], pos = 1
輸出: true
解釋: 鏈表中有一個環，尾節點連接到第二個節點（索引 1）

視覺化:
3 → 2 → 0 → -4
    ↑__________|
範例 2:
輸入: head = [1,2], pos = 0
輸出: true
解釋: 鏈表中有一個環，尾節點連接到第一個節點

視覺化:
1 → 2
↑___|
範例 3:
輸入: head = [1], pos = -1
輸出: false
解釋: 鏈表中沒有環

視覺化:
1 → null
限制條件:
鏈表中節點的數目範圍是 [0, 10^4]
-10^5 <= Node.val <= 10^5
pos 為 -1 或者鏈表中的一個有效索引
進階:

你能用 O(1)（即常數）內存解決此問題嗎？

解題思路

方法一：哈希表記錄訪問過的節點

使用 Set 記錄已經訪問過的節點，如果再次遇到說明有環。

步驟：

  1. 創建一個 Set 存儲訪問過的節點
  2. 遍歷鏈表，每次將當前節點加入 Set
  3. 如果遇到已在 Set 中的節點，返回 true
  4. 如果遍歷到 null，返回 false
方法二：快慢指標（Floyd's Cycle Detection）⭐

使用兩個指標，快指標每次走兩步，慢指標每次走一步。如果有環，兩個指標一定會相遇。

核心思想：

就像兩個人在環形跑道上跑步，跑得快的人最終會追上跑得慢的人。

為什麼一定會相遇？

  • 假設慢指標進入環時，快指標在環中某個位置
  • 每次移動後，快指標與慢指標的距離減少 1
  • 最終距離變為 0，兩個指標相遇

步驟：

  1. 初始化快慢指標都指向 head
  2. 慢指標每次走一步，快指標每次走兩步
  3. 如果快指標到達 null，返回 false（無環）
  4. 如果快指標與慢指標相遇，返回 true（有環）

視覺化範例：有環的情況

鏈表: 3 → 2 → 0 → -4 → (回到 2)

步驟 0: slow=3, fast=3
步驟 1: slow=2, fast=0
步驟 2: slow=0, fast=2
步驟 3: slow=-4, fast=-4  ✓ 相遇！有環

無環的情況:
鏈表: 1 → 2 → 3 → null

步驟 0: slow=1, fast=1
步驟 1: slow=2, fast=3
步驟 2: slow=3, fast=null  ✓ 快指標到達 null，無環
為什麼快指標每次走兩步？
  • 走一步 → 無法和慢指標形成相對速度差
  • 走兩步 → 相對速度差為 1，保證能相遇
  • 走三步或更多 → 可能跳過慢指標，錯過相遇點

程式碼實現

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        """
        方法二：快慢指標（推薦）
        時間複雜度: O(n)
        空間複雜度: O(1)
        """
        # 邊界情況：空鏈表或只有一個節點
        if not head or not head.next:
            return False
        
        # 初始化快慢指標
        slow = head
        fast = head
        
        # 快指標能走兩步時繼續
        while fast and fast.next:
            slow = slow.next        # 慢指標走一步
            fast = fast.next.next   # 快指標走兩步
            
            # 如果相遇，說明有環
            if slow == fast:
                return True
        
        # 快指標到達 null，無環
        return False
    
    def hasCycle_hashset(self, head: Optional[ListNode]) -> bool:
        """
        方法一：哈希表
        時間複雜度: O(n)
        空間複雜度: O(n)
        """
        visited = set()
        current = head
        
        while current:
            # 如果節點已經訪問過，說明有環
            if current in visited:
                return True
            
            # 標記當前節點為已訪問
            visited.add(current)
            current = current.next
        
        # 遍歷完成，沒有環
        return False
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode(int x) : val(x), next(NULL) {}
 * };
 */

#include <unordered_set>

class Solution {
public:
    /**
     * 方法二：快慢指標（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    bool hasCycle(ListNode *head) {
        // 邊界情況
        if (!head || !head->next) {
            return false;
        }
        
        // 初始化快慢指標
        ListNode* slow = head;
        ListNode* fast = head;
        
        // 快指標能走兩步時繼續
        while (fast && fast->next) {
            slow = slow->next;          // 慢指標走一步
            fast = fast->next->next;    // 快指標走兩步
            
            // 如果相遇，說明有環
            if (slow == fast) {
                return true;
            }
        }
        
        // 快指標到達 null，無環
        return false;
    }
    
    /**
     * 方法一：哈希表
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    bool hasCycle_hashset(ListNode *head) {
        std::unordered_set visited;
        ListNode* current = head;
        
        while (current) {
            // 如果節點已經訪問過，說明有環
            if (visited.find(current) != visited.end()) {
                return true;
            }
            
            // 標記當前節點為已訪問
            visited.insert(current);
            current = current->next;
        }
        
        // 遍歷完成，沒有環
        return false;
    }
};
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */

import java.util.HashSet;
import java.util.Set;

public class Solution {
    /**
     * 方法二：快慢指標（推薦）
     * 時間複雜度: O(n)
     * 空間複雜度: O(1)
     */
    public boolean hasCycle(ListNode head) {
        // 邊界情況
        if (head == null || head.next == null) {
            return false;
        }
        
        // 初始化快慢指標
        ListNode slow = head;
        ListNode fast = head;
        
        // 快指標能走兩步時繼續
        while (fast != null && fast.next != null) {
            slow = slow.next;           // 慢指標走一步
            fast = fast.next.next;      // 快指標走兩步
            
            // 如果相遇，說明有環
            if (slow == fast) {
                return true;
            }
        }
        
        // 快指標到達 null，無環
        return false;
    }
    
    /**
     * 方法一：哈希表
     * 時間複雜度: O(n)
     * 空間複雜度: O(n)
     */
    public boolean hasCycle_hashset(ListNode head) {
        Set visited = new HashSet<>();
        ListNode current = head;
        
        while (current != null) {
            // 如果節點已經訪問過，說明有環
            if (visited.contains(current)) {
                return true;
            }
            
            // 標記當前節點為已訪問
            visited.add(current);
            current = current.next;
        }
        
        // 遍歷完成，沒有環
        return false;
    }
}

複雜度分析

方法一：哈希表
  • 時間複雜度：O(n) - 最多訪問每個節點一次
  • 空間複雜度：O(n) - Set 存儲所有節點
方法二：快慢指標（推薦）
  • 時間複雜度：O(n) - 最壞情況下快指標繞環一圈
  • 空間複雜度：O(1) - 只用兩個指標

重點提示

  • Floyd's Cycle Detection：這是經典的快慢指標算法
  • 為什麼叫龜兔賽跑？慢指標像烏龜，快指標像兔子
  • 最優解：快慢指標 O(1) 空間
  • 邊界情況：
    • 空鏈表 → false
    • 只有一個節點 → false
    • 兩個節點成環 → true
  • 循環條件：while (fast && fast.next) 確保可以走兩步
  • 不能用值比較：必須比較節點對象本身
  • 面試技巧：
    • 先說哈希表解法（簡單直觀）
    • 再優化到快慢指標（空間最優）
    • 畫圖演示快慢指標相遇過程
