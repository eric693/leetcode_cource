347. Top K Frequent Elements
Medium

題目描述

給定一個整數陣列 nums 和一個整數 k，返回出現頻率前 k 高的元素。答案可以按任意順序返回。

範例 1:
輸入: nums = [1,1,1,2,2,3], k = 2
輸出: [1,2]
範例 2:
輸入: nums = [1], k = 1
輸出: [1]
限制條件:
  • 1 ≤ nums.length ≤ 105
  • k 的範圍在 [1, 陣列中不同元素的個數] 之間
  • 題目保證答案是唯一的

解題思路

方法一：雜湊表 + 排序

先統計每個元素的出現頻率，然後根據頻率排序，取前 k 個元素。

步驟：

  1. 使用雜湊表統計每個元素的出現次數
  2. 將雜湊表轉換為列表並根據頻率排序
  3. 返回前 k 個元素

時間複雜度：O(n log n)，主要消耗在排序

空間複雜度：O(n)

方法二：堆積（推薦）

使用堆積（Heap）可以在 O(n log k) 時間內找到前 k 個高頻元素。

核心思想：

維護一個大小為 k 的最小堆積，堆積中始終保持頻率最高的 k 個元素。

步驟：

  1. 使用雜湊表統計每個元素的出現頻率
  2. 遍歷雜湊表，使用最小堆積維護頻率最高的 k 個元素
  3. 如果堆積大小超過 k，移除頻率最小的元素
  4. 最後堆積中的元素就是答案

Python 簡化方案：

Python 的 heapq.nlargest() 函數可以直接找出前 k 個最大元素。

方法三：桶排序

可以達到 O(n) 時間複雜度。使用桶排序的思想，將頻率相同的元素放入同一個桶中。

步驟：

  1. 統計頻率
  2. 建立長度為 n+1 的桶陣列，桶的索引代表頻率
  3. 將元素放入對應頻率的桶中
  4. 從高頻率的桶開始收集元素，直到收集 k 個

程式碼實現

from collections import Counter
import heapq

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        """
        方法：Counter + heapq.nlargest
        時間複雜度: O(n log k)
        空間複雜度: O(n)
        """
        # 統計每個元素的出現次數
        count = Counter(nums)
        
        # 使用 heapq.nlargest 找出頻率最高的 k 個元素
        # key=count.get 表示根據頻率進行比較
        return heapq.nlargest(k, count.keys(), key=count.get)
    
    # 方法二：手動維護最小堆積
    def topKFrequent_heap(self, nums: List[int], k: int) -> List[int]:
        """
        手動維護大小為 k 的最小堆積
        """
        count = Counter(nums)
        
        # 建立最小堆積，存儲 (頻率, 元素)
        heap = []
        for num, freq in count.items():
            heapq.heappush(heap, (freq, num))
            # 如果堆積大小超過 k，移除頻率最小的
            if len(heap) > k:
                heapq.heappop(heap)
        
        # 返回堆積中的所有元素
        return [num for freq, num in heap]
#include <vector>
#include <unordered_map>
#include <queue>

class Solution {
public:
    /**
     * 使用最小堆積找出前 k 個高頻元素
     * 時間複雜度: O(n log k)
     * 空間複雜度: O(n)
     */
    std::vector<int> topKFrequent(std::vector<int>& nums, int k) {
        // 統計頻率
        std::unordered_map<int, int> count;
        for (int num : nums) {
            count[num]++;
        }
        
        // 定義最小堆積的比較函數 (頻率小的在堆頂)
        auto cmp = [](std::pair<int, int> a, std::pair<int, int> b) {
            return a.second > b.second;
        };
        
        // 建立最小堆積
        std::priority_queue<std::pair<int, int>, 
                          std::vector<std::pair<int, int>>,
                          decltype(cmp)> heap(cmp);
        
        // 維護大小為 k 的堆積
        for (auto& [num, freq] : count) {
            heap.push({num, freq});
            if (heap.size() > k) {
                heap.pop();
            }
        }
        
        // 收集結果
        std::vector<int> result;
        while (!heap.empty()) {
            result.push_back(heap.top().first);
            heap.pop();
        }
        
        return result;
    }
};
import java.util.*;

class Solution {
    /**
     * 使用最小堆積找出前 k 個高頻元素
     * 時間複雜度: O(n log k)
     * 空間複雜度: O(n)
     */
    public int[] topKFrequent(int[] nums, int k) {
        // 統計頻率
        Map<Integer, Integer> count = new HashMap<>();
        for (int num : nums) {
            count.put(num, count.getOrDefault(num, 0) + 1);
        }
        
        // 建立最小堆積，根據頻率排序
        PriorityQueue<Integer> heap = new PriorityQueue<>(
            (a, b) -> count.get(a) - count.get(b)
        );
        
        // 維護大小為 k 的堆積
        for (int num : count.keySet()) {
            heap.offer(num);
            if (heap.size() > k) {
                heap.poll();
            }
        }
        
        // 收集結果
        int[] result = new int[k];
        for (int i = 0; i < k; i++) {
            result[i] = heap.poll();
        }
        
        return result;
    }
}

複雜度分析

時間複雜度: O(n log k)

統計頻率需要 O(n)，維護大小為 k 的堆積需要 O(n log k)。 如果使用排序方法，時間複雜度為 O(n log n)。

空間複雜度: O(n)

需要雜湊表來儲存頻率統計，最壞情況下需要 O(n) 空間。 堆積最多儲存 k 個元素，空間為 O(k)。

重點提示

  • Python 簡化：使用 Counter 和 heapq.nlargest 可以大幅簡化程式碼
  • 堆積選擇：使用最小堆積而非最大堆積，可以節省空間（只需維護 k 個元素）
  • 時間優化：如果 k 接近 n，排序方法可能更快；如果 k 很小，堆積方法更優
  • 空間優化：桶排序可以達到 O(n) 時間，但實作較複雜
